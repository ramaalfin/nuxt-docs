# Dependent Queries

Dependent queries (also called serial queries or waterfall queries) are queries that must execute sequentially because each one depends on the result of the previous one. TanStack Query handles this elegantly with the `enabled` option, allowing you to declaratively express data dependencies without complex promise chains.

## 1. Basic Dependent Query

The `enabled` option controls whether a query is active. Setting it to a computed reactive value creates a dependency chain.

```typescript
// Fetch user first, then fetch their organization using the user's orgId
export function useUserAndOrg(userId: Ref<string>) {
  // Step 1: Fetch the user
  const userQuery = useQuery({
    queryKey: ['user', userId],
    queryFn: () => $fetch<User>(`/api/users/${userId.value}`)
  })

  // Step 2: Fetch the org — only runs after user data is available
  const orgQuery = useQuery({
    queryKey: ['org', userQuery.data],  // Key includes user data for proper cache keying
    queryFn: () => $fetch<Org>(`/api/orgs/${userQuery.data.value!.orgId}`),
    enabled: computed(() => !!userQuery.data.value?.orgId) // Only enabled when orgId exists
  })

  return {
    user: userQuery.data,
    org: orgQuery.data,
    // Combine loading states
    isPending: computed(() => userQuery.isPending.value || orgQuery.isPending.value),
    isError: computed(() => userQuery.isError.value || orgQuery.isError.value)
  }
}
```

## 2. Multi-Level Dependency Chain

```typescript
// Order → Customer → Customer's payment methods
export function useOrderDetails(orderId: Ref<string>) {
  const order = useQuery({
    queryKey: ['order', orderId],
    queryFn: () => $fetch<Order>(`/api/orders/${orderId.value}`)
  })

  const customer = useQuery({
    queryKey: ['customer', computed(() => order.data.value?.customerId)],
    queryFn: () => $fetch<Customer>(`/api/customers/${order.data.value!.customerId}`),
    enabled: computed(() => !!order.data.value?.customerId)
  })

  const paymentMethods = useQuery({
    queryKey: ['payment-methods', computed(() => customer.data.value?.id)],
    queryFn: () => $fetch<PaymentMethod[]>(`/api/customers/${customer.data.value!.id}/payment-methods`),
    enabled: computed(() => !!customer.data.value?.id)
  })

  return { order, customer, paymentMethods }
}
```

## 3. Conditional Queries

Enable a query based on a user action or UI state:

```typescript
// Only search when the user has typed at least 3 characters
const searchQuery = ref('')

const searchResults = useQuery({
  queryKey: ['search', searchQuery],
  queryFn: () => $fetch('/api/search', { query: { q: searchQuery.value } }),
  enabled: computed(() => searchQuery.value.trim().length >= 3),
  // Keep previous results visible while new query is loading
  placeholderData: keepPreviousData,
})
```

## 4. Avoiding Dependent Query Waterfalls

Dependent queries execute serially, which means 3 dependent queries = 3 round trips in sequence. When possible, restructure your API to return combined data, or use `prefetchQuery` to run them closer to simultaneously.

```typescript
// ❌ SLOW: 3 serial round trips = Total time: RTT1 + RTT2 + RTT3
const user = useQuery({ queryKey: ['user'], queryFn: fetchUser })
const org = useQuery({
  queryKey: ['org'],
  queryFn: () => fetchOrg(user.data.value!.orgId),
  enabled: computed(() => !!user.data.value)
})

// ✅ FAST: Single round trip with combined data
const userWithOrg = useQuery({
  queryKey: ['user', 'with-org'],
  queryFn: () => $fetch('/api/users/me?include=org')
})
```

## 5. Using `select` for Data Transformation

Instead of creating a dependent query just to transform data, use the `select` option:

```typescript
const ordersQuery = useQuery({
  queryKey: ['orders'],
  queryFn: fetchOrders,
  // Transform data in-cache without a separate query
  select: (orders) => ({
    pending: orders.filter(o => o.status === 'pending'),
    completed: orders.filter(o => o.status === 'completed'),
    total: orders.length
  })
})

// Data is already transformed when accessed
const { pending, completed, total } = ordersQuery.data.value ?? {}
```
