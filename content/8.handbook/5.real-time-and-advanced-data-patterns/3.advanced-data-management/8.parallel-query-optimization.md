# Parallel Query Optimization

Parallel queries execute multiple independent queries simultaneously, reducing total data loading time. If you need data from 3 independent endpoints, running them in parallel takes the time of the slowest request instead of the sum of all three.

## 1. `useQueries` for Dynamic Parallel Queries

`useQueries` allows you to run a variable number of queries in parallel, where the number of queries isn't known at compile time.

```typescript
import { useQueries } from '@tanstack/vue-query'

// Fetch multiple products in parallel
export function useMultipleProducts(productIds: Ref<string[]>) {
  return useQueries({
    queries: computed(() =>
      productIds.value.map(id => ({
        queryKey: ['product', id],
        queryFn: () => $fetch<Product>(`/api/products/${id}`),
        staleTime: 1000 * 60 * 5
      }))
    ),
    // Combine all results into a single computed object
    combine: (results) => ({
      data: results.map(r => r.data).filter(Boolean) as Product[],
      isPending: results.some(r => r.isPending),
      isError: results.some(r => r.isError)
    })
  })
}
```

## 2. Side-by-Side Independent Queries (Static)

For a fixed number of independent queries (e.g., a dashboard loading multiple widgets), simply call `useQuery` multiple times — they execute in parallel by default.

```typescript
// Dashboard: All 4 queries execute simultaneously
export function useDashboard() {
  const stats = useQuery({
    queryKey: ['dashboard', 'stats'],
    queryFn: fetchDashboardStats
  })

  const recentOrders = useQuery({
    queryKey: ['orders', 'recent'],
    queryFn: fetchRecentOrders
  })

  const revenue = useQuery({
    queryKey: ['revenue', 'monthly'],
    queryFn: fetchMonthlyRevenue
  })

  const topProducts = useQuery({
    queryKey: ['products', 'top'],
    queryFn: fetchTopProducts
  })

  // Combined loading state
  const isPending = computed(() =>
    stats.isPending.value ||
    recentOrders.isPending.value ||
    revenue.isPending.value ||
    topProducts.isPending.value
  )

  return { stats, recentOrders, revenue, topProducts, isPending }
}
```

## 3. Prefetching for Parallel Page Hydration

Prefetch multiple queries simultaneously on the server side to hydrate the page without any loading states.

```typescript
// pages/dashboard.vue
<script setup lang="ts">
const queryClient = useQueryClient()

// Run all prefetches in parallel on the server
await Promise.all([
  queryClient.prefetchQuery({ queryKey: ['dashboard', 'stats'], queryFn: fetchDashboardStats }),
  queryClient.prefetchQuery({ queryKey: ['orders', 'recent'], queryFn: fetchRecentOrders }),
  queryClient.prefetchQuery({ queryKey: ['revenue', 'monthly'], queryFn: fetchMonthlyRevenue }),
])

// All queries are already in cache — no loading states on mount
const { data: stats } = useQuery({ queryKey: ['dashboard', 'stats'], queryFn: fetchDashboardStats })
const { data: orders } = useQuery({ queryKey: ['orders', 'recent'], queryFn: fetchRecentOrders })
</script>
```

## 4. Batching API Requests

When you need data for many entities (e.g., profiles for 50 user IDs), sending 50 individual requests is inefficient. Use request batching.

```typescript
// composables/useUserBatch.ts
// DataLoader-style batching: collect individual requests, fire one batch request
let batchIds: string[] = []
let batchTimer: ReturnType<typeof setTimeout> | null = null
let batchResolvers: Map<string, (user: User) => void> = new Map()

const executeBatch = async () => {
  const ids = [...batchIds]
  batchIds = []
  
  const users = await $fetch<User[]>('/api/users/batch', {
    method: 'POST',
    body: { ids }
  })
  
  users.forEach(user => {
    batchResolvers.get(user.id)?.(user)
    batchResolvers.delete(user.id)
  })
}

export function useUserById(userId: string) {
  return useQuery({
    queryKey: ['user', userId],
    queryFn: () => new Promise<User>((resolve) => {
      batchIds.push(userId)
      batchResolvers.set(userId, resolve)
      
      // Debounce — collect IDs for 10ms then send one batch request
      batchTimer && clearTimeout(batchTimer)
      batchTimer = setTimeout(executeBatch, 10)
    }),
    staleTime: 1000 * 60 * 5
  })
}
```

## 5. Waterfall Prevention Checklist

- [ ] All independent queries are called in parallel (same component, no `await` between them).
- [ ] SSR prefetches multiple queries with `Promise.all`.
- [ ] List views use `useQueries` instead of mapping over IDs with sequential fetches.
- [ ] API endpoints support batch parameters (`/api/users?ids=1,2,3`) where needed.
- [ ] Dependent queries are minimized — prefer combined API endpoints over serial queries.
