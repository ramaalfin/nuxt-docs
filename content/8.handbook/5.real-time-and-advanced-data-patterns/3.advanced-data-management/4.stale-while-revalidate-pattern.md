# Stale-While-Revalidate Pattern

Stale-While-Revalidate (SWR) is a cache invalidation strategy where the cache immediately returns stale (possibly outdated) data to the user for instant display, while simultaneously fetching fresh data from the network in the background to update the cache.

The result is an application that feels instantaneous (no loading spinner on revisit) while eventually displaying up-to-date data.

## 1. The Core Flow

```
User visits /products
  ↓
Is there cached data?
  ├── YES → Return stale cache immediately (user sees content instantly)
  │          → Fetch fresh data from network in background
  │          → When fetch completes → Update cache + re-render with fresh data
  └── NO  → Show loading state → Fetch → Cache → Display
```

## 2. SWR in TanStack Query

TanStack Query implements SWR natively. The `staleTime` controls when data is considered stale and the background revalidation is triggered.

```typescript
const { data, isFetching } = useQuery({
  queryKey: ['products'],
  queryFn: fetchProducts,
  
  // Data is "fresh" for 60 seconds — no background refetch within this window
  staleTime: 1000 * 60,
  
  // After staleTime expires, the next mount/focus triggers a background refetch
  // 'isFetching' is true during the background refetch, 'isPending' is false
  // (because we have stale data to show)
})
```

**Key distinction:**
- `isPending`: `true` when there is NO data in cache at all (first load).
- `isFetching`: `true` whenever a network request is in-flight (including background revalidation).

## 3. Visual Indicators for Background Revalidation

Show a subtle indicator so users know the data may be updating, without a jarring full reload.

```vue
<template>
  <div class="product-list">
    <!-- Subtle global refresh indicator -->
    <div v-if="isFetching && !isPending" class="revalidation-indicator">
      <SpinnerIcon class="animate-spin text-xs opacity-50" />
      <span class="sr-only">Refreshing data...</span>
    </div>

    <!-- Full skeleton only on initial load -->
    <ProductSkeleton v-if="isPending" />
    
    <!-- Show stale data immediately, update in place when fresh data arrives -->
    <ProductGrid v-else :products="data" :class="{ 'opacity-80': isFetching }" />
  </div>
</template>
```

## 4. SWR for Initial Page Loads (SSR + Hydration)

In Nuxt, you can pre-populate the TanStack Query cache on the server, send it to the client, and avoid any loading flash on initial page load.

```typescript
// pages/products.vue
<script setup lang="ts">
// useAsyncData runs on the server, populates the cache
const queryClient = useQueryClient()

await useAsyncData('products', async () => {
  const products = await $fetch('/api/products')
  
  // Pre-populate TanStack Query cache with server-fetched data
  queryClient.setQueryData(['products'], products)
  
  return products // Also passed as Nuxt's payload for hydration
})

// useQuery on client picks up the pre-populated cache (no flash, no refetch)
const { data: products } = useQuery({
  queryKey: ['products'],
  queryFn: fetchProducts,
  staleTime: 1000 * 60 * 5
})
</script>
```

## 5. Controlled Revalidation Triggers

```typescript
// Revalidate when user returns to the tab (default: true)
useQuery({ queryKey: ['products'], queryFn: fetchProducts, refetchOnWindowFocus: true })

// Revalidate when network reconnects (default: true)
useQuery({ queryKey: ['products'], queryFn: fetchProducts, refetchOnReconnect: true })

// Active polling — revalidate every 30 seconds regardless of focus
useQuery({
  queryKey: ['dashboard-metrics'],
  queryFn: fetchMetrics,
  refetchInterval: 1000 * 30,
  // Only poll when the tab is visible (saves resources on background tabs)
  refetchIntervalInBackground: false
})

// Manual trigger
const { refetch } = useQuery({ queryKey: ['products'], queryFn: fetchProducts })
// User clicks "Refresh" button
const onRefresh = () => refetch()
```

## 6. Comparing SWR Approaches

| Approach | First Load | Revisit | Background Update |
|---|---|---|---|
| No cache | Spinner | Spinner | N/A |
| Server-side only | SSR fast | Spinner | N/A |
| SWR (TanStack Query) | SSR fast | Instant (stale) | Yes, silently |
| Cache-only | Instant | Instant | No |
