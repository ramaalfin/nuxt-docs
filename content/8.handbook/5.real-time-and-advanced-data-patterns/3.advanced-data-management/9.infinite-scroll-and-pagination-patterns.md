# Infinite Scroll & Pagination Patterns

Loading large datasets requires careful UX and architectural decisions. The two primary patterns — traditional pagination and infinite scroll — have different use cases, tradeoffs, and implementation requirements.

## 1. Choosing the Right Pattern

| Pattern | Best For | Avoid For |
|---|---|---|
| **Pagination** | Tables, admin UIs, search results, data users need to navigate | Feeds, social timelines, discovery |
| **Infinite Scroll** | Content feeds, social timelines, photo galleries | Tables, search results requiring navigation |
| **Load More (button)** | Middle ground — user controls loading | N/A |

**Key rule:** If users need to return to a specific position, use pagination. If users are browsing linearly, use infinite scroll.

## 2. TanStack Query Infinite Scroll

```typescript
// composables/useFeed.ts
import { useInfiniteQuery } from '@tanstack/vue-query'

export function useFeed(filters: Ref<FeedFilters>) {
  return useInfiniteQuery({
    queryKey: ['feed', filters],
    
    queryFn: async ({ pageParam }) => {
      const response = await $fetch<FeedResponse>('/api/feed', {
        query: {
          cursor: pageParam,  // Cursor-based pagination (preferred over offset)
          limit: 20,
          ...filters.value
        }
      })
      return response
    },
    
    initialPageParam: undefined as string | undefined,
    
    // How to get the cursor for the next page from the last page's response
    getNextPageParam: (lastPage) => lastPage.nextCursor ?? undefined,
    
    // Optional: Support "load previous" direction
    getPreviousPageParam: (firstPage) => firstPage.prevCursor ?? undefined,
    
    staleTime: 1000 * 60 * 2,
  })
}
```

## 3. Intersection Observer for Auto-Load

```vue
<script setup lang="ts">
const { data, fetchNextPage, hasNextPage, isFetchingNextPage } = useFeed(filters)
const allItems = computed(() => data.value?.pages.flatMap(p => p.items) ?? [])

// Sentinel element at the bottom of the list
const sentinel = ref<HTMLElement | null>(null)

// Load more when sentinel comes into view
useIntersectionObserver(
  sentinel,
  ([entry]) => {
    if (entry.isIntersecting && hasNextPage.value && !isFetchingNextPage.value) {
      fetchNextPage()
    }
  },
  { rootMargin: '200px' } // Start loading 200px before the bottom is reached
)
</script>

<template>
  <ul>
    <FeedItem v-for="item in allItems" :key="item.id" :item="item" />
  </ul>
  
  <!-- Loading state for next page -->
  <div v-if="isFetchingNextPage">
    <FeedItemSkeleton v-for="i in 3" :key="i" />
  </div>
  
  <!-- Sentinel: triggers fetchNextPage via IntersectionObserver -->
  <div ref="sentinel" v-if="hasNextPage" aria-hidden="true" />
  
  <p v-else-if="!isFetchingNextPage" class="text-center text-muted">
    You've seen everything! 🎉
  </p>
</template>
```

## 4. Cursor-Based vs Offset-Based Pagination

**Offset Pagination:** `GET /api/posts?page=3&limit=20`
- ❌ Items shift when new content is inserted (user sees duplicates or skips items).
- ❌ Slow on large datasets (`OFFSET 10000` requires scanning 10,000 rows).
- ✅ Simple to implement. Supports jumping to arbitrary pages.

**Cursor-Based Pagination:** `GET /api/posts?cursor=<lastId>&limit=20`
- ✅ Stable — new content insertions don't affect pagination.
- ✅ O(1) database query using the indexed cursor column.
- ❌ Cannot jump to arbitrary pages.

**Recommendation:** Use cursor-based for infinite scroll/feeds. Use offset-based for admin tables that need page jumping.

## 5. Virtual List for Extremely Long Lists

When rendering thousands of items (even with infinite scroll), the DOM becomes slow. Use virtual scrolling to render only visible items.

```vue
<script setup lang="ts">
// Using @tanstack/vue-virtual (recommended)
import { useVirtualizer } from '@tanstack/vue-virtual'

const parentEl = ref<HTMLElement | null>(null)

const virtualizer = useVirtualizer({
  count: allItems.value.length,
  getScrollElement: () => parentEl.value,
  estimateSize: () => 72, // Estimated item height in px
  overscan: 5 // Render 5 extra items outside the viewport
})

const virtualItems = computed(() => virtualizer.value.getVirtualItems())
</script>

<template>
  <div ref="parentEl" style="height: 600px; overflow-y: auto;">
    <div :style="{ height: `${virtualizer.getTotalSize()}px`, position: 'relative' }">
      <div
        v-for="vItem in virtualItems"
        :key="vItem.key"
        :style="{
          position: 'absolute',
          top: 0,
          left: 0,
          transform: `translateY(${vItem.start}px)`,
          width: '100%',
          height: `${vItem.size}px`
        }"
      >
        <FeedItem :item="allItems[vItem.index]" />
      </div>
    </div>
  </div>
</template>
```
