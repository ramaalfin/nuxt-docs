# Mutation Synchronization

Mutation synchronization refers to managing the lifecycle of write operations (creates, updates, deletes) — ensuring mutations are properly sequenced, deduplicated, tracked, and reflected across all related cached data. In applications with high-frequency mutations, this becomes a critical concern.

## 1. The Problem with Concurrent Mutations

When users can trigger multiple mutations rapidly (fast clicking, form auto-save), uncontrolled concurrent requests can cause:
- **Race conditions:** Response to mutation #2 arrives before mutation #1, displaying stale data.
- **Duplicate mutations:** Double-clicking a submit button sends two requests.
- **UI thrashing:** Optimistic updates reversing and re-applying rapidly.

## 2. Mutation Deduplication & Queuing

```typescript
// composables/useMutationQueue.ts
// Ensures mutations execute serially, not concurrently
export function useMutationQueue() {
  const queue = ref<Array<() => Promise<any>>>([])
  const isProcessing = ref(false)

  const process = async () => {
    if (isProcessing.value || queue.value.length === 0) return
    isProcessing.value = true

    while (queue.value.length > 0) {
      const mutation = queue.value.shift()!
      try {
        await mutation()
      } catch (error) {
        console.error('Mutation failed:', error)
      }
    }

    isProcessing.value = false
  }

  const enqueue = (mutation: () => Promise<any>) => {
    queue.value.push(mutation)
    process()
  }

  return { enqueue, isProcessing, queueLength: computed(() => queue.value.length) }
}
```

## 3. TanStack Query Mutation Tracking

Use `useMutationState` to observe the state of in-flight mutations globally.

```typescript
import { useMutationState } from '@tanstack/vue-query'

// Observe all mutations with a specific key (useful for global indicators)
const pendingMutations = useMutationState({
  filters: { mutationKey: ['task-updates'], status: 'pending' },
  select: (mutation) => mutation.state.variables
})

// Show a "Saving..." indicator when any mutation is in-flight
const isSaving = computed(() => pendingMutations.value.length > 0)
```

## 4. Auto-Save with Debounced Mutations

For form auto-save, debounce the mutation to avoid sending a request on every keystroke.

```typescript
// composables/useAutoSave.ts
export function useAutoSave<T>(data: Ref<T>, saveFn: (data: T) => Promise<void>) {
  const saveStatus = ref<'idle' | 'saving' | 'saved' | 'error'>('idle')
  const lastSavedData = ref<T | null>(null)

  const save = useDebounceFn(async () => {
    // Skip if data hasn't changed
    if (JSON.stringify(data.value) === JSON.stringify(lastSavedData.value)) return

    saveStatus.value = 'saving'
    try {
      await saveFn(data.value)
      lastSavedData.value = JSON.parse(JSON.stringify(data.value))
      saveStatus.value = 'saved'
    } catch {
      saveStatus.value = 'error'
    }
  }, 1000) // Debounce — save 1 second after last change

  // Trigger save on data change
  watch(data, save, { deep: true })

  // Force save before the user leaves the page
  useEventListener(window, 'beforeunload', (e) => {
    if (saveStatus.value === 'saving') {
      e.preventDefault()
      e.returnValue = 'Changes are being saved...'
    }
  })

  return { saveStatus }
}
```

## 5. Mutation Side Effects: Updating Related Queries

A single mutation often affects multiple cached queries. Explicitly manage all affected caches.

```typescript
const { mutate: completeOrder } = useMutation({
  mutationFn: (orderId: string) => $fetch(`/api/orders/${orderId}/complete`, { method: 'POST' }),
  
  onSuccess: (completedOrder) => {
    // 1. Update the specific order in cache
    queryClient.setQueryData(['order', completedOrder.id], completedOrder)
    
    // 2. Remove from "pending orders" list
    queryClient.setQueryData<Order[]>(['orders', 'pending'], (old) =>
      old?.filter(o => o.id !== completedOrder.id)
    )
    
    // 3. Invalidate dashboard (which shows order statistics)
    queryClient.invalidateQueries({ queryKey: ['dashboard'] })
    
    // 4. Invalidate user's order history
    queryClient.invalidateQueries({ queryKey: ['user', completedOrder.userId, 'orders'] })
  }
})
```

## 6. Mutation State Persistence

For critical mutations (payments, order submissions), persist their state to survive page reloads.

```typescript
// Persist pending mutation variables to localStorage
const { mutate: submitOrder } = useMutation({
  mutationKey: ['submit-order'],
  mutationFn: submitOrderFn,
  
  onMutate: (variables) => {
    // Persist the mutation intent — survive a page refresh
    localStorage.setItem('pending-order', JSON.stringify(variables))
  },
  
  onSettled: () => {
    localStorage.removeItem('pending-order')
  }
})

// On app startup: resume any interrupted mutations
onMounted(async () => {
  const pending = localStorage.getItem('pending-order')
  if (pending) {
    const order = JSON.parse(pending)
    // Verify with server if the order was actually submitted
    const status = await $fetch(`/api/orders/check?idempotencyKey=${order.idempotencyKey}`)
    if (status.submitted) {
      localStorage.removeItem('pending-order') // Already done
    } else {
      submitOrder(order) // Resume the mutation
    }
  }
})
```
