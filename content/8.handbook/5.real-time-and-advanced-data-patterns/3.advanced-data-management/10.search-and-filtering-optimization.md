# Search & Filtering Optimization

Search and filtering are among the most performance-sensitive UI patterns. Every keystroke can trigger an expensive API call or heavy local computation. Poorly optimized search creates a sluggish experience that drives users away.

## 1. Debounced Search

Never search on every keystroke. Debounce the search to wait until the user pauses typing.

```typescript
// composables/useSearch.ts
export function useSearch<T>(fetcher: (query: string) => Promise<T[]>) {
  const rawQuery = ref('')
  const debouncedQuery = refDebounced(rawQuery, 300) // Wait 300ms after last keystroke

  const { data, isPending, isError } = useQuery({
    queryKey: ['search', debouncedQuery],
    queryFn: () => fetcher(debouncedQuery.value),
    enabled: computed(() => debouncedQuery.value.trim().length >= 2),
    placeholderData: keepPreviousData, // Show previous results while new query loads
    staleTime: 1000 * 60 * 5,
  })

  return {
    query: rawQuery,   // Bind to input for instant visual feedback
    results: data,
    isPending: computed(() => isPending.value && debouncedQuery.value.length > 0),
    isError
  }
}
```

```vue
<template>
  <div class="search-container">
    <input
      v-model="query"
      type="search"
      placeholder="Search products..."
      :aria-busy="isPending"
    />
    <!-- Show spinner only when actively searching (not on empty query) -->
    <SpinnerIcon v-if="isPending" class="search-spinner" />
    <SearchResultsList :items="results ?? []" />
  </div>
</template>
```

## 2. URL-Synced Filters

Persist search and filter state in the URL so users can share, bookmark, and use the browser back button.

```typescript
// composables/useFilteredProducts.ts
export function useFilteredProducts() {
  const route = useRoute()
  const router = useRouter()

  // Read filters from URL
  const filters = computed<ProductFilters>(() => ({
    search: route.query.q as string ?? '',
    category: route.query.category as string ?? 'all',
    minPrice: Number(route.query.minPrice) || 0,
    maxPrice: Number(route.query.maxPrice) || Infinity,
    sortBy: (route.query.sort as string) ?? 'relevance'
  }))

  // Write filters to URL (debounced for search query)
  const updateFilter = useDebounceFn((key: string, value: string | number) => {
    router.push({
      query: {
        ...route.query,
        [key]: value || undefined // Remove empty values from URL
      }
    })
  }, 300)

  const { data: products, isPending } = useQuery({
    queryKey: ['products', filters],
    queryFn: () => $fetch('/api/products', { query: filters.value }),
    placeholderData: keepPreviousData,
  })

  return { filters, products, isPending, updateFilter }
}
```

## 3. Client-Side Filtering for Small Datasets

When the full dataset is small enough to fetch upfront (< 500 items), filter locally for instant response.

```typescript
// Fetch everything once, filter/sort on client
const { data: allProducts } = useQuery({
  queryKey: ['products'],
  queryFn: () => $fetch<Product[]>('/api/products'),
  staleTime: Infinity // Static data — never refetch
})

const filteredProducts = computed(() => {
  if (!allProducts.value) return []
  
  return allProducts.value
    .filter(p => {
      if (filters.search && !p.name.toLowerCase().includes(filters.search.toLowerCase())) return false
      if (filters.category !== 'all' && p.categoryId !== filters.category) return false
      if (p.price < filters.minPrice || p.price > filters.maxPrice) return false
      return true
    })
    .sort((a, b) => {
      if (filters.sortBy === 'price-asc') return a.price - b.price
      if (filters.sortBy === 'price-desc') return b.price - a.price
      return 0
    })
})
```

## 4. Search-as-You-Type with keepPreviousData

Avoid the jarring "flash to empty" state between search terms by keeping previous results visible while the new query loads.

```typescript
import { keepPreviousData } from '@tanstack/vue-query'

const { data, isPending, isFetching } = useQuery({
  queryKey: ['search', debouncedQuery],
  queryFn: () => searchProducts(debouncedQuery.value),
  // Keep showing the previous results while new ones load
  placeholderData: keepPreviousData,
})

// Dim the results slightly to indicate staleness
// isPending = true only on initial load (no previous data)
// isFetching = true on every background refetch
```

```vue
<!-- Results dim while fetching new results (previous ones stay visible) -->
<div :class="{ 'opacity-60 pointer-events-none': isFetching && !isPending }">
  <SearchResult v-for="result in data" :key="result.id" :result="result" />
</div>
```

## 5. Faceted Filtering with Independent Queries

Load filter options (categories, brands, price ranges) independently from the results, so filters don't reload when results change.

```typescript
// Filter options are static — load once
const { data: categories } = useQuery({
  queryKey: ['categories'],
  queryFn: fetchCategories,
  staleTime: Infinity
})

const { data: brands } = useQuery({
  queryKey: ['brands'],
  queryFn: fetchBrands,
  staleTime: 1000 * 60 * 60 // 1 hour
})

// Results depend on the active filters
const { data: results } = useQuery({
  queryKey: ['search', activeFilters],
  queryFn: () => searchWithFilters(activeFilters.value),
  placeholderData: keepPreviousData
})
```
