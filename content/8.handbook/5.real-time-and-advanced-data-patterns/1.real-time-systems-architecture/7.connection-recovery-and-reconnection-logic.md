# Connection Recovery & Reconnection Logic

WebSocket connections are inherently fragile. Mobile networks drop, laptops sleep, Wi-Fi routers restart, and servers deploy new versions. A production-quality real-time application must handle disconnections gracefully—reconnecting automatically, recovering missed messages, and restoring UI state without requiring the user to manually refresh the page.

## 1. The Reconnection State Machine

A WebSocket connection exists in a state machine with well-defined transitions:

```
         connect()
CLOSED ──────────────► CONNECTING
  ▲                        │
  │          onopen         │
  │    ┌─────────────────► OPEN
  │    │                    │
  │  close()           onerror/onclose
  │    │                    │
  └────┴─────────────── RECONNECTING ──► (back to CONNECTING)
```

## 2. Exponential Backoff Reconnection

Never reconnect at a fixed rate after a failure — you will hammer the server. Use **exponential backoff with jitter** to spread reconnection attempts across time.

```typescript
// composables/useReconnectingWebSocket.ts
interface ReconnectOptions {
  maxRetries?: number
  baseDelay?: number
  maxDelay?: number
  onReconnect?: (attempt: number) => void
  onMaxRetriesExceeded?: () => void
}

export function useReconnectingWebSocket(url: string, options: ReconnectOptions = {}) {
  const {
    maxRetries = 10,
    baseDelay = 1000,
    maxDelay = 30_000,
    onReconnect,
    onMaxRetriesExceeded
  } = options

  const socket = ref<WebSocket | null>(null)
  const status = ref<'connecting' | 'open' | 'reconnecting' | 'closed'>('closed')
  const retryCount = ref(0)
  const lastMessage = ref<any>(null)
  let reconnectTimer: ReturnType<typeof setTimeout> | null = null

  const getBackoffDelay = (attempt: number): number => {
    // Exponential backoff + random jitter to prevent thundering herd
    const exponential = baseDelay * Math.pow(2, attempt)
    const jitter = Math.random() * 1000
    return Math.min(exponential + jitter, maxDelay)
  }

  const connect = () => {
    if (socket.value?.readyState === WebSocket.OPEN) return

    socket.value = new WebSocket(url)
    status.value = retryCount.value > 0 ? 'reconnecting' : 'connecting'

    socket.value.onopen = () => {
      status.value = 'open'
      retryCount.value = 0 // Reset on successful connection
      onReconnect?.(retryCount.value)
    }

    socket.value.onmessage = (event: MessageEvent) => {
      try {
        lastMessage.value = JSON.parse(event.data)
      } catch {
        lastMessage.value = event.data
      }
    }

    socket.value.onclose = (event: CloseEvent) => {
      // Code 1000 = normal closure — don't reconnect
      if (event.code === 1000 || event.wasClean) {
        status.value = 'closed'
        return
      }

      if (retryCount.value >= maxRetries) {
        status.value = 'closed'
        onMaxRetriesExceeded?.()
        return
      }

      const delay = getBackoffDelay(retryCount.value)
      retryCount.value++
      status.value = 'reconnecting'

      console.log(`[WS] Reconnecting in ${Math.round(delay)}ms (attempt ${retryCount.value}/${maxRetries})`)
      reconnectTimer = setTimeout(connect, delay)
    }
  }

  const close = () => {
    reconnectTimer && clearTimeout(reconnectTimer)
    socket.value?.close(1000, 'Normal closure') // 1000 = intentional, prevents reconnect
  }

  const send = (data: object) => {
    if (socket.value?.readyState === WebSocket.OPEN) {
      socket.value.send(JSON.stringify(data))
    }
  }

  onMounted(connect)
  onUnmounted(close)

  return { status, lastMessage, retryCount, send, connect, close }
}
```

## 3. Heartbeat / Keep-Alive Mechanism

Some network proxies and load balancers silently close idle WebSocket connections (typically after 60-90 seconds). A heartbeat keeps the connection alive.

```typescript
// Add heartbeat to the reconnecting WebSocket composable
const startHeartbeat = () => {
  const heartbeatInterval = setInterval(() => {
    if (socket.value?.readyState === WebSocket.OPEN) {
      socket.value.send(JSON.stringify({ type: 'PING' }))
    }
  }, 30_000) // Ping every 30 seconds

  // If PONG is not received within 5 seconds, force reconnect
  const pongTimeout = ref<ReturnType<typeof setTimeout> | null>(null)
  
  socket.value!.addEventListener('message', (event) => {
    const data = JSON.parse(event.data)
    if (data.type === 'PONG') {
      pongTimeout.value && clearTimeout(pongTimeout.value)
    }
  })

  onUnmounted(() => clearInterval(heartbeatInterval))
}
```

## 4. Message Recovery: Catching Up After Reconnection

When a client reconnects, it may have missed messages that were broadcast while it was offline. Use a **cursor-based message log** to replay missed messages.

```typescript
// Server: Store a short message history with sequence numbers
const messageLog = new Map<string, { seq: number; messages: Message[] }>()

// Client: Send last known sequence number on reconnection
const reconnectWithCatchup = (lastSeq: number) => {
  send({ type: 'SUBSCRIBE', roomId, lastSeq }) // Tell server where we left off
}

// Server: Replay missed messages
if (data.type === 'SUBSCRIBE' && data.lastSeq) {
  const log = messageLog.get(data.roomId)
  const missed = log?.messages.filter(m => m.seq > data.lastSeq) ?? []
  missed.forEach(m => peer.send(JSON.stringify(m)))
}
```

## 5. UI Feedback During Reconnection

```vue
<template>
  <Transition name="slide-down">
    <div v-if="status === 'reconnecting'" class="reconnect-banner" role="status">
      <SpinnerIcon class="animate-spin" />
      Reconnecting... (attempt {{ retryCount }})
    </div>
    <div v-else-if="status === 'closed'" class="disconnect-banner" role="alert">
      Connection lost. 
      <button @click="connect()">Reconnect manually</button>
    </div>
  </Transition>
</template>
```

## 6. Reconnection on Tab Visibility Change

When a user returns to a hidden tab, reconnect immediately rather than waiting for the next backoff timer.

```typescript
useEventListener(document, 'visibilitychange', () => {
  if (document.visibilityState === 'visible' && status.value !== 'open') {
    reconnectTimer && clearTimeout(reconnectTimer) // Cancel the scheduled retry
    connect() // Attempt immediately
  }
})
```
