# Presence Awareness

Presence awareness is the feature that shows users who else is currently viewing or editing the same content. It includes online/offline status, cursor positions, active selections, and "typing" indicators. Done well, it dramatically increases the collaborative feeling of an application.

## 1. Online Status (User Presence)

The simplest form of presence: showing which users are currently connected to the application.

### Server-Side: Tracking Connected Clients

```typescript
// server/utils/presence-store.ts
// In-memory store (use Redis in production for multi-server setups)
const onlineUsers = new Map<string, { userId: string; connectedAt: Date }>()

export const presenceStore = {
  add: (peerId: string, userId: string) => {
    onlineUsers.set(peerId, { userId, connectedAt: new Date() })
  },
  remove: (peerId: string) => {
    onlineUsers.delete(peerId)
  },
  getOnlineUserIds: () => [...new Set([...onlineUsers.values()].map(u => u.userId))],
}
```

```typescript
// server/routes/ws.ts
import { defineWebSocketHandler } from 'h3'
import { presenceStore } from '../utils/presence-store'

export default defineWebSocketHandler({
  open(peer) {
    const userId = peer.request.headers.get('X-User-ID')!
    presenceStore.add(peer.id, userId)

    // Broadcast updated presence list to all connected clients
    peer.publish('presence', JSON.stringify({
      type: 'PRESENCE_UPDATE',
      onlineUsers: presenceStore.getOnlineUserIds()
    }))
  },

  close(peer) {
    presenceStore.remove(peer.id)
    peer.publish('presence', JSON.stringify({
      type: 'PRESENCE_UPDATE',
      onlineUsers: presenceStore.getOnlineUserIds()
    }))
  }
})
```

### Client-Side: Reactive Online Status

```typescript
// composables/usePresence.ts
export function usePresence() {
  const onlineUsers = ref<string[]>([])
  const { lastMessage } = useWebSocket('wss://api.yourapp.com/ws')

  watch(lastMessage, (message) => {
    if (message?.type === 'PRESENCE_UPDATE') {
      onlineUsers.value = message.onlineUsers
    }
  })

  return { onlineUsers }
}
```

## 2. Cursor Tracking (Live Cursor Positions)

Showing other users' cursor positions in real-time, as seen in Figma, Notion, and Google Docs.

### Throttling Cursor Events (Critical)

Mouse move events fire at 60fps. Sending a WebSocket message at 60fps per user would destroy your server. Always **throttle** cursor position broadcasts.

```typescript
// composables/useCursorTracking.ts
import { useThrottleFn } from '@vueuse/core'

export function useCursorTracking(send: (data: object) => void, roomId: string) {
  const remoteCursors = ref<Map<string, { x: number; y: number; user: User }>>(new Map())

  // Throttle to max 20 updates per second (50ms intervals)
  const sendCursorPosition = useThrottleFn((event: MouseEvent) => {
    const container = document.getElementById('canvas-area')
    if (!container) return

    const rect = container.getBoundingClientRect()
    send({
      type: 'CURSOR_MOVE',
      roomId,
      position: {
        x: ((event.clientX - rect.left) / rect.width) * 100, // % based for responsive
        y: ((event.clientY - rect.top) / rect.height) * 100,
      }
    })
  }, 50)

  return { remoteCursors, sendCursorPosition }
}
```

### Rendering Remote Cursors

```vue
<template>
  <div id="canvas-area" @mousemove="sendCursorPosition">
    <!-- Your collaborative content -->
    <slot />

    <!-- Render a cursor for each remote user -->
    <TransitionGroup name="cursor">
      <div
        v-for="[userId, cursor] in remoteCursors"
        :key="userId"
        class="remote-cursor"
        :style="{
          left: `${cursor.position.x}%`,
          top: `${cursor.position.y}%`,
          '--cursor-color': cursor.user.color
        }"
      >
        <CursorIcon :style="{ color: cursor.user.color }" />
        <span class="cursor-label">{{ cursor.user.name }}</span>
      </div>
    </TransitionGroup>
  </div>
</template>

<style scoped>
.remote-cursor {
  position: absolute;
  pointer-events: none; /* Don't interfere with interactions */
  transition: left 0.1s linear, top 0.1s linear; /* Smooth interpolation */
  z-index: 100;
}
</style>
```

## 3. Typing Indicators

Show when another user is actively typing in a message input.

```typescript
// composables/useTypingIndicator.ts
export function useTypingIndicator(send: (data: object) => void, channelId: string) {
  const typingUsers = ref<Set<string>>(new Set())
  const typingTimers = new Map<string, ReturnType<typeof setTimeout>>()
  
  // Debounced "stop typing" — user is considered to have stopped after 2s of inactivity
  const sendTyping = useDebounceFn(() => {
    send({ type: 'TYPING_STOP', channelId })
  }, 2000)
  
  const onInputChange = () => {
    send({ type: 'TYPING_START', channelId })
    sendTyping() // Schedule "stop typing" after 2s of inactivity
  }

  // Handle incoming typing events from other users
  const handleMessage = (message: any) => {
    if (message.type === 'TYPING_START') {
      typingUsers.value.add(message.userId)
      // Auto-clear if stop message is delayed
      const timer = setTimeout(() => typingUsers.value.delete(message.userId), 3000)
      typingTimers.set(message.userId, timer)
    } else if (message.type === 'TYPING_STOP') {
      typingUsers.value.delete(message.userId)
      clearTimeout(typingTimers.get(message.userId))
    }
  }

  return { typingUsers, onInputChange, handleMessage }
}
```

## 4. Yjs Awareness for Presence (Recommended for Complex Cases)

For applications already using **Yjs** (CRDTs) for collaborative editing, the built-in **Awareness** protocol provides presence and cursor tracking with minimal implementation effort.

```typescript
const { awareness } = useCollaborativeDoc(docId)

// Set the local user's state (shared with all connected clients)
awareness.setLocalStateField('user', {
  name: currentUser.name,
  color: currentUser.color,
  cursor: { anchor: null, head: null } // Cursor position in the document
})

// Listen for remote changes
awareness.on('change', () => {
  const states = Array.from(awareness.getStates().entries())
  const remoteUsers = states
    .filter(([clientId]) => clientId !== awareness.clientID)
    .map(([, state]) => state.user)
  
  collaborators.value = remoteUsers
})
```
