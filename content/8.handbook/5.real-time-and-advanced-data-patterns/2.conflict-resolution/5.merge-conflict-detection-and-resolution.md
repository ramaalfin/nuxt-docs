# Merge Conflict Detection & Resolution

Even with sophisticated CRDTs and semantic merging strategies, some conflicts require explicit human resolution. A production system must be able to detect these true conflicts, present them clearly to users, and provide tools to resolve them without data loss.

## 1. What is a True Conflict?

A true conflict (also called a "semantic conflict") occurs when two concurrent changes cannot be automatically merged because they express contradictory intent that a computer cannot resolve:
- User A and User B both edit the same title field to different values.
- Two team members assign the same task to different people simultaneously.
- Two users delete and modify the same record simultaneously.

## 2. Conflict Detection with Version Vectors

Version vectors (also called vector clocks) track which operations each node has seen, enabling precise detection of concurrent updates.

```typescript
// types/version-vector.ts
type VersionVector = Map<string, number> // nodeId -> last-seen sequence number

const increment = (vv: VersionVector, nodeId: string): VersionVector => {
  const newVV = new Map(vv)
  newVV.set(nodeId, (newVV.get(nodeId) ?? 0) + 1)
  return newVV
}

const isConcurrent = (vv1: VersionVector, vv2: VersionVector): boolean => {
  // vv1 and vv2 are concurrent if neither "dominates" the other
  let vv1HasHigher = false
  let vv2HasHigher = false

  const allNodes = new Set([...vv1.keys(), ...vv2.keys()])
  
  for (const node of allNodes) {
    const v1 = vv1.get(node) ?? 0
    const v2 = vv2.get(node) ?? 0
    if (v1 > v2) vv1HasHigher = true
    if (v2 > v1) vv2HasHigher = true
  }

  return vv1HasHigher && vv2HasHigher // Both have operations the other doesn't
}

// Example:
const vv1 = new Map([['alice', 3], ['bob', 1]])  // Alice made op 3, saw Bob's op 1
const vv2 = new Map([['alice', 2], ['bob', 2]])  // Bob made op 2, only saw Alice's op 2

console.log(isConcurrent(vv1, vv2)) // true — concurrent conflict!
```

## 3. Conflict Storage Model

```typescript
// Types for conflict tracking
interface ConflictRecord<T> {
  id: string
  entityId: string
  entityType: string
  conflictDetectedAt: string
  versions: Array<{
    versionVector: Record<string, number>
    value: T
    author: string
    modifiedAt: string
  }>
  status: 'pending' | 'resolved'
  resolvedValue?: T
  resolvedBy?: string
}
```

## 4. Conflict Resolution UI

When a conflict is detected, present it clearly to the user with full context.

```vue
<!-- components/ConflictResolver.vue -->
<script setup lang="ts">
const props = defineProps<{
  conflict: ConflictRecord<DocumentState>
}>()
const emit = defineEmits<{ resolve: [value: DocumentState] }>()

const selectedVersion = ref<DocumentState | null>(null)

const resolveWithVersion = (version: DocumentState) => {
  emit('resolve', version)
}

const resolveWithCustomValue = () => {
  if (selectedVersion.value) emit('resolve', selectedVersion.value)
}
</script>

<template>
  <div class="conflict-resolver" role="dialog" aria-label="Resolve editing conflict">
    <div class="conflict-header">
      <ExclamationTriangleIcon />
      <h2>Editing Conflict Detected</h2>
      <p>Two people made conflicting changes to this document. Please choose which version to keep.</p>
    </div>

    <div class="conflict-versions">
      <div
        v-for="(version, index) in conflict.versions"
        :key="index"
        class="conflict-version"
        :class="{ selected: selectedVersion === version.value }"
        @click="selectedVersion = version.value"
      >
        <div class="version-meta">
          <UserAvatar :user="version.author" />
          <span>{{ version.author }}</span>
          <time>{{ formatRelativeTime(version.modifiedAt) }}</time>
        </div>
        <div class="version-preview">
          <pre>{{ JSON.stringify(version.value, null, 2) }}</pre>
        </div>
        <button @click.stop="resolveWithVersion(version.value)">
          Keep this version
        </button>
      </div>
    </div>
  </div>
</template>
```

## 5. Server-Side Conflict Detection Pattern

```typescript
// server/api/documents/[id].patch.ts
export default defineEventHandler(async (event) => {
  const { id } = getRouterParams(event)
  const { value, versionVector } = await readBody(event)

  const current = await db.documents.findById(id)

  if (isConcurrent(current.versionVector, versionVector)) {
    // Store the conflict for user resolution
    await db.conflicts.create({
      entityId: id,
      entityType: 'document',
      versions: [
        { value: current.value, versionVector: current.versionVector, author: current.lastModifiedBy },
        { value, versionVector, author: getAuthUser(event).id }
      ],
      status: 'pending'
    })

    throw createError({
      statusCode: 409,
      statusMessage: 'Conflict Detected',
      data: { conflictId: conflict.id }
    })
  }

  // No conflict — apply the update
  await db.documents.update(id, { value, versionVector: merge(current.versionVector, versionVector) })
  return { success: true }
})
```

## 6. Automatic Resolution Policies

For lower-priority fields, define automatic resolution policies to reduce user interruption:

```typescript
const autoResolve = (conflict: ConflictRecord<any>): any | null => {
  const [v1, v2] = conflict.versions

  // Policy: For 'tags', always take the union
  if (Array.isArray(v1.value) && Array.isArray(v2.value)) {
    return [...new Set([...v1.value, ...v2.value])]
  }

  // Policy: For numeric counters, take the max
  if (typeof v1.value === 'number' && typeof v2.value === 'number') {
    return Math.max(v1.value, v2.value)
  }

  // Cannot auto-resolve — return null to trigger user resolution UI
  return null
}
```
