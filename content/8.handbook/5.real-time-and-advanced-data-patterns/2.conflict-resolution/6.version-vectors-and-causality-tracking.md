# Version Vectors & Causality Tracking

Version vectors (also called vector clocks) are a mathematical tool for tracking causality in distributed systems. They allow you to answer the question: "Did event A happen before event B, or did they happen concurrently?"

This causality information is the foundation of correct conflict detection and CRDT merging.

## 1. The Causality Problem

In a distributed system with no global clock, timestamps alone cannot determine the order of events:
- Server A records an event at `T=100`.
- Server B records an event at `T=99`.
- Is Server A's event truly "after" Server B's? Not necessarily — the clocks may be skewed by 10ms due to NTP drift.

Vector clocks solve this by counting events per node, rather than relying on wall-clock time.

## 2. How Vector Clocks Work

Each node in the system maintains a counter for every known node. When an event occurs:
1. The node increments its own counter.
2. The updated vector is attached to every outgoing message.
3. When a message is received, the receiver merges its vector with the message's vector (taking the max of each component).

```typescript
// types/vector-clock.ts
type VectorClock = Record<string, number>

export const vectorClock = {
  create: (): VectorClock => ({}),
  
  increment: (vc: VectorClock, nodeId: string): VectorClock => ({
    ...vc,
    [nodeId]: (vc[nodeId] ?? 0) + 1
  }),
  
  // Merge two vector clocks by taking max of each component
  merge: (vc1: VectorClock, vc2: VectorClock): VectorClock => {
    const allNodes = new Set([...Object.keys(vc1), ...Object.keys(vc2)])
    const merged: VectorClock = {}
    allNodes.forEach(node => {
      merged[node] = Math.max(vc1[node] ?? 0, vc2[node] ?? 0)
    })
    return merged
  },
  
  // vc1 "happened before" vc2 (vc1 ≤ vc2 component-wise)
  happenedBefore: (vc1: VectorClock, vc2: VectorClock): boolean => {
    const allNodes = new Set([...Object.keys(vc1), ...Object.keys(vc2)])
    return [...allNodes].every(node => (vc1[node] ?? 0) <= (vc2[node] ?? 0))
  },

  // Events are concurrent if neither happened before the other
  areConcurrent: (vc1: VectorClock, vc2: VectorClock): boolean => (
    !vectorClock.happenedBefore(vc1, vc2) && !vectorClock.happenedBefore(vc2, vc1)
  )
}
```

## 3. Practical Example

```typescript
// Node "alice" and "bob" both modify a shared document

let aliceClock: VectorClock = {}
let bobClock: VectorClock = {}

// Alice makes edit 1
aliceClock = vectorClock.increment(aliceClock, 'alice') // { alice: 1 }
// Alice sends her edit to bob

// Bob receives alice's edit and merges
bobClock = vectorClock.merge(bobClock, aliceClock)      // { alice: 1 }

// Bob makes his own edit (he has seen alice's op 1)
bobClock = vectorClock.increment(bobClock, 'bob')       // { alice: 1, bob: 1 }

// Alice makes another edit WITHOUT seeing bob's latest
aliceClock = vectorClock.increment(aliceClock, 'alice') // { alice: 2 }

// Are alice's { alice: 2 } and bob's { alice: 1, bob: 1 } concurrent?
console.log(vectorClock.areConcurrent(aliceClock, bobClock)) // true — concurrent conflict!
```

## 4. Hybrid Logical Clocks (HLC)

Pure vector clocks grow with the number of nodes (O(N) space). For large distributed systems, **Hybrid Logical Clocks (HLC)** combine physical timestamps with logical clocks to provide:
- Total ordering compatible with NTP (unlike pure logical clocks).
- Compact representation (a single 64-bit integer).
- Causality guarantees (unlike pure wall-clock time).

```typescript
// Simplified HLC implementation
interface HLC {
  wallTime: number  // Physical time (ms)
  logical: number   // Logical counter for same-millisecond events
  nodeId: string
}

const hlc = {
  now: (nodeId: string, lastHLC?: HLC): HLC => {
    const wall = Date.now()
    const lastWall = lastHLC?.wallTime ?? 0
    const lastLogical = lastHLC?.logical ?? 0

    if (wall > lastWall) {
      return { wallTime: wall, logical: 0, nodeId }
    } else {
      return { wallTime: lastWall, logical: lastLogical + 1, nodeId }
    }
  },

  compare: (a: HLC, b: HLC): number => {
    if (a.wallTime !== b.wallTime) return a.wallTime - b.wallTime
    if (a.logical !== b.logical) return a.logical - b.logical
    return a.nodeId.localeCompare(b.nodeId) // Tie-break by nodeId
  }
}
```

## 5. Using Causality in a Nuxt Application

```typescript
// stores/collaborative-doc.ts
export const useCollaborativeDocStore = defineStore('collab-doc', () => {
  const myNodeId = useRuntimeConfig().public.nodeId || Math.random().toString(36)
  let myClock: VectorClock = {}
  
  const applyOperation = (op: Operation & { clock: VectorClock }) => {
    // 1. Check if this operation is causally ready
    // (all operations it depends on have been applied)
    if (!isCausallyReady(op.clock, myClock)) {
      // Buffer it until its dependencies arrive
      causalBuffer.push(op)
      return
    }
    
    // 2. Apply the operation
    applyOpToState(op)
    
    // 3. Update our clock
    myClock = vectorClock.merge(myClock, op.clock)
    
    // 4. Check if any buffered operations are now ready
    flushCausalBuffer()
  }
  
  const submitOp = (payload: any) => {
    myClock = vectorClock.increment(myClock, myNodeId)
    
    send({
      type: 'OPERATION',
      payload,
      clock: { ...myClock }  // Attach our current vector clock
    })
  }

  return { applyOperation, submitOp }
})
```
