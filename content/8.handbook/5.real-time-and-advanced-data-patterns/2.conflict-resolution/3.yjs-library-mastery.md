# Y.js Library Mastery

Yjs is a high-performance CRDT framework for building collaborative applications. It is used in production by Loom, Dynaboard, Cargo, and many others. This guide covers the advanced Yjs patterns needed to build production-quality collaborative features in Nuxt.

## 1. Yjs Data Types Cheatsheet

```typescript
import * as Y from 'yjs'

const ydoc = new Y.Doc()

// Y.Text — collaborative text (plain or rich)
const ytext = ydoc.getText('content')
ytext.insert(0, 'Hello')
ytext.insert(5, ' World')
console.log(ytext.toString()) // "Hello World"

// Y.Map — collaborative key-value store
const ymap = ydoc.getMap('settings')
ymap.set('theme', 'dark')
ymap.set('language', 'en')
ymap.get('theme') // 'dark'

// Y.Array — collaborative ordered list
const yarray = ydoc.getArray('tasks')
yarray.push([{ id: 1, title: 'Task 1', done: false }])
yarray.insert(0, [{ id: 0, title: 'First Task', done: false }])
yarray.toArray() // [{ id: 0... }, { id: 1... }]

// Y.XmlFragment — for rich text editors (ProseMirror, TipTap)
const yfragment = ydoc.getXmlFragment('prosemirror')
```

## 2. Observers & Reactivity

```typescript
// Observe Y.Text changes
ytext.observe((event) => {
  console.log('Text changed:', ytext.toString())
  console.log('Delta:', event.changes.delta) // [{retain: 5}, {insert: ' World'}]
})

// Deep observe Y.Map (includes nested changes)
ymap.observeDeep((events) => {
  events.forEach(event => {
    event.changes.keys.forEach((change, key) => {
      console.log(`Key "${key}" was ${change.action}:`, ymap.get(key))
    })
  })
})

// Batch multiple changes into a single transaction (single undo step)
ydoc.transact(() => {
  ymap.set('title', 'New Title')
  ymap.set('updatedAt', Date.now())
  yarray.push([{ id: Date.now() }])
}, 'my-origin') // Second arg is the "origin" (for filtering local vs remote)
```

## 3. Sync Providers

Providers handle the network transport of Yjs updates between peers.

### y-websocket (Self-Hosted)
```typescript
import { WebsocketProvider } from 'y-websocket'

const provider = new WebsocketProvider(
  'wss://sync.yourapp.com',  // Your y-websocket server
  'room-id',                  // Unique document/room identifier
  ydoc,
  {
    connect: true,
    // Include auth token in WebSocket URL
    params: { token: authStore.accessToken }
  }
)

provider.on('status', ({ status }) => {
  console.log('Sync status:', status) // 'connecting', 'connected', 'disconnected'
})

provider.on('synced', (isSynced) => {
  console.log('Document synced with server:', isSynced)
})
```

### y-indexeddb (Offline Persistence)
```typescript
import { IndexeddbPersistence } from 'y-indexeddb'

const persistence = new IndexeddbPersistence('room-id', ydoc)

persistence.on('synced', () => {
  // Local IndexedDB is synced with in-memory Yjs doc
  // The document is now ready to display (even if WebSocket isn't connected yet)
  isReady.value = true
})
```

### Combining Providers (Offline-First Pattern)
```typescript
// Load from IndexedDB first (instant load, even offline)
const persistence = new IndexeddbPersistence(roomId, ydoc)

// Then sync with server (merges remote changes into local doc)
const provider = new WebsocketProvider(wsUrl, roomId, ydoc)

// The two providers work together: local changes save to IndexedDB AND sync to server
```

## 4. Undo Manager

```typescript
import { UndoManager } from 'yjs'

// Create undo manager tracking specific shared types
const undoManager = new UndoManager([ytext, ytasks], {
  captureTimeout: 500,       // Group changes within 500ms as one undo step
  trackedOrigins: new Set([null]) // Only track local changes (null origin)
})

// Expose undo/redo to the UI
const canUndo = computed(() => undoManager.canUndo())
const canRedo = computed(() => undoManager.canRedo())

const undo = () => undoManager.undo()
const redo = () => undoManager.redo()
```

## 5. Awareness for Presence

```typescript
// Each client can share ephemeral state (cursor, name, color)
// Awareness data is NOT stored in the document — it's transient
const { awareness } = provider

// Set local user state
awareness.setLocalStateField('user', {
  name: currentUser.name,
  color: currentUser.profileColor,
  cursor: null  // Will be updated on selection change
})

// React to remote awareness changes
const collaborators = ref<User[]>([])
awareness.on('change', () => {
  collaborators.value = Array.from(awareness.getStates().entries())
    .filter(([clientId]) => clientId !== awareness.clientID)
    .map(([, state]) => state.user)
    .filter(Boolean)
})
```

## 6. Selective Sync (Subdocuments)

For large documents, use Yjs subdocuments to lazy-load content.

```typescript
// Parent document holds references to subdocuments
const parentDoc = new Y.Doc()
const pages = parentDoc.getMap('pages')

// Create a subdocument for each page (only sync the active one)
const createPage = (pageId: string) => {
  const pageDoc = new Y.Doc()
  pages.set(pageId, pageDoc)
  return pageDoc
}

// Load a specific page's content on demand
const loadPage = (pageId: string) => {
  const pageDoc = pages.get(pageId) as Y.Doc
  pageDoc.load() // Triggers sync for this subdoc only
  return pageDoc
}
```
