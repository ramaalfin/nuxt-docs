# CRDT — Conflict-free Replicated Data Types

CRDTs are data structures designed from the ground up to be merged without conflicts. Unlike OT (which requires a server to order operations), CRDTs provide **mathematical guarantees** that any two nodes that have received the same set of updates will converge to the same state—regardless of the order in which those updates arrived.

This makes CRDTs ideal for offline-first, peer-to-peer, and distributed systems.

## 1. The Core CRDT Guarantee

A CRDT must satisfy three mathematical properties:
1. **Commutativity:** `merge(A, B) = merge(B, A)` — Order of merging doesn't matter.
2. **Associativity:** `merge(A, merge(B, C)) = merge(merge(A, B), C)` — Grouping doesn't matter.
3. **Idempotency:** `merge(A, A) = A` — Merging the same update twice has no effect.

Any data structure satisfying these properties can be safely replicated without central coordination.

## 2. Common CRDT Types

### G-Counter (Grow-only Counter)
```typescript
// Each node has its own counter. Total = sum of all node counters.
class GCounter {
  private counts: Map<string, number> = new Map()
  
  increment(nodeId: string) {
    this.counts.set(nodeId, (this.counts.get(nodeId) ?? 0) + 1)
  }
  
  value(): number {
    return [...this.counts.values()].reduce((sum, v) => sum + v, 0)
  }
  
  merge(other: GCounter): GCounter {
    const merged = new GCounter()
    const allNodes = new Set([...this.counts.keys(), ...other.counts.keys()])
    allNodes.forEach(node => {
      merged.counts.set(node, Math.max(
        this.counts.get(node) ?? 0,
        other.counts.get(node) ?? 0
      ))
    })
    return merged
  }
}
```

### LWW-Register (Last Write Wins)
```typescript
// The write with the highest timestamp wins on merge
class LWWRegister<T> {
  constructor(
    private value: T,
    private timestamp: number = Date.now()
  ) {}
  
  set(value: T) {
    this.value = value
    this.timestamp = Date.now()
  }
  
  merge(other: LWWRegister<T>): LWWRegister<T> {
    if (other.timestamp > this.timestamp) {
      return new LWWRegister(other.value, other.timestamp)
    }
    return this
  }
}
```

### OR-Set (Observed-Remove Set)
An OR-Set allows both add and remove operations without conflicts. Each element carries a unique tag; remove operations only remove specific tagged versions.

## 3. Yjs: The Production CRDT Library

Building CRDTs from scratch is error-prone. **Yjs** is the battle-tested production CRDT library for JavaScript. It provides:
- `Y.Text` — Collaborative plain text
- `Y.XmlFragment` — Collaborative rich text (for ProseMirror, TipTap, Quill)
- `Y.Map` — Collaborative key-value map
- `Y.Array` — Collaborative ordered list

```typescript
// composables/useCRDTDocument.ts
import * as Y from 'yjs'
import { WebsocketProvider } from 'y-websocket'

export function useCRDTDocument(roomId: string) {
  const ydoc = new Y.Doc()
  
  // Connect to a Yjs sync server (handles CRDT merging between peers)
  const provider = new WebsocketProvider('wss://yjs.yourapp.com', roomId, ydoc)
  
  // Shared data structures — automatically synchronized
  const sharedText = ydoc.getText('content')
  const sharedMeta = ydoc.getMap('metadata')
  const sharedTasks = ydoc.getArray('tasks')
  
  // Make Vue reactive
  const textContent = ref(sharedText.toString())
  sharedText.observe(() => {
    textContent.value = sharedText.toString()
  })
  
  const tasks = ref<any[]>(sharedTasks.toArray())
  sharedTasks.observe(() => {
    tasks.value = sharedTasks.toArray()
  })

  const addTask = (task: object) => {
    sharedTasks.push([task])
  }

  const updateMeta = (key: string, value: any) => {
    sharedMeta.set(key, value)
  }

  onUnmounted(() => {
    provider.destroy()
    ydoc.destroy()
  })

  return { textContent, tasks, addTask, updateMeta, ydoc, provider }
}
```

## 4. CRDT State Persistence

```typescript
// Persist Yjs document to server for late joiners
import { encodeStateAsUpdate, applyUpdate } from 'yjs'

// Save the full document state
const saveDoc = async (ydoc: Y.Doc, roomId: string) => {
  const update = encodeStateAsUpdate(ydoc)
  await $fetch(`/api/rooms/${roomId}/state`, {
    method: 'PUT',
    body: update
  })
}

// Load document state for new joiners
const loadDoc = async (ydoc: Y.Doc, roomId: string) => {
  const savedState = await $fetch<Uint8Array>(`/api/rooms/${roomId}/state`)
  if (savedState) {
    applyUpdate(ydoc, new Uint8Array(savedState))
  }
}
```

## 5. CRDTs for Offline-First

CRDTs shine in offline scenarios. Changes made offline are stored as CRDT updates. When the client reconnects, updates are merged without any server coordination.

```typescript
import { encodeStateAsUpdate, mergeUpdates } from 'yjs'
import { IndexeddbPersistence } from 'y-indexeddb'

// Persist changes locally while offline
const persistence = new IndexeddbPersistence(roomId, ydoc)

persistence.on('synced', () => {
  console.log('Offline changes loaded from IndexedDB')
})

// When back online, the WebsocketProvider automatically syncs and merges
```
