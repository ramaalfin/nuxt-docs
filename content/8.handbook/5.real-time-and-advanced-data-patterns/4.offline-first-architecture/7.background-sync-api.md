# Background Sync API

The Background Sync API allows Service Workers to defer actions until the user has a stable internet connection. Unlike the application-layer sync queue (which only runs when the app is open), Background Sync works even when the browser tab is closed or the app is in the background—making it the most resilient sync mechanism available.

## 1. How Background Sync Works

```
User makes a change (offline)
  ↓
Change saved to IndexedDB
Service Worker registers a 'sync' tag
  ↓
Browser waits for stable connectivity (even if app is closed)
  ↓
Browser fires 'sync' event in Service Worker
Service Worker reads from IndexedDB and sends to server
  ↓
Success → Remove from queue
Failure → Browser retries automatically (exponential backoff, for up to several days)
```

## 2. Registering a Sync Event

```typescript
// composables/useBackgroundSync.ts
export function useBackgroundSync() {
  const isSupported = 'serviceWorker' in navigator && 'SyncManager' in window

  const registerSync = async (tag: string = 'mutation-queue') => {
    if (!isSupported) {
      console.warn('[BackgroundSync] Not supported — using foreground sync fallback')
      return false
    }

    try {
      const registration = await navigator.serviceWorker.ready
      await (registration as any).sync.register(tag)
      console.log(`[BackgroundSync] Registered sync tag: ${tag}`)
      return true
    } catch (error) {
      console.error('[BackgroundSync] Failed to register:', error)
      return false
    }
  }

  // Call this after enqueuing a mutation
  const notifySync = async () => {
    const registered = await registerSync('mutation-queue')
    if (!registered) {
      // Fallback: run sync in the foreground immediately if online
      if (navigator.onLine) {
        const { processBatch } = useSyncWorker()
        await processBatch()
      }
    }
  }

  return { isSupported, registerSync, notifySync }
}
```

## 3. Service Worker Sync Handler

```typescript
// public/sw.js (or sw.ts processed by Vite PWA)
import { openDB } from 'idb'

// Sync handler fires when connectivity is available
self.addEventListener('sync', (event) => {
  if (event.tag === 'mutation-queue') {
    event.waitUntil(processMutationQueue())
  }
  
  if (event.tag === 'analytics-flush') {
    event.waitUntil(flushAnalytics())
  }
})

async function processMutationQueue() {
  const db = await openDB('app-db', 1)
  const tx = db.transaction('syncQueue', 'readwrite')
  const allOps = await tx.store.index('status').getAll('pending')
  
  for (const op of allOps) {
    try {
      const response = await fetch(op.endpoint, {
        method: op.method,
        headers: { 'Content-Type': 'application/json' },
        body: op.body ? JSON.stringify(op.body) : undefined
      })
      
      if (response.ok) {
        await db.delete('syncQueue', op.id)
      } else if (response.status === 409) {
        // Conflict — mark and stop retrying automatically
        await db.put('syncQueue', { ...op, status: 'conflict' })
      }
      // Non-OK, non-409: throw to trigger auto-retry by the browser
      else if (!response.ok) throw new Error(`HTTP ${response.status}`)
      
    } catch {
      // Throwing from the sync event causes the browser to retry
      throw new Error('Sync failed — will retry')
    }
  }
}
```

## 4. Periodic Background Sync (Experimental)

The **Periodic Background Sync API** allows Service Workers to run on a schedule even when the app is closed — useful for pre-fetching content and refreshing stale data.

```typescript
// Register periodic sync (requires PWA install and user permission)
const registerPeriodicSync = async () => {
  const registration = await navigator.serviceWorker.ready
  
  if ('periodicSync' in registration) {
    const status = await navigator.permissions.query({ name: 'periodic-background-sync' as any })
    
    if (status.state === 'granted') {
      await (registration as any).periodicSync.register('content-refresh', {
        minInterval: 24 * 60 * 60 * 1000 // At most once per day
      })
    }
  }
}
```

```typescript
// sw.ts — Handle periodic sync
self.addEventListener('periodicsync', (event: any) => {
  if (event.tag === 'content-refresh') {
    event.waitUntil(refreshCachedContent())
  }
})

async function refreshCachedContent() {
  const cache = await caches.open('runtime-cache')
  const requests = await cache.keys()
  
  await Promise.all(
    requests.map(async (request) => {
      try {
        const response = await fetch(request)
        if (response.ok) await cache.put(request, response)
      } catch { /* Ignore failures */ }
    })
  )
}
```

## 5. Browser Support & Fallbacks

| Feature | Chrome | Firefox | Safari |
|---|---|---|---|
| **Background Sync** | ✅ Stable | ❌ Not supported | ❌ Not supported |
| **Periodic Background Sync** | ✅ (Chrome 80+) | ❌ | ❌ |

Always implement a fallback for browsers without Background Sync support:

```typescript
const { notifySync, isSupported } = useBackgroundSync()

const saveChanges = async (data: any) => {
  // 1. Always save locally first
  await db.syncQueue.add({ ...data, status: 'pending' })
  
  // 2. Try Background Sync for reliable delivery
  await notifySync() // Falls back to foreground sync if not supported
}
```
