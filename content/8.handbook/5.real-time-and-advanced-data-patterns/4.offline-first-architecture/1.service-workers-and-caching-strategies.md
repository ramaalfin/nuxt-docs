# Service Workers & Caching Strategies

A Service Worker is a JavaScript file that runs in a separate thread from the main browser thread. It acts as a programmable network proxy, intercepting all network requests made by your application and enabling powerful offline capabilities, advanced caching, and background sync.

## 1. Service Worker Lifecycle

```
Install → Waiting → Activate → Idle → Fetch/Message/Sync
```

- **Install:** SW downloads and caches specified assets (`precaching`).
- **Waiting:** New SW waits for old SW's clients to close before activating.
- **Activate:** Old SW removed, new SW takes control. Clean up old caches here.
- **Idle:** SW is ready, listening for events (fetch, push, sync).

## 2. Nuxt with PWA Module (Recommended)

Rather than hand-writing Service Workers, use `@vite-pwa/nuxt` which generates a production-ready Service Worker powered by Workbox.

```bash
pnpm add -D @vite-pwa/nuxt
```

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  modules: ['@vite-pwa/nuxt'],
  pwa: {
    registerType: 'autoUpdate',
    manifest: {
      name: 'My App',
      short_name: 'App',
      theme_color: '#1a56db',
      icons: [
        { src: '/icons/icon-192.png', sizes: '192x192', type: 'image/png' },
        { src: '/icons/icon-512.png', sizes: '512x512', type: 'image/png' },
      ]
    },
    workbox: {
      navigateFallback: '/',
      globPatterns: ['**/*.{js,css,html,png,svg,ico}'],
      runtimeCaching: [
        {
          urlPattern: /^https:\/\/api\.yourapp\.com\/.*/i,
          handler: 'StaleWhileRevalidate',
          options: {
            cacheName: 'api-cache',
            expiration: { maxEntries: 100, maxAgeSeconds: 60 * 60 * 24 } // 24 hours
          }
        }
      ]
    }
  }
})
```

## 3. Workbox Caching Strategies

Workbox provides pre-built, battle-tested caching strategy implementations.

### Cache First
Serve from cache, update cache in the background. Best for static assets.
```
Request → Cache? → YES → Return cache → Update cache from network (async)
                   NO  → Network → Cache it → Return response
```

### Network First
Try network, fall back to cache on failure. Best for frequently-changing data where freshness matters.
```
Request → Network → Success → Update cache → Return response
                 → Fail    → Cache? → Return cache (stale)
                            → NO    → Offline fallback
```

### Stale-While-Revalidate
Return cache immediately (fast), update cache in the background. Best for APIs where some staleness is acceptable.
```
Request → Return cache immediately (if available)
        → Fetch from network in background → Update cache for next time
```

### Cache Only
Only serve from cache — no network. Use for pre-cached assets that never change.

## 4. Precaching App Shell

The App Shell model precaches the minimal HTML, CSS, and JS required to display the UI framework. The content is then loaded dynamically.

```typescript
// sw.ts — Custom service worker (if not using Workbox)
import { precacheAndRoute } from 'workbox-precaching'

// Injected by vite-pwa at build time
precacheAndRoute(self.__WB_MANIFEST)

// Catch-all for navigation requests (SPA support)
import { NavigationRoute, Router } from 'workbox-routing'
import { NetworkFirst } from 'workbox-strategies'

const navigationRoute = new NavigationRoute(
  new NetworkFirst({ cacheName: 'pages' }),
  { denylist: [/\/api\//] } // Don't cache API routes
)
```

## 5. Updating Service Workers

When a new version of your app is deployed, inform users and prompt a reload.

```typescript
// composables/usePWAUpdate.ts
export function usePWAUpdate() {
  const { needRefresh, updateServiceWorker } = useRegisterSW({
    onRegistered(r) {
      // Check for updates every hour
      setInterval(() => r?.update(), 60 * 60 * 1000)
    }
  })

  return { needRefresh, updateServiceWorker }
}
```

```vue
<template>
  <!-- Update banner shown when new version is available -->
  <Transition name="slide-up">
    <div v-if="needRefresh" class="update-banner">
      🎉 A new version is available!
      <button @click="updateServiceWorker()">Update Now</button>
    </div>
  </Transition>
</template>
```
