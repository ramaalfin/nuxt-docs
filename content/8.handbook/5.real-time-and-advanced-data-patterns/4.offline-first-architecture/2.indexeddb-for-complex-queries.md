# IndexedDB for Complex Queries

IndexedDB is the browser's built-in transactional NoSQL database. It is the only browser storage mechanism capable of storing large amounts of structured data (gigabytes), supporting complex queries, and providing ACID transactions—making it the foundation of serious offline-first applications.

## 1. Why Not localStorage?

| Feature | localStorage | IndexedDB |
|---|---|---|
| **Storage limit** | ~5MB | Hundreds of MB to GB |
| **Data types** | Strings only | Any structured data (objects, blobs, files) |
| **API** | Synchronous (blocks main thread) | Asynchronous |
| **Indexing** | None | Multiple indexes |
| **Transactions** | None | Full ACID transactions |
| **Suitable for** | Simple key-value preferences | Complex offline data |

## 2. Using Dexie.js (Recommended Wrapper)

Raw IndexedDB API is verbose and complex. **Dexie.js** provides a clean, Promise-based API with TypeScript support.

```bash
pnpm add dexie
```

### Defining the Database Schema

```typescript
// db/index.ts
import Dexie, { type EntityTable } from 'dexie'

interface Product {
  id: string
  name: string
  categoryId: string
  price: number
  updatedAt: number
  syncStatus: 'synced' | 'pending' | 'conflict'
}

interface SyncQueueItem {
  id?: number        // Auto-increment primary key
  endpoint: string
  method: string
  body: any
  enqueuedAt: number
  retries: number
}

class AppDatabase extends Dexie {
  products!: EntityTable<Product, 'id'>
  syncQueue!: EntityTable<SyncQueueItem, 'id'>

  constructor() {
    super('my-app-db')
    this.version(1).stores({
      products: 'id, categoryId, price, updatedAt, syncStatus',
      // Index on categoryId, price for filtering queries
      syncQueue: '++id, enqueuedAt, retries'
    })
  }
}

export const db = new AppDatabase()
```

### Complex Queries

```typescript
// composables/useOfflineProducts.ts
export function useOfflineProducts() {
  
  // Filter + sort query
  const getByCategory = async (categoryId: string, sortBy: 'price' | 'name' = 'price') => {
    return db.products
      .where('categoryId').equals(categoryId)
      .sortBy(sortBy)
  }

  // Range query (IndexedDB excels at this)
  const getByPriceRange = async (min: number, max: number) => {
    return db.products
      .where('price').between(min, max)
      .toArray()
  }

  // Full-text search (IndexedDB doesn't support this natively — use a manual scan)
  const searchByName = async (query: string) => {
    return db.products
      .filter(product => product.name.toLowerCase().includes(query.toLowerCase()))
      .limit(50)
      .toArray()
  }

  // Bulk upsert (efficient for syncing server data to IndexedDB)
  const upsertProducts = async (products: Product[]) => {
    await db.products.bulkPut(products)
  }

  return { getByCategory, getByPriceRange, searchByName, upsertProducts }
}
```

## 3. Reactive Queries with `useLiveQuery`

Dexie's `useLiveQuery` hook for React is available as a Vue composable via VueUse's `useLiveQuery` or a custom wrapper.

```typescript
// composables/useLiveProducts.ts
import { liveQuery } from 'dexie'
import { useObservable } from '@vueuse/rxjs'
import { from } from 'rxjs'

export function useLiveProducts(categoryId: Ref<string>) {
  // This automatically re-runs whenever IndexedDB data changes
  const products = useObservable(
    from(
      liveQuery(() =>
        db.products
          .where('categoryId').equals(categoryId.value)
          .toArray()
      )
    )
  )

  return { products }
}
```

## 4. Database Migrations

When your schema changes across versions, Dexie handles migrations gracefully.

```typescript
class AppDatabase extends Dexie {
  constructor() {
    super('my-app-db')
    
    // Version 1: Original schema
    this.version(1).stores({
      products: 'id, categoryId, price'
    })
    
    // Version 2: Added 'brand' index + renamed field
    this.version(2).stores({
      products: 'id, categoryId, price, brand, updatedAt' // Add new indexes
    }).upgrade(trans => {
      // Migrate existing data
      return trans.table('products').toCollection().modify(product => {
        product.brand = 'Unknown' // Provide default for new field
        product.updatedAt = Date.now()
      })
    })
  }
}
```

## 5. Storage Quota Management

```typescript
// Check available storage quota
const checkStorageQuota = async () => {
  if (!navigator.storage?.estimate) return null

  const { usage, quota } = await navigator.storage.estimate()
  const usedMB = Math.round((usage ?? 0) / 1024 / 1024)
  const quotaMB = Math.round((quota ?? 0) / 1024 / 1024)
  const percentUsed = Math.round(((usage ?? 0) / (quota ?? 1)) * 100)

  if (percentUsed > 80) {
    // Purge old or synced records to free space
    await db.products
      .where('syncStatus').equals('synced')
      .and(p => p.updatedAt < Date.now() - 7 * 24 * 60 * 60 * 1000) // Older than 7 days
      .delete()
  }

  return { usedMB, quotaMB, percentUsed }
}
```
