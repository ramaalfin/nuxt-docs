# Network Failure Handling

Network failures are the most common class of runtime errors in web applications. Unlike bugs in your own code, network failures are entirely outside your control—the user's connection can drop, DNS can fail, your CDN can have an outage, or your API server can become temporarily unresponsive.

The goal is not to prevent network failures (impossible) but to handle them gracefully so the user experience remains as smooth as possible.

## 1. Detecting Network Status

The first step is knowing whether the user is online or offline.

```typescript
// composables/useNetworkStatus.ts
export function useNetworkStatus() {
  const isOnline = ref(navigator.onLine)
  
  useEventListener(window, 'online', () => { isOnline.value = true })
  useEventListener(window, 'offline', () => { isOnline.value = false })
  
  return { isOnline }
}
```

**Show a persistent banner when offline:**
```vue
<!-- components/OfflineBanner.vue -->
<script setup>
const { isOnline } = useNetworkStatus()
</script>

<template>
  <Transition name="slide-down">
    <div v-if="!isOnline" class="offline-banner" role="alert">
      <WifiOffIcon />
      You're offline. Some features may be unavailable.
    </div>
  </Transition>
</template>
```

## 2. Preventing Actions When Offline

Do not let users submit forms or trigger mutations when offline, as these actions will silently fail.

```vue
<script setup>
const { isOnline } = useNetworkStatus()

const submitOrder = async () => {
  if (!isOnline.value) {
    showNotification({ type: 'warning', message: 'Cannot place order while offline.' })
    return
  }
  // ... proceed with checkout
}
</script>

<template>
  <button :disabled="!isOnline" @click="submitOrder">
    {{ isOnline ? 'Place Order' : 'Offline — Cannot Order' }}
  </button>
</template>
```

## 3. Retry Logic and Timeout Handling

Not every network failure means the user is fully offline. Temporary server hiccups, DNS resolution timeouts, and flaky connections require intelligent retry logic.

### Request Timeouts with `$fetch`
```typescript
// composables/useApiFetch.ts
export function useApiFetch<T>(url: string, options = {}) {
  return useFetch<T>(url, {
    ...options,
    // Abort request if it takes longer than 10 seconds
    signal: AbortSignal.timeout(10_000),
    
    onRequestError({ error }) {
      if (error.name === 'TimeoutError') {
        console.warn('[API] Request timed out:', url)
      }
    }
  })
}
```

### Automatic Retry for Idempotent Requests
```typescript
const fetchWithRetry = async (url: string, retries = 3): Promise<any> => {
  for (let i = 0; i < retries; i++) {
    try {
      return await $fetch(url)
    } catch (error: any) {
      const isLastAttempt = i === retries - 1
      const isRetryable = !error.statusCode || error.statusCode >= 500
      
      if (isLastAttempt || !isRetryable) throw error
      
      // Wait before retrying (exponential backoff)
      await new Promise(r => setTimeout(r, 500 * Math.pow(2, i)))
    }
  }
}
```

## 4. Optimistic UI with Network Resilience

For write operations, use optimistic updates with guaranteed rollback on network failure.

```typescript
// Implemented with clear "pending" and "error" visual indicators
const updateProfile = async (updates: Partial<UserProfile>) => {
  const rollbackState = { ...profile.value }
  
  // Optimistically update the UI
  Object.assign(profile.value, updates)
  savingStatus.value = 'saving'
  
  try {
    await $fetch('/api/profile', { method: 'PATCH', body: updates })
    savingStatus.value = 'saved'
  } catch (error) {
    // Rollback on network failure
    Object.assign(profile.value, rollbackState)
    savingStatus.value = 'error'
  }
}
```

## 5. Offline Queue for Deferred Actions

For critical actions that a user wants to complete even while offline, queue them locally and execute when the connection is restored.

```typescript
// composables/useOfflineQueue.ts
export function useOfflineQueue() {
  const { isOnline } = useNetworkStatus()
  const queue = useLocalStorage<any[]>('offline_action_queue', [])

  const enqueue = (action: { url: string; method: string; body?: any }) => {
    queue.value.push(action)
  }

  const flush = async () => {
    if (!isOnline.value || queue.value.length === 0) return
    
    const pending = queue.value.splice(0) // Remove and take all items
    
    for (const action of pending) {
      try {
        await $fetch(action.url, { method: action.method, body: action.body })
      } catch {
        queue.value.unshift(action) // Put failed action back at the front
        break // Stop processing on failure
      }
    }
  }

  // Auto-flush when coming back online
  watch(isOnline, (online) => { if (online) flush() })
  
  return { enqueue, pendingCount: computed(() => queue.value.length) }
}
```
