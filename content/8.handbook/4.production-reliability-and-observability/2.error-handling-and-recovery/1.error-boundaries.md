# Error Boundaries in Vue and Nuxt

In React, "Error Boundaries" are class components that catch JavaScript errors anywhere in their child component tree. Vue and Nuxt provide equivalent mechanisms for containing and recovering from runtime errors without crashing the entire application.

## 1. Vue's `onErrorCaptured` (Component-Level)

The `onErrorCaptured` lifecycle hook is Vue's built-in equivalent of React's `componentDidCatch`. It captures errors thrown by any descendant component.

```typescript
// components/ErrorBoundary.vue
<script setup lang="ts">
import { ref, onErrorCaptured } from 'vue'

const hasError = ref(false)
const errorMessage = ref('')

onErrorCaptured((error: Error, instance, info: string) => {
  hasError.value = true
  errorMessage.value = error.message
  
  // Log the error to your observability platform
  console.error(`[Error in ${info}]:`, error)
  
  // Return false to STOP the error from propagating further up the tree
  // Return true (or nothing) to allow it to propagate
  return false
})
</script>

<template>
  <div v-if="hasError" class="error-fallback">
    <h2>Something went wrong.</h2>
    <p>{{ errorMessage }}</p>
    <button @click="hasError = false">Try again</button>
  </div>
  <slot v-else />
</template>
```

**Usage: Wrap any risky subtree with the boundary:**
```vue
<ErrorBoundary>
  <UserDashboard />
</ErrorBoundary>
```

## 2. Nuxt's `<NuxtErrorBoundary>` (Built-In)

Nuxt 3 ships with a `<NuxtErrorBoundary>` component that provides the same concept out of the box, with built-in support for the `#error` template slot.

```vue
<!-- pages/dashboard.vue -->
<template>
  <NuxtErrorBoundary @error="logError">
    <!-- Default slot: Renders if no error -->
    <template #default>
      <DashboardWidgets />
    </template>

    <!-- Error slot: Renders when an error is caught -->
    <template #error="{ error, clearError }">
      <div class="error-panel">
        <h3>Dashboard failed to load</h3>
        <p>{{ error.message }}</p>
        <button @click="clearError()">Reload</button>
      </div>
    </template>
  </NuxtErrorBoundary>
</template>

<script setup lang="ts">
const logError = (error: Error) => {
  // Send to Sentry, Datadog, etc.
  $sentry.captureException(error)
}
</script>
```

## 3. Global Error Handling in Nuxt

For errors that escape component boundaries (e.g., errors thrown in Pinia actions, plugins, or route middleware), Nuxt provides `app.vue` level hooks.

### `nuxtApp.hook('vue:error')`
```typescript
// plugins/error-handler.ts
export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.hook('vue:error', (error, context, info) => {
    // Global catch-all for uncaught Vue errors
    console.error('Uncaught Vue error:', error)
    // Forward to error tracking service
  })
})
```

### `useError` and `showError`
For server-side and page-level errors (e.g., a 404 from `useFetch`), Nuxt provides dedicated error utilities.

```typescript
// Inside a page or composable
const { data, error } = await useFetch('/api/user')

if (error.value) {
  // Trigger Nuxt's full-page error handling (renders error.vue)
  throw createError({
    statusCode: 404,
    statusMessage: 'User not found',
    fatal: true // Marks as unrecoverable, shows error.vue
  })
}
```

## 4. The `error.vue` Page

For unrecoverable fatal errors, Nuxt renders a special `error.vue` page in the root of your project. This replaces the entire app layout.

```vue
<!-- error.vue -->
<script setup lang="ts">
import type { NuxtError } from '#app'

const props = defineProps<{ error: NuxtError }>()

const handleError = () => clearError({ redirect: '/' })
</script>

<template>
  <div class="error-page">
    <h1>{{ error.statusCode }}</h1>
    <p>{{ error.statusMessage }}</p>
    <button @click="handleError">Go Home</button>
  </div>
</template>
```

## Summary: Choosing the Right Error Boundary Level

| Scenario | Tool |
|---|---|
| A single widget crashes | `<NuxtErrorBoundary>` around that widget |
| An entire page section fails | Custom `ErrorBoundary.vue` with `onErrorCaptured` |
| A route/page fails to load data | `createError()` + `error.vue` |
| Uncaught global errors | `nuxtApp.hook('vue:error')` in a plugin |
