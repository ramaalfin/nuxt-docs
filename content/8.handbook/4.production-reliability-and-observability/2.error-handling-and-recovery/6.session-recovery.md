# Session Recovery

A session represents a user's authenticated state and interaction context within your application. Sessions can be interrupted by network drops, server errors, token expiry, or browser crashes. Robust session recovery ensures users are never unexpectedly logged out or lose their work without warning.

## 1. Token Expiry and Silent Refresh

In most Nuxt applications, authentication uses short-lived JWTs (Access Tokens) paired with longer-lived Refresh Tokens.

**The Silent Refresh Pattern:**
When an API call returns a `401 Unauthorized` because the Access Token has expired, the application should automatically attempt to exchange the Refresh Token for a new Access Token—invisibly to the user.

```typescript
// plugins/auth-interceptor.ts
export default defineNuxtPlugin((nuxtApp) => {
  const { $authStore } = nuxtApp

  // Add a response interceptor to $fetch
  globalThis.$fetch = $fetch.create({
    onResponseError: async ({ response, request, options }) => {
      if (response.status === 401) {
        try {
          // Attempt to silently refresh the token
          await $authStore.refreshToken()
          
          // Retry the original request with the new token
          return $fetch(request, options)
        } catch (refreshError) {
          // Refresh failed — session is truly expired
          await $authStore.logout()
          // Redirect to login, preserving the original destination
          await nuxtApp.$router.push(`/login?redirect=${nuxtApp.$route.fullPath}`)
        }
      }
    }
  })
})
```

## 2. Persisting Application State Across Refreshes

Users refresh the page. When they do, the entire in-memory application state (Pinia stores, Vue refs) is lost. Persisting critical state to `localStorage` or `sessionStorage` prevents jarring UX.

### Using `pinia-plugin-persistedstate`

```typescript
// plugins/pinia-persist.ts
import { createPersistedState } from 'pinia-plugin-persistedstate'

export default defineNuxtPlugin(({ $pinia }) => {
  $pinia.use(createPersistedState())
})
```

```typescript
// stores/cart.ts
export const useCartStore = defineStore('cart', {
  state: () => ({ items: [] as CartItem[] }),
  persist: {
    key: 'shopping-cart',
    storage: persistedState.localStorage, // Persist across tabs and refreshes
    pick: ['items'] // Only persist the 'items' array, not loading states
  }
})
```

## 3. Form State Preservation

Nothing is more frustrating for a user than filling out a long form, having a network hiccup, and losing all their input. Implement auto-save or local persistence for long forms.

```typescript
// composables/useFormPersistence.ts
export function useFormPersistence<T extends object>(key: string, initialState: T) {
  const savedState = useLocalStorage<T>(key, initialState)
  const form = reactive({ ...savedState.value })

  // Auto-save on every change (debounced to prevent excessive writes)
  const debouncedSave = useDebounceFn(() => {
    savedState.value = { ...form }
  }, 500)

  watch(form, debouncedSave, { deep: true })

  const clearSavedState = () => {
    savedState.value = initialState
  }

  return { form, clearSavedState }
}
```

```vue
<script setup lang="ts">
const { form, clearSavedState } = useFormPersistence('checkout-form', {
  name: '', address: '', city: ''
})

const submitForm = async () => {
  await $fetch('/api/orders', { body: form })
  clearSavedState() // Wipe saved state on successful submission
}
</script>
```

## 4. Handling Multi-Tab Session Conflicts

In modern apps, users often have multiple tabs open. If a user logs out in Tab A, Tab B should not remain in an "authenticated" state.

```typescript
// composables/useSessionSync.ts
export function useSessionSync() {
  // Listen for storage events from OTHER tabs
  useEventListener(window, 'storage', (event: StorageEvent) => {
    if (event.key === 'auth-token' && event.newValue === null) {
      // Token was cleared in another tab — log out this tab too
      const authStore = useAuthStore()
      authStore.handleExternalLogout()
    }
  })
}
```

Call `useSessionSync()` in `app.vue` to enable this across the application.

## 5. Session Expiry Warning

Before a session expires, warn the user so they can extend it—especially critical in banking or healthcare applications.

```typescript
// composables/useSessionTimer.ts
export function useSessionTimer(expiryTimestamp: Ref<number>) {
  const showWarning = ref(false)
  const WARNING_THRESHOLD_MS = 5 * 60 * 1000 // Show warning 5 minutes before expiry

  watchEffect(() => {
    const msUntilExpiry = expiryTimestamp.value - Date.now()
    
    if (msUntilExpiry <= WARNING_THRESHOLD_MS && msUntilExpiry > 0) {
      showWarning.value = true
    } else {
      showWarning.value = false
    }
  })

  return { showWarning }
}
```
