# ARIA Roles & Properties

ARIA (Accessible Rich Internet Applications) is a set of HTML attributes that extend native HTML semantics to make dynamic, JavaScript-driven content accessible to assistive technologies like screen readers.

> **The first rule of ARIA:** Don't use ARIA if a native HTML element with the required semantics exists.

ARIA does not change the behavior of an element—it only changes how it is announced to assistive technologies. A `<div role="button">` still does not receive keyboard focus or trigger on Enter/Space without additional JavaScript.

## 1. ARIA Roles

ARIA roles override or supplement the semantic role of an element.

### Landmark Roles
These create navigable regions for screen reader users:

```html
<div role="banner">      <!-- equivalent to <header> (top-level) -->
<div role="navigation">  <!-- equivalent to <nav> -->
<div role="main">        <!-- equivalent to <main> -->
<div role="complementary"> <!-- equivalent to <aside> -->
<div role="contentinfo"> <!-- equivalent to <footer> (top-level) -->
<div role="search">      <!-- no native equivalent: search form region -->
```

*Prefer native HTML elements over role attributes. Only use role attributes on legacy `<div>`/`<span>` wrappers.*

### Widget Roles
For custom interactive widgets with no native HTML equivalent:

```html
<div role="dialog" aria-modal="true" aria-labelledby="dialog-title">
<div role="tablist">
<button role="tab" aria-selected="true" aria-controls="panel-1">Tab 1</button>
<div role="tabpanel" id="panel-1">...</div>
<div role="tooltip" id="tooltip-1">Tooltip text</div>
<div role="alert">           <!-- For important, time-sensitive messages -->
<div role="status">          <!-- For polite, non-urgent status updates -->
<div role="progressbar" aria-valuenow="60" aria-valuemin="0" aria-valuemax="100">
```

## 2. ARIA States & Properties

ARIA states describe the current condition of a widget. They change dynamically via JavaScript.

### Essential States

```html
<!-- Indicates an element is expanded or collapsed -->
<button aria-expanded="false" @click="toggleMenu">Menu</button>

<!-- Indicates the current selection in a list -->
<li role="option" aria-selected="true">Option 1</li>

<!-- Indicates a widget is disabled -->
<button aria-disabled="true">Submit</button>

<!-- Marks a form field as required -->
<input aria-required="true" />

<!-- Marks a form field as having an invalid value -->
<input aria-invalid="true" aria-describedby="error-msg" />

<!-- Hides a decorative element from screen readers -->
<svg aria-hidden="true"><use href="#icon-arrow" /></svg>
```

### Essential Properties

```html
<!-- Names a region or landmark -->
<nav aria-label="Secondary navigation">

<!-- Associates a label (by ID) with an element -->
<section aria-labelledby="section-heading">
<h2 id="section-heading">About Us</h2>

<!-- Associates a description (by ID) with an element -->
<input aria-describedby="password-hint" />
<p id="password-hint">Must be at least 8 characters</p>

<!-- Announces live region changes to screen readers -->
<div aria-live="polite">       <!-- Non-urgent updates (e.g., form status) -->
<div aria-live="assertive">    <!-- Urgent updates (e.g., error messages) -->
```

## 3. Common ARIA Patterns in Vue Components

### Custom Modal Dialog

```vue
<template>
  <Teleport to="body">
    <div 
      v-if="isOpen"
      role="dialog"
      aria-modal="true"
      :aria-labelledby="titleId"
      class="modal-overlay"
      @keydown.esc="close"
    >
      <div class="modal-content">
        <h2 :id="titleId">{{ title }}</h2>
        <slot />
        <button @click="close" aria-label="Close dialog">✕</button>
      </div>
    </div>
  </Teleport>
</template>

<script setup lang="ts">
import { useId } from 'vue'
const titleId = useId()
const props = defineProps<{ isOpen: boolean; title: string }>()
const emit = defineEmits(['close'])
const close = () => emit('close')
</script>
```

### Loading State Announcements

```vue
<template>
  <!-- 
    aria-live="polite" announces changes when the user is idle.
    This announces to screen readers when loading completes.
  -->
  <div aria-live="polite" aria-atomic="true" class="sr-only">
    <span v-if="isLoading">Loading results, please wait...</span>
    <span v-else-if="results.length === 0">No results found.</span>
    <span v-else>{{ results.length }} results loaded.</span>
  </div>
  
  <div v-if="isLoading" aria-hidden="true">
    <!-- Visual skeleton or spinner (hidden from screen readers) -->
    <SkeletonList />
  </div>
  <ResultsList v-else :results="results" />
</template>
```

## 4. What to Avoid

- **Don't redundantly add roles to semantic elements:** `<button role="button">` is redundant.
- **Don't use `aria-label` on elements that already have visible text:** Use `aria-labelledby` instead to reference the visible text.
- **Don't use `role="presentation"` or `aria-hidden="true"` on focusable elements:** This creates "ghost" elements that are focusable but produce no output for screen reader users.
- **Don't use `aria-live="assertive"` liberally:** It interrupts the screen reader mid-sentence. Reserve it for error messages only.
