# Animation & Motion Sensitivity

Animations and transitions can greatly enhance user experience—but for users with vestibular disorders, migraines, or epilepsy, certain types of motion can trigger nausea, disorientation, seizures, or physical pain.

WCAG 2.1 Success Criterion 2.3.3 (Animation from Interactions, Level AAA) recommends disabling motion-based animations. However, even at the AA level, it is a best practice and ethical obligation to respect the user's motion preferences.

## 1. The `prefers-reduced-motion` Media Query

The Operating System allows users to set a "reduce motion" preference (e.g., macOS: System Settings → Accessibility → Motion → Reduce Motion). CSS and JavaScript can detect this preference.

```css
/* Default: Animations are on */
.card {
  transition: transform 0.3s ease, box-shadow 0.3s ease;
}
.card:hover {
  transform: translateY(-8px);
  box-shadow: 0 20px 40px rgba(0, 0, 0, 0.15);
}

/* When the user prefers reduced motion: disable or simplify */
@media (prefers-reduced-motion: reduce) {
  .card {
    transition: none; /* Remove transition entirely */
  }
  .card:hover {
    transform: none; /* Remove movement */
    /* Keep the box-shadow change — it doesn't involve movement */
    box-shadow: 0 20px 40px rgba(0, 0, 0, 0.15);
  }
}
```

### The "Safe Motion" Alternative Approach

Rather than disabling everything, prefer providing a safe, minimal alternative:

```css
/* A modern, concise approach using the @layer cascade */
@layer base {
  /* Apply animations by default */
  * {
    transition-duration: 0.3s;
    animation-duration: 0.3s;
  }
}

@media (prefers-reduced-motion: reduce) {
  * {
    /* Drastically reduce all durations — nearly instant, but not jarring */
    transition-duration: 0.01ms !important;
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    scroll-behavior: auto !important;
  }
}
```

## 2. Dangerous Animation Patterns to Avoid or Control

| Pattern | Risk | Safe Alternative |
|---|---|---|
| **Parallax scrolling** | Can cause severe nausea | Disable under `reduce` |
| **Infinite looping animations** | Distracting, vestibular trigger | Pause or remove |
| **Large-scale zoom effects** | Vestibular trigger | Fade instead |
| **Rapid blinking (> 3 Hz)** | Seizure risk | **Never use, regardless of preference** |
| **Auto-playing video with motion** | Vestibular trigger | Require user interaction to play |
| **Scroll-jacking** | Disorienting | Avoid entirely |

## 3. JavaScript Detection for Complex Animations

For animations driven by JavaScript (e.g., GSAP, Vue Transition hooks, Three.js), use the JavaScript API.

```typescript
// composables/useMotionPreference.ts
export function useMotionPreference() {
  const mediaQuery = window.matchMedia('(prefers-reduced-motion: reduce)')
  const prefersReducedMotion = ref(mediaQuery.matches)
  
  // Listen for changes (user can change their OS setting while the app is open)
  const handler = (e: MediaQueryListEvent) => {
    prefersReducedMotion.value = e.matches
  }
  mediaQuery.addEventListener('change', handler)
  onUnmounted(() => mediaQuery.removeEventListener('change', handler))
  
  return { prefersReducedMotion }
}
```

```vue
<script setup lang="ts">
import { useMotionPreference } from '~/composables/useMotionPreference'
import gsap from 'gsap'

const { prefersReducedMotion } = useMotionPreference()
const heroRef = ref<HTMLElement>()

onMounted(() => {
  if (prefersReducedMotion.value) {
    // Skip animation entirely for users who prefer reduced motion
    return
  }
  
  gsap.from(heroRef.value, {
    y: 40, opacity: 0, duration: 0.8, ease: 'power2.out'
  })
})
</script>
```

## 4. Vue Transitions with Reduced Motion

```vue
<!-- Graceful transition that respects motion preference -->
<Transition
  :css="!prefersReducedMotion"
  enter-active-class="transition-opacity duration-300"
  enter-from-class="opacity-0"
  enter-to-class="opacity-100"
  leave-active-class="transition-opacity duration-300"
  leave-from-class="opacity-100"
  leave-to-class="opacity-0"
>
  <div v-if="isVisible">Content</div>
</Transition>
```

Or use a global Vue plugin to disable all `<Transition>` animations:

```typescript
// plugins/reduced-motion.client.ts
export default defineNuxtPlugin((nuxtApp) => {
  const mediaQuery = window.matchMedia('(prefers-reduced-motion: reduce)')
  
  if (mediaQuery.matches) {
    // Override Vue Transition globally
    nuxtApp.vueApp.config.globalProperties.$motionDisabled = true
  }
})
```

## 5. Scroll-Driven Animations

CSS Scroll-driven animations (using `@keyframes` with `animation-timeline: scroll()`) are increasingly common. Always wrap them in a motion preference check.

```css
@media (prefers-reduced-motion: no-preference) {
  .fade-in-on-scroll {
    animation: fade-in linear both;
    animation-timeline: view();
    animation-range: entry 0% entry 50%;
  }
}

@keyframes fade-in {
  from { opacity: 0; transform: translateY(20px); }
  to { opacity: 1; transform: translateY(0); }
}
```

## 6. Testing Motion Preferences

1. **macOS:** System Settings → Accessibility → Motor → Reduce Motion → On.
2. **Windows:** Settings → Ease of Access → Display → Show animations in Windows → Off.
3. **Chrome DevTools:** Rendering tab → Emulate CSS media features → `prefers-reduced-motion: reduce`.
