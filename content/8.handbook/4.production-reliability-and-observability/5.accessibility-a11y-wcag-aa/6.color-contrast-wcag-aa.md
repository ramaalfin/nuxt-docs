# Color Contrast (WCAG AA Standards)

Color contrast refers to the luminance ratio between foreground text and its background. Insufficient contrast makes text difficult or impossible to read for users with low vision, color blindness, or anyone using a screen in bright sunlight.

WCAG 2.1 Success Criterion 1.4.3 (Level AA) defines minimum contrast ratios that all production applications must meet.

## 1. WCAG AA Contrast Requirements

| Content Type | Minimum Ratio (AA) | Enhanced Ratio (AAA) |
|---|---|---|
| **Normal text** (< 18pt or < 14pt bold) | **4.5:1** | 7:1 |
| **Large text** (≥ 18pt or ≥ 14pt bold) | **3:1** | 4.5:1 |
| **UI components** (borders, icons, focus indicators) | **3:1** | N/A |
| **Decorative** elements or logos | None | None |

*Note: 18pt ≈ 24px. 14pt bold ≈ 18.67px bold.*

## 2. Understanding the Contrast Ratio

The contrast ratio is calculated using the relative luminance of two colors, ranging from 1:1 (identical colors, no contrast) to 21:1 (black text on white, maximum contrast).

```
Examples:
- Black (#000000) on White (#FFFFFF) = 21:1 ✅
- Medium grey (#767676) on White (#FFFFFF) = 4.54:1 ✅ (just passes AA)
- Light grey (#AAAAAA) on White (#FFFFFF) = 2.32:1 ❌ (fails AA)
- Blue (#0000FF) on White (#FFFFFF) = 8.59:1 ✅
```

## 3. Tools for Checking Contrast

### Browser DevTools
Chrome DevTools displays the contrast ratio in the color picker and shows a ✅ or ❌ for WCAG AA/AAA compliance.

### Design Tools
- **Figma:** The "Contrast" plugin or built-in contrast checker in Dev Mode.
- **Adobe XD:** Built-in accessibility checker.

### Online Tools
- **WebAIM Contrast Checker:** `https://webaim.org/resources/contrastchecker/`
- **Colour Contrast Analyser (App):** `https://www.tpgi.com/color-contrast-checker/`
- **Accessible Color Palette Generators:** `https://www.learnui.design/tools/accessible-color-generator.html`

### Automated Testing
- **axe-core** (integrated in Vitest/Playwright) detects contrast failures automatically.
- **Lighthouse** in Chrome DevTools includes contrast checks.

## 4. Building an Accessible Color System in Nuxt

Design your color palette with contrast requirements built in, not added as an afterthought.

### Defining Tokens with Contrast in Mind

```css
/* styles/tokens.css */
:root {
  /* Primary — tested at 4.5:1+ on white */
  --color-primary-500: hsl(221, 83%, 40%);  /* #1a56db - 5.8:1 on white ✅ */
  --color-primary-600: hsl(221, 83%, 30%);  /* Darker for hover states */
  
  /* Text — always use these on backgrounds */
  --color-text-primary: hsl(220, 9%, 12%);   /* Near-black: 16:1 on white ✅ */
  --color-text-secondary: hsl(220, 9%, 40%); /* 7.1:1 on white ✅ */
  --color-text-disabled: hsl(220, 9%, 63%);  /* 3.1:1 - only for truly disabled */
  
  /* Surface / Background */
  --color-bg-primary: hsl(0, 0%, 100%);      /* White */
  --color-bg-secondary: hsl(220, 14%, 96%);  /* Very light grey */
  
  /* Error states */
  --color-error: hsl(0, 72%, 42%);           /* Red: 5.2:1 on white ✅ */
  --color-error-bg: hsl(0, 100%, 97%);       /* Light red background */
}
```

### Dark Mode Contrast
Dark mode requires re-testing every text/background combination. Light-mode colors rarely meet contrast requirements when simply inverted.

```css
@media (prefers-color-scheme: dark) {
  :root {
    --color-text-primary: hsl(220, 9%, 95%);    /* Near-white on dark: 14:1 ✅ */
    --color-text-secondary: hsl(220, 9%, 72%);  /* 5.4:1 on dark bg ✅ */
    --color-bg-primary: hsl(220, 9%, 12%);      /* Dark background */
    --color-primary-500: hsl(221, 83%, 65%);    /* Lighter primary for dark mode */
  }
}
```

## 5. Color Is Not Sufficient Alone

WCAG 1.4.1 requires that **color is not the only means of conveying information**. Users with color blindness (8% of males) cannot distinguish red from green.

```vue
<!-- ❌ Bad: Relies on color alone to show error state -->
<input :class="{ 'border-red': hasError }" />

<!-- ✅ Good: Color + icon + text label -->
<div class="input-wrapper">
  <input 
    :class="{ 'border-error': hasError }"
    :aria-invalid="hasError"
    :aria-describedby="hasError ? 'field-error' : undefined"
  />
  <div v-if="hasError" id="field-error" class="error-message">
    <ExclamationIcon aria-hidden="true" />
    <!-- Red color + icon + text = accessible for all users -->
    Please enter a valid email address.
  </div>
</div>
```

## 6. Common Failures to Avoid

- **Placeholder text as labels:** Placeholder text typically has very low contrast (`#999` on white = 2.85:1). Never use placeholder text as a substitute for a proper `<label>`.
- **Disabled state contrast:** Disabled elements are exempt from contrast requirements, but disabled elements should be rare. If information must be visible, don't disable it.
- **Focus indicator contrast:** Your custom `:focus-visible` ring must meet the 3:1 contrast ratio against adjacent colors. A thin blue outline on a blue button can fail.
