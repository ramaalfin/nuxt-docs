# Font Sizing & Spacing

Typography accessibility encompasses more than just contrast. Font size, line height, letter spacing, and the ability to resize text without breaking layouts all directly impact readability—especially for users with low vision, dyslexia, or cognitive disabilities.

WCAG 2.1 Success Criterion 1.4.4 (Resize Text, Level AA) requires text to be resizable up to 200% without loss of content or functionality.

## 1. Use Relative Units (`rem` and `em`)

Never define font sizes in absolute pixels (`px`) for body text or UI copy. When users increase their browser's base font size (a common accessibility accommodation), pixel-based text does not scale.

**The Root Rule:** `1rem` = the browser's base font size (default: 16px, but users may change this to 20px, 24px, etc.)

```css
/* ❌ BAD: Ignores user's browser font size preference */
body { font-size: 16px; }
h1 { font-size: 32px; }
p { font-size: 14px; }

/* ✅ GOOD: Scales with user preferences */
html { font-size: 100%; } /* Respect the browser default */
body { font-size: 1rem; } /* 16px by default, scales with browser setting */
h1 { font-size: 2rem; }   /* 32px by default, scales proportionally */
p { font-size: 0.875rem; } /* 14px by default, scales proportionally */
```

## 2. Minimum Font Sizes

WCAG does not prescribe a specific minimum font size, but industry best practices recommend:

| Content Type | Minimum Size |
|---|---|
| Body text | **16px (1rem)** — do not go smaller for running text |
| Secondary text (captions, labels) | **14px (0.875rem)** |
| Legal / fine print | **12px (0.75rem)** — use sparingly |

Do not use font sizes below 12px for any readable text.

## 3. Line Height and Line Length

Sufficient line height and line length dramatically improve readability, especially for users with dyslexia.

```css
/* WCAG 1.4.12 (Text Spacing) minimum recommendations */
body {
  line-height: 1.5;      /* WCAG requires at least 1.5x the font size */
  letter-spacing: 0.12em; /* WCAG requires ability to set to 0.12em */
  word-spacing: 0.16em;   /* WCAG requires ability to set to 0.16em */
}

p {
  /* Optimal reading line length: 60-80 characters */
  /* `ch` unit = width of the "0" character in the current font */
  max-width: 75ch;
}
```

## 4. WCAG 1.4.12 — Text Spacing

Users must be able to override your text spacing CSS without loss of content or functionality. This means your layout must not break when these overrides are applied:
- Line height to at least 1.5x the font size.
- Letter spacing to at least 0.12em.
- Word spacing to at least 0.16em.
- Space between paragraphs to at least 2x the font size.

**Testing:** Use the [Text Spacing Bookmarklet](https://www.html5accessibility.com/tests/tsbookmarklet.html) to simulate user overrides on any page.

## 5. Fluid Typography with `clamp()`

`clamp()` creates fluid typography that scales between a minimum and maximum size based on the viewport width, without JavaScript.

```css
/* Typography that scales from 1rem (min) to 1.25rem (max) */
/* fluidly between 320px and 1200px viewport widths */
:root {
  --font-size-body: clamp(1rem, 0.875rem + 0.625vw, 1.25rem);
  --font-size-h1: clamp(1.75rem, 1.5rem + 1.25vw, 2.5rem);
  --font-size-h2: clamp(1.5rem, 1.25rem + 1vw, 2rem);
}

body { font-size: var(--font-size-body); }
h1 { font-size: var(--font-size-h1); }
```

## 6. Paragraph Spacing

```css
/* Well-spaced body text */
p {
  font-size: 1rem;
  line-height: 1.6;       /* 160% of font size — generous for reading */
  margin-bottom: 1.5em;   /* Space between paragraphs (1.5x current font size) */
  max-width: 70ch;        /* Optimal reading width */
}
```

## 7. Respecting User Motion Preferences in Typography

Some users have font size and layout settings tied to their OS accessibility preferences. Respect these:

```css
/* Respect OS-level large text settings */
@media (prefers-reduced-motion: reduce) {
  /* Disable text animations for users who prefer no motion */
  .animated-text {
    animation: none;
    transition: none;
  }
}
```

## 8. Zoom Testing

Test your application at 200% and 400% browser zoom:
1. `Ctrl/Cmd + +` to zoom in (test at 200%).
2. All text must remain readable.
3. No text should be clipped, hidden, or overflow its container.
4. Horizontal scrolling should not be required at 400% on a 1280px viewport (WCAG 1.4.10 Reflow).

**In Nuxt/Vue:** Avoid fixed-height containers for text content. Always use `min-height` instead of `height` to allow text to expand when enlarged.

```css
/* ❌ BAD: Clips text when zoomed */
.card-description { height: 48px; overflow: hidden; }

/* ✅ GOOD: Expands to fit enlarged text */
.card-description { min-height: 48px; }
```
