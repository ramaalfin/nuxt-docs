# Content Security Policy (CSP)

A Content Security Policy (CSP) is an HTTP response header that instructs the browser about which sources of content (scripts, styles, images, fonts, etc.) are considered trustworthy. Even if an attacker successfully injects a `<script>` tag via XSS, a properly configured CSP will prevent the browser from executing it.

CSP is your last line of defense—it does not replace input sanitization or output encoding, but it dramatically limits the damage an XSS attack can do.

## 1. How CSP Works

You define a policy in the `Content-Security-Policy` HTTP header. The browser enforces it by refusing to load or execute resources that don't match the policy.

```http
Content-Security-Policy: 
  default-src 'self';
  script-src 'self' https://cdn.example.com;
  style-src 'self' 'unsafe-inline';
  img-src 'self' data: https:;
  connect-src 'self' https://api.yourapp.com;
  frame-ancestors 'none';
```

## 2. Key CSP Directives

| Directive | Controls | Example Value |
|---|---|---|
| `default-src` | Fallback for all unspecified types | `'self'` |
| `script-src` | JavaScript sources | `'self' 'nonce-{nonce}'` |
| `style-src` | CSS sources | `'self' 'unsafe-inline'` |
| `img-src` | Image sources | `'self' data: https:` |
| `connect-src` | `fetch`, WebSocket, XHR | `'self' https://api.example.com` |
| `font-src` | Web fonts | `'self' https://fonts.gstatic.com` |
| `frame-ancestors` | Who can embed this page | `'none'` (prevents clickjacking) |
| `base-uri` | Restricts `<base>` tag | `'self'` |
| `form-action` | Form submission targets | `'self'` |

## 3. Implementing CSP in Nuxt with Nuxt Security

The simplest way to implement a strict, nonce-based CSP in Nuxt 3 is to use the `nuxt-security` module, which handles the complexity of nonce generation for SSR.

```bash
pnpm add nuxt-security
```

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  modules: ['nuxt-security'],
  security: {
    headers: {
      contentSecurityPolicy: {
        'default-src': ["'self'"],
        'script-src': [
          "'self'",
          "'nonce-{{nonce}}'",  // Auto-generated per-request nonce for SSR
          "'strict-dynamic'"    // Allows scripts loaded by trusted scripts
        ],
        'style-src': ["'self'", "'nonce-{{nonce}}'"],
        'img-src': ["'self'", "data:", "https:"],
        'connect-src': ["'self'", process.env.NUXT_PUBLIC_API_BASE_URL!],
        'font-src': ["'self'", "https://fonts.gstatic.com"],
        'frame-ancestors': ["'none'"],
        'base-uri': ["'self'"],
        'form-action': ["'self'"],
      }
    }
  }
})
```

## 4. Nonces for Inline Scripts

Avoid `'unsafe-inline'` in `script-src`. It completely defeats the purpose of CSP for scripts. Instead, use **nonces**—a cryptographically random value generated per-request and embedded in both the CSP header and the `<script>` tag.

The `nuxt-security` module handles nonce generation and injection automatically in SSR mode. For client-side only scripts, access the nonce from the Nuxt context:

```typescript
// server/api/some-handler.ts
export default defineEventHandler((event) => {
  const nonce = event.context.security?.nonce
  // Use nonce when dynamically generating script tags
})
```

## 5. CSP Report-Only Mode (Start Here)

Never deploy a strict CSP to production without first testing it. Use `Content-Security-Policy-Report-Only` to receive reports of violations without actually blocking anything.

```typescript
// nuxt.config.ts (initial rollout)
security: {
  headers: {
    // Report-Only: Violations are logged but NOT blocked
    contentSecurityPolicy: false,
    reportOnlyContentSecurityPolicy: {
      'default-src': ["'self'"],
      // ...your policy...
      'report-uri': ['/api/csp-report']  // Collect violation reports
    }
  }
}
```

Monitor the violation reports for 1-2 weeks. Once the policy is clean (no legitimate sources being blocked), switch to the enforcing `contentSecurityPolicy`.

## 6. CSP Violation Reporting Endpoint

```typescript
// server/api/csp-report.post.ts
export default defineEventHandler(async (event) => {
  const body = await readBody(event)
  
  // Log violations to your observability platform for analysis
  logger.warn('CSP Violation', {
    blockedUri: body['csp-report']['blocked-uri'],
    violatedDirective: body['csp-report']['violated-directive'],
    documentUri: body['csp-report']['document-uri'],
  })
  
  return null
})
```
