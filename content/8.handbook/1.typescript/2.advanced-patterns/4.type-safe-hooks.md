# Type-Safe Hooks (Composables)

In modern frontend frameworks like Vue (Composables) and React (Hooks), we extract reusable logic into standalone functions. When building these abstractions, it is crucial that they are deeply type-safe, providing excellent developer experience (DX) and preventing runtime errors for whoever consumes them.

Writing a **Type-Safe Hook** means utilizing advanced TypeScript features like Generics, Function Overloads, and Template Literal Types to ensure the hook intelligently adapts to its inputs.

---

## 1. The Power of Generics in Hooks

Generics allow your hook to be flexible while maintaining strict type safety.

Consider a simple `useLocalStorage` hook. Without generics, it might look like this:

```typescript
// ❌ Untyped: We have no idea what `value` actually is.
function useLocalStorage(key: string, initialValue: any) {
  const value = ref(localStorage.getItem(key) ?? initialValue);
  
  const setValue = (newValue: any) => {
    value.value = newValue;
    localStorage.setItem(key, String(newValue));
  };

  return { value, setValue };
}
```

By adding a generic type parameter `<T>`, we let TypeScript infer the type based on the `initialValue` provided by the consumer.

```typescript
// ✅ Type-Safe with Generics
import { ref, watch, type Ref } from 'vue';

export function useLocalStorage<T>(key: string, initialValue: T) {
  // `value` is now strictly typed as Ref<T>
  const value = ref<T>(initialValue) as Ref<T>;

  const rawData = localStorage.getItem(key);
  if (rawData) {
    try {
      value.value = JSON.parse(rawData);
    } catch (e) {
      console.error("Failed to parse localStorage item");
    }
  }

  // Watch for changes and strictly enforce `T`
  watch(value, (newValue) => {
    localStorage.setItem(key, JSON.stringify(newValue));
  }, { deep: true });

  return { value };
}
```

**Usage:**
```typescript
// TypeScript infers T as `number` automatically!
const { value: count } = useLocalStorage("click-count", 0);

count.value = 1; // ✅ OK
count.value = "string"; // ❌ ERROR: Type 'string' is not assignable to type 'number'.
```

## 2. Function Overloads for Different Return Types

Sometimes a hook's return type changes drastically depending on its input arguments.

Imagine a `useQuery` hook. If the user provides a `searchId`, it returns a specific item. If they don't, it returns an array of items.

```typescript
// How do we type this?
function useData(id?: string) {
  if (id) {
    return { data: { id, name: "Item" } }; // Returns a single object
  }
  return { data: [{ id: "1", name: "Item" }] }; // Returns an array
}
```

We can use **Function Overloads** to define multiple signatures for the same hook.

```typescript
type Item = { id: string; name: string };

// Signature 1: When an ID is provided
export function useData(id: string): { data: Item };

// Signature 2: When no ID is provided
export function useData(): { data: Item[] };

// Actual implementation (the signature here is broader)
export function useData(id?: string): { data: Item | Item[] } {
  if (id) {
    return { data: { id, name: "Item" } };
  }
  return { data: [{ id: "1", name: "Item" }] };
}
```

**Usage:**
```typescript
const single = useData("123"); 
// single.data is exactly `Item`

const list = useData(); 
// list.data is exactly `Item[]`
```

## 3. Template Literal Types for Event Hooks

If you are building a hook that attaches event listeners, you can use **Template Literal Types** to enforce strict naming conventions.

Let's build a `useKeyboardShortcut` hook that only accepts combinations starting with `Ctrl+` or `Alt+`.

```typescript
// Define the allowed modifiers using template literals
type Modifier = "Ctrl" | "Alt";
type Key = "A" | "B" | "C" | "Enter" | "Escape";

type ShortcutCombo = `${Modifier}+${Key}`;

export function useKeyboardShortcut(combo: ShortcutCombo, callback: () => void) {
  // Implementation details...
  window.addEventListener('keydown', (e) => {
    // Check keys...
  });
}
```

**Usage:**
```typescript
useKeyboardShortcut("Ctrl+S", saveDocument); // ✅ Valid
useKeyboardShortcut("Alt+Enter", submitForm); // ✅ Valid

useKeyboardShortcut("Shift+A", doSomething); 
// ❌ ERROR: Type '"Shift+A"' is not assignable to type 'ShortcutCombo'.
```

## 4. Extracting Types from Other Frameworks (Unwrapping Refs)

When building composables in Vue 3, you often deal with `Ref` or `ComputedRef` objects. If your hook accepts arguments that *might* be a Ref or might be a raw value, you should use `MaybeRef` and `unref`.

```typescript
import { unref, type MaybeRef } from 'vue';

// The consumer can pass a raw string OR a reactive Ref<string>
export function useGreeting(name: MaybeRef<string>) {
  // `unref` safely extracts the string whether it was a Ref or not
  const rawName = unref(name);
  
  return `Hello, ${rawName}!`;
}
```

This makes your hooks incredibly flexible without sacrificing type safety.

---

## Summary

- **Generics (`<T>`)**: Use generics to let the consumer define the types your hook works with (e.g., `useLocalStorage<User>`).
- **Function Overloads**: Use overloads when the return type of your hook changes significantly based on the arguments provided.
- **Template Literal Types**: Excellent for enforcing strict string patterns, like event names or custom identifiers.
- **Framework Utilities**: Familiarize yourself with framework-specific utility types like `MaybeRef` (Vue) to build flexible, DX-friendly APIs.
