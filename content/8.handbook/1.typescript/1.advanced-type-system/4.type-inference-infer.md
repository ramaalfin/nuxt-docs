# Type Inference & The `infer` Keyword

TypeScript is designed to be statically typed without requiring you to explicitly write types for every single variable. **Type Inference** is the mechanism TypeScript uses to guess the types of variables, function return values, and array elements when you don't explicitly provide them.

The `infer` keyword takes inference to the next level: it allows you to **declare a type variable within a conditional type** and let TypeScript figure out what that type should be, effectively extracting type information from existing structures.

---

## 1. How Basic Type Inference Works

Before diving into `infer`, it's crucial to understand how TypeScript infers types naturally.

### Variable Initialization
When you initialize a variable, TypeScript infers its type based on the assigned value.

```typescript
let message = "Hello World"; 
// Inferred as `string`

let count = 42; 
// Inferred as `number`

const pi = 3.14; 
// Inferred as the literal type `3.14` because `const` guarantees it won't change
```

### Best Common Type
When inferring the type of an array from multiple mixed values, TypeScript looks for the "best common type" that accommodates all elements.

```typescript
let arr = [0, 1, null];
// Inferred as `(number | null)[]`

let zoo = [new Rhino(), new Elephant(), new Snake()];
// Inferred as `(Rhino | Elephant | Snake)[]`
// Note: It doesn't automatically infer a base class like `Animal[]` unless you explicitly type it:
let animalZoo: Animal[] = [new Rhino(), new Elephant(), new Snake()];
```

### Contextual Typing
TypeScript infers types based on the **context** where they are used. This typically happens with callbacks.

```typescript
window.onmousedown = function (mouseEvent) {
  // TypeScript infers `mouseEvent` is of type `MouseEvent`
  // based on the context of `window.onmousedown`
  console.log(mouseEvent.button);
};
```

## 2. Advanced Inference with `infer`

While standard inference works on values, the `infer` keyword works strictly inside **Conditional Types** (`T extends U ? X : Y`). It allows you to say: *"If this type matches a certain pattern, extract a piece of that pattern into a new type variable."*

### The Syntax

```typescript
type ExtractThing<T> = T extends SomePattern<infer U> ? U : FallbackType;
```
If `T` matches `SomePattern`, TypeScript binds the inner type to `U` and returns `U`.

### Extracting Array Element Types

How do we get the type of an item inside an array?

```typescript
type ElementType<T> = T extends (infer Item)[] ? Item : T;

type StringArray = string[];
type Extracted = ElementType<StringArray>; 
// Result: `string`

type NotAnArray = number;
type Failed = ElementType<NotAnArray>; 
// Result: `number` (falls back to T)
```

### Extracting Function Return Types

This is how the built-in `ReturnType<T>` utility works:

```typescript
type GetReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

function createPerson() {
  return { name: "Alice", age: 30 };
}

type Person = GetReturnType<typeof createPerson>;
// Result: { name: string, age: number }
```

### Extracting Function Arguments

Just as we extracted the return type, we can extract the parameters using `infer`. This mirrors the built-in `Parameters<T>` utility:

```typescript
type GetFirstArgument<T> = T extends (first: infer A, ...args: any[]) => any ? A : never;

function logMessage(level: "info" | "error", message: string) {}

type Level = GetFirstArgument<typeof logMessage>;
// Result: "info" | "error"
```

## 3. Extracting Types from Promises

Working with async data often means unwrapping Promises. `infer` makes this simple.

```typescript
type UnpackPromise<T> = T extends Promise<infer U> ? U : T;

type Data = UnpackPromise<Promise<User[]>>; 
// Result: User[]
```

**Recursive Unpacking:**
Promises can be nested (`Promise<Promise<string>>`). We can use `infer` recursively to unwrap them completely. This is how the built-in `Awaited<T>` works.

```typescript
type DeepUnpack<T> = T extends Promise<infer U> ? DeepUnpack<U> : T;

type DeepData = DeepUnpack<Promise<Promise<Promise<number>>>>; 
// Result: number
```

## 4. `infer` with Template Literal Types

TypeScript 4.1 introduced template literal types, and `infer` works beautifully with them to parse strings at the type level.

### Extracting parts of a string

```typescript
type ExtractProtocol<T> = T extends `${infer Protocol}://${string}` ? Protocol : never;

type Http = ExtractProtocol<"http://example.com">; // "http"
type Ftp = ExtractProtocol<"ftp://server.com">;    // "ftp"
```

### Type-Safe Route Parameters

You can recursively extract path parameters from a URL string:

```typescript
type ExtractRouteParams<T extends string> = 
  T extends `${string}:${infer Param}/${infer Rest}`
    ? Param | ExtractRouteParams<`/${Rest}`>
    : T extends `${string}:${infer Param}`
      ? Param
      : never;

type Params = ExtractRouteParams<"/users/:userId/posts/:postId">;
// Result: "userId" | "postId"
```

## 5. Multiple `infer` Declarations

You are not limited to one `infer` per condition. You can extract multiple types simultaneously.

### Swapping Tuple Elements

```typescript
type Swap<T extends [any, any]> = T extends [infer A, infer B] ? [B, A] : T;

type Swapped = Swap<[string, number]>;
// Result: [number, string]
```

### Deconstructing Arrays (Head and Tail)

You can use the spread operator (`...`) with `infer` to separate the first element of a tuple from the rest.

```typescript
type Shift<T extends any[]> = T extends [infer Head, ...infer Tail] ? Tail : [];

type Colors = ["red", "green", "blue"];
type WithoutRed = Shift<Colors>;
// Result: ["green", "blue"]
```

## 6. Constraints on `infer` (TS 4.7+)

In older versions of TypeScript, whatever you inferred was basically `unknown` until you narrowed it down with additional nested conditional types. TypeScript 4.7 introduced the ability to add `extends` constraints directly to `infer` declarations.

### Before TS 4.7 (Nested Conditions)
```typescript
type FirstString<T> = 
  T extends [infer U, ...any[]] 
    ? U extends string 
      ? U 
      : never 
    : never;
```

### After TS 4.7 (Inline Constraints)
```typescript
type FirstString<T> = T extends [infer U extends string, ...any[]] ? U : never;

type A = FirstString<["hello", 42]>; // "hello"
type B = FirstString<[42, "hello"]>; // never (first element isn't a string)
```

---

## Summary

- **Type Inference** is TypeScript's automatic deduction of types based on values and context.
- The **`infer` keyword** is used inside **Conditional Types** (`extends ? :`).
- It acts as a **type-level pattern matcher**, extracting types from inside arrays, functions, Promises, and even string literals.
- You can use multiple `infer` declarations in a single condition to deconstruct complex types (like tuples).
- From TS 4.7+, you can apply **inline constraints** (`infer U extends string`) to make your type extractions cleaner and more precise.
