# Type Predicates & Type Guards

In TypeScript, you often encounter situations where a variable has a broad type (like `any`, `unknown`, or a union like `string | number`), but you need to execute code that only applies to a narrower, specific type.

TypeScript uses **Type Guards** to narrow down types within conditional blocks. A **Type Predicate** is a special return type for custom type guard functions that tells the TypeScript compiler: *"If this function returns true, the argument is of this specific type."*

---

## 1. Built-in Type Guards

TypeScript's control flow analysis automatically understands common JavaScript patterns and narrows types accordingly. These are built-in type guards.

### `typeof` Type Guards

The `typeof` operator is perfect for narrowing down JavaScript primitives.

```typescript
function printValue(value: string | number) {
  if (typeof value === "string") {
    // TypeScript knows 'value' is a string here
    console.log(value.toUpperCase()); 
  } else {
    // TypeScript knows 'value' must be a number here
    console.log(value.toFixed(2)); 
  }
}
```

*Note: `typeof` only works for basic types: `"string"`, `"number"`, `"bigint"`, `"boolean"`, `"symbol"`, `"undefined"`, `"object"`, and `"function"`.*

### `instanceof` Type Guards

For classes and objects constructed with `new`, you can use `instanceof`.

```typescript
class User {
  login() {}
}

class Admin extends User {
  deleteDatabase() {}
}

function handlePerson(person: User | Admin) {
  if (person instanceof Admin) {
    // Narrowed to Admin
    person.deleteDatabase();
  } else {
    // Narrowed to User
    person.login();
  }
}
```

### `in` Operator Type Guards

You can check if a specific property exists on an object to narrow down a union type.

```typescript
type Fish = { swim: () => void };
type Bird = { fly: () => void };

function move(animal: Fish | Bird) {
  if ("swim" in animal) {
    // TypeScript knows it's a Fish
    animal.swim();
  } else {
    // TypeScript knows it's a Bird
    animal.fly();
  }
}
```

## 2. Custom Type Guards (Type Predicates)

Built-in type guards are great, but sometimes logic to determine a type is complex. If you extract this logic into a helper function, TypeScript loses the type narrowing context by default.

```typescript
type Fish = { swim: () => void };
type Bird = { fly: () => void };

// A normal boolean returning function
function isFish(animal: Fish | Bird): boolean {
  return (animal as Fish).swim !== undefined;
}

function move(animal: Fish | Bird) {
  if (isFish(animal)) {
    // ❌ Error: Property 'swim' does not exist on type 'Fish | Bird'.
    // TypeScript doesn't know the type was narrowed!
    animal.swim(); 
  }
}
```

To fix this, we change the return type from `boolean` to a **Type Predicate**.

A type predicate takes the form `parameterName is Type`.

```typescript
// ✅ Using a Type Predicate: `animal is Fish`
function isFish(animal: Fish | Bird): animal is Fish {
  return (animal as Fish).swim !== undefined;
}

function move(animal: Fish | Bird) {
  if (isFish(animal)) {
    // ✅ TypeScript now knows this is a Fish
    animal.swim();
  } else {
    // ✅ TypeScript knows this is a Bird
    animal.fly();
  }
}
```

When `isFish` returns `true`, TypeScript will narrow the type of the `animal` variable to `Fish` in that block.

## 3. Filtering Arrays with Type Predicates

One of the most common and powerful uses for custom type predicates is filtering arrays. 

Often, when you use `.filter()` to remove `null` or `undefined` values from an array, TypeScript still thinks the resulting array contains those values.

```typescript
const mixedArray = ["apple", null, "banana", undefined, "cherry"];

// The type is still (string | null | undefined)[]
const stringsOnly = mixedArray.filter(item => item !== null && item !== undefined);

// ❌ Error: Object is possibly 'null' or 'undefined'.
stringsOnly[0].toUpperCase(); 
```

To fix this, provide a type predicate to the `.filter()` callback:

```typescript
// Define a generic type guard
function isDefined<T>(value: T | null | undefined): value is T {
  return value !== null && value !== undefined;
}

// The type is now strictly string[]!
const stringsOnly = mixedArray.filter(isDefined);

// ✅ Works perfectly
stringsOnly[0].toUpperCase();
```

## 4. Assertion Functions (Advanced)

Sometimes, instead of returning a boolean to narrow a type in an `if` block, you want a function to simply throw an error if the type isn't what you expect, and narrow the type for the rest of the current scope.

TypeScript 3.7 introduced **Assertion Functions** (`asserts parameter is Type`).

```typescript
type User = { name: string; age: number };

// The `asserts` keyword
function assertIsUser(obj: any): asserts obj is User {
  if (typeof obj !== "object" || obj === null) {
    throw new Error("Not an object");
  }
  if (typeof obj.name !== "string" || typeof obj.age !== "number") {
    throw new Error("Invalid User object");
  }
}

function processData(data: unknown) {
  // data is unknown here
  
  assertIsUser(data); 

  // If the function didn't throw, TypeScript knows `data` is a User!
  console.log(data.name.toUpperCase());
}
```

---

## Summary

- **Type Guards** are checks (like `typeof`, `instanceof`, or `in`) that TypeScript's control flow analysis uses to narrow types.
- **Type Predicates** (`param is Type`) are custom return types that tell TypeScript that a boolean-returning function acts as a type guard.
- Type predicates are essential for maintaining type safety when extracting check logic into separate functions or when filtering arrays.
- **Assertion Functions** (`asserts param is Type`) act as type guards that throw errors instead of returning booleans, narrowing the type for the rest of the scope.
