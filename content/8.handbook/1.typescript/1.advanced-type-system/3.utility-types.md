# Utility Types Mastery

TypeScript comes with a rich set of **global utility types** built into the compiler. These utilities facilitate common type transformations, saving you from writing complex mapped or conditional types from scratch.

Mastering these built-in utilities is essential for writing clean, DRY (Don't Repeat Yourself), and maintainable TypeScript code.

---

## 1. Modifiers: `Partial`, `Required`, `Readonly`

These utilities transform the modifiers of an object type's properties.

### `Partial<Type>`
Constructs a type with all properties of `Type` set to optional. This is incredibly useful for update/patch APIs where you only want to update a subset of fields.

```typescript
interface User {
  id: string;
  name: string;
  age: number;
  email: string;
}

function updateUser(id: string, updates: Partial<User>) {
  // `updates` can be { name: "Alice" } or { age: 31 }
  // You don't need to pass the full User object
}

updateUser('123', { name: 'Bob' }); // ✅
```

### `Required<Type>`
The opposite of `Partial`. Constructs a type consisting of all properties of `Type` set to required, stripping away any `?` modifiers.

```typescript
interface ComponentProps {
  title?: string;
  theme?: 'light' | 'dark';
}

// In the component logic, we might merge with default props,
// guaranteeing that the values exist.
const defaultProps: Required<ComponentProps> = {
  title: 'Default Title',
  theme: 'light',
};
```

### `Readonly<Type>`
Constructs a type with all properties of `Type` set to `readonly`, meaning the properties of the constructed type cannot be reassigned.

```typescript
interface Config {
  apiUrl: string;
  retries: number;
}

const config: Readonly<Config> = {
  apiUrl: 'https://api.example.com',
  retries: 3,
};

// config.retries = 5; // ❌ Error: Cannot assign to 'retries' because it is a read-only property
```

## 2. Structural Transformations: `Pick` & `Omit`

When you have a large interface and need a smaller variation of it, never duplicate the interface. Use `Pick` or `Omit`.

### `Pick<Type, Keys>`
Constructs a type by picking the set of properties `Keys` (string literal or union of string literals) from `Type`.

```typescript
interface Product {
  id: string;
  name: string;
  description: string;
  price: number;
  inventoryCount: number;
  supplierId: string;
}

// For a product listing, we only need a few fields
type ProductPreview = Pick<Product, 'id' | 'name' | 'price'>;

const preview: ProductPreview = {
  id: 'p-1',
  name: 'Wireless Mouse',
  price: 29.99,
  // description: "..." // ❌ Error: Object literal may only specify known properties
};
```

### `Omit<Type, Keys>`
Constructs a type by picking all properties from `Type` and then removing `Keys`. It is essentially the inverse of `Pick`.

```typescript
// When creating a new product, the DB generates the ID
type CreateProductInput = Omit<Product, 'id' | 'inventoryCount'>;

const newProduct: CreateProductInput = {
  name: 'Keyboard',
  description: 'Mechanical keyboard',
  price: 99.99,
  supplierId: 'sup-1',
};
```

**Pro Tip:** Prefer `Pick` when you are defining what you *want*, and `Omit` when you are defining what you *don't want*. If an interface has 20 properties and you need 3, use `Pick`. If you need 17, use `Omit`.

## 3. Dynamic Objects: `Record`

### `Record<Keys, Type>`
Constructs an object type whose property keys are `Keys` and whose property values are `Type`. This utility can be used to map the properties of a type to another type, or to create dictionary/map structures.

```typescript
// A generic dictionary map
type StringMap = Record<string, string>;

const translations: StringMap = {
  hello: 'hola',
  goodbye: 'adiós',
};

// Strongly typed keys
type UserRole = 'admin' | 'editor' | 'viewer';

interface RolePermissions {
  canWrite: boolean;
  canDelete: boolean;
}

// Ensure every single UserRole is accounted for
const permissions: Record<UserRole, RolePermissions> = {
  admin: { canWrite: true, canDelete: true },
  editor: { canWrite: true, canDelete: false },
  viewer: { canWrite: false, canDelete: false },
  // If we forget one, or add a new role later, TS will error here!
};
```

## 4. Union Transformations: `Exclude`, `Extract`, `NonNullable`

These utilities operate on **Union Types** rather than Object Types.

### `Exclude<UnionType, ExcludedMembers>`
Constructs a type by excluding from `UnionType` all union members that are assignable to `ExcludedMembers`.

```typescript
type AvailableColors = 'red' | 'green' | 'blue' | 'yellow';

type PrimaryColors = 'red' | 'blue' | 'yellow';

// "Give me all AvailableColors EXCEPT PrimaryColors"
type SecondaryColors = Exclude<AvailableColors, PrimaryColors>; 
// Result: 'green'
```

### `Extract<Type, Union>`
Constructs a type by extracting from `Type` all union members that are assignable to `Union`. (The inverse of `Exclude`).

```typescript
type MixedUnion = string | number | boolean | string[] | { a: string };

// "Extract only the array or object types"
type ComplexTypes = Extract<MixedUnion, object>; 
// Result: string[] | { a: string }
```

### `NonNullable<Type>`
Constructs a type by excluding `null` and `undefined` from `Type`.

```typescript
type MaybeString = string | null | undefined;

type DefinitelyString = NonNullable<MaybeString>; 
// Result: string
```

## 5. Function Transformations: `Parameters` & `ReturnType`

These utilities extract type information from function signatures. They are incredibly useful when you are using third-party libraries and need to type a variable based on a function they expose, but they didn't export the types directly.

### `ReturnType<Type>`
Constructs a type consisting of the return type of function `Type`.

```typescript
function createDatabaseConnection(config: any) {
  return {
    connect: () => true,
    disconnect: () => false,
    query: (sql: string) => [],
  };
}

// We want to type a variable holding the connection, but the library
// didn't export a 'DatabaseConnection' interface.
type DbConnection = ReturnType<typeof createDatabaseConnection>;

let conn: DbConnection;
```

### `Parameters<Type>`
Constructs a tuple type from the types used in the parameters of a function type `Type`.

```typescript
function registerUser(username: string, age: number, isVip: boolean) {}

// Extracts parameters as a Tuple: [username: string, age: number, isVip: boolean]
type RegistrationArgs = Parameters<typeof registerUser>;

// We can use this to strongly type an array of arguments
const args: RegistrationArgs = ['alice123', 25, true];

// And spread them into the function!
registerUser(...args); 
```

### `Awaited<Type>` (Promises)
This type is meant to model operations like `await` in `async` functions, or the `.then()` method on Promises - specifically, the way they recursively unwrap Promises.

```typescript
async function fetchUser() {
  return { name: 'Alice', age: 30 };
}

// ReturnType gives us Promise<{ name: string, age: number }>
type FetchPromise = ReturnType<typeof fetchUser>;

// Awaited unwraps the Promise to give us { name: string, age: number }
type User = Awaited<FetchPromise>;
```

## 6. String Manipulation Utilities

Introduced in TypeScript 4.1 to work alongside template literal types, these utilities change the casing of string literal types.

```typescript
// Uppercase
type UppercaseGreeting = Uppercase<"hello">; // "HELLO"

// Lowercase
type LowercaseGreeting = Lowercase<"HELLO">; // "hello"

// Capitalize (capitalizes the first letter)
type CapitalizedGreeting = Capitalize<"hello world">; // "Hello world"

// Uncapitalize (uncapitalizes the first letter)
type UncapitalizedGreeting = Uncapitalize<"Hello World">; // "hello World"
```

### Practical Example with String Utilities

You can combine these with mapped types to automatically generate types for generated code, like getters and setters:

```typescript
type State = {
  name: string;
  age: number;
};

// Generate an object type with 'set' prefix and capitalized keys
type Setters<T> = {
  [K in keyof T as `set${Capitalize<string & K>}`]: (value: T[K]) => void;
};

type StateSetters = Setters<State>;
/* Result:
{
  setName: (value: string) => void;
  setAge: (value: number) => void;
}
*/
```

## Summary Cheat Sheet

| Utility | Purpose | Example |
|---|---|---|
| `Partial<T>` | Make all properties optional | `Partial<{a: string}>` → `{a?: string}` |
| `Required<T>` | Make all properties required | `Required<{a?: string}>` → `{a: string}` |
| `Readonly<T>` | Make all properties readonly | `Readonly<{a: string}>` → `{readonly a: string}` |
| `Record<K, T>` | Create map/dictionary | `Record<'x', number>` → `{x: number}` |
| `Pick<T, K>` | Keep only specified properties | `Pick<{a:1, b:2}, 'a'>` → `{a:1}` |
| `Omit<T, K>` | Remove specified properties | `Omit<{a:1, b:2}, 'a'>` → `{b:2}` |
| `Exclude<T, U>` | Remove U from Union T | `Exclude<'a'|'b', 'a'>` → `'b'` |
| `Extract<T, U>` | Keep only U in Union T | `Extract<'a'|'b'|1, string>` → `'a'|'b'` |
| `NonNullable<T>` | Remove null/undefined | `NonNullable<string|null>` → `string` |
| `ReturnType<T>` | Get function return type | `ReturnType<()=>string>` → `string` |
| `Parameters<T>` | Get function params as tuple | `Parameters<(x: string)=>void>` → `[x: string]` |
| `Awaited<T>` | Unwrap Promise types deeply | `Awaited<Promise<string>>` → `string` |
