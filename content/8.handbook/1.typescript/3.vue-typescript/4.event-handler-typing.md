# Event Handler Typing in Vue + TypeScript

Handling native DOM events and custom component emits securely is a core part of building type-safe Vue applications. TypeScript provides specific types for different DOM events, which ensures that you only access properties that truly exist on the event payload.

---

## 1. Typing Native DOM Events

When you attach an event listener in a Vue template (like `@click` or `@input`), Vue automatically passes the native DOM event to your handler.

If you write your handler inline, Vue infers the type automatically:

```vue
<!-- ✅ TypeScript knows `$event` is a MouseEvent -->
<button @click="(e) => console.log(e.clientX)">Click Me</button>
```

However, if you extract the handler to a function in `<script setup>`, you must explicitly type the event parameter, otherwise it defaults to `any` (or throws an error in strict mode).

### Common DOM Event Types

```vue
<script setup lang="ts">
// 🖱️ Mouse Events
const handleClick = (event: MouseEvent) => {
  console.log(event.clientX, event.clientY);
};

// ⌨️ Keyboard Events
const handleKeyDown = (event: KeyboardEvent) => {
  if (event.key === 'Enter') {
    console.log("Enter pressed");
  }
};

// 폼 Form/Input Events
const handleInput = (event: Event) => {
  // TypeScript doesn't know what HTML element triggered the Event!
  // We must cast the target.
  const target = event.target as HTMLInputElement;
  console.log(target.value); 
};
</script>

<template>
  <button @click="handleClick">Click</button>
  <input @keydown="handleKeyDown" @input="handleInput" />
</template>
```

### The `Event` vs `HTMLInputElement` Gotcha

Notice `handleInput` above. Why is the type just `Event` and not `InputEvent`?

Because `event.target` could technically be any DOM element, standard DOM types type `event.target` as a generic `EventTarget`. You must use type assertions (`as HTMLInputElement`) to access element-specific properties like `.value` or `.checked`.

## 2. Typing Custom Component Emits (`defineEmits`)

When building your own components, you use `defineEmits` to trigger custom events. Vue 3.3+ provides two excellent ways to strongly type these.

### Type-Based Emits (Vue 3.3+)

The most modern and readable way to type emits is using the type-based syntax, which looks like a standard TypeScript interface.

```vue
<script setup lang="ts">
// Define what events can be emitted, and their specific payloads
const emit = defineEmits<{
  // 'eventName': [argument1: Type, argument2: Type]
  submit: [id: number, rawData: string];
  update: [value: string];
  close: []; // No payload
}>();

const triggerSubmit = () => {
  // ✅ Autocomplete and type-checking works here!
  emit('submit', 123, "data_string");
  
  // ❌ Error: Expected 2 arguments, but got 1
  // emit('submit', 123); 
};
</script>
```

### The Alternative Tuple Syntax (Older / Optional)

Prior to Vue 3.3, you had to use a slightly more verbose call signature syntax. You will still see this heavily in existing codebases.

```vue
<script setup lang="ts">
const emit = defineEmits<{
  (e: 'submit', id: number, rawData: string): void;
  (e: 'update', value: string): void;
  (e: 'close'): void;
}>();
</script>
```
Both are perfectly valid, but the newer tuple syntax (`submit: [id: number]`) is generally preferred for readability.

## 3. Typing `v-model` Updates

A very common pattern in Vue is building custom inputs using `v-model`. Under the hood, `v-model="myVal"` expands to a prop (`modelValue`) and an emit (`update:modelValue`).

To type this strictly:

```vue
<script setup lang="ts">
defineProps<{
  modelValue: string;
}>();

const emit = defineEmits<{
  // Note the quotes around the event name because of the colon
  'update:modelValue': [value: string]; 
}>();

const updateValue = (event: Event) => {
  const target = event.target as HTMLInputElement;
  emit('update:modelValue', target.value);
};
</script>

<template>
  <input :value="modelValue" @input="updateValue" />
</template>
```

*(Note: Vue 3.4+ introduced the `defineModel()` macro which abstracts this entirely, making it dramatically simpler!)*

```vue
<script setup lang="ts">
// Vue 3.4+ magic: Automatically handles the prop AND the emit securely!
const model = defineModel<string>({ required: true });
</script>

<template>
  <!-- Directly binds, fully type-safe -->
  <input v-model="model" /> 
</template>
```

---

## Summary

- Use specific DOM Event types (`MouseEvent`, `KeyboardEvent`, `Event`) when extracting native handlers to `<script setup>`.
- Remember to cast `event.target as HTMLInputElement` (or similar) when accessing `.value`.
- Use the modern type-based syntax in `defineEmits<{ eventName: [payload: Type] }>()` to strictly type your custom component events.
- If using Vue 3.4+, rely on `defineModel<Type>()` to handle two-way binding types instantly without manually writing `update:modelValue` emits.
