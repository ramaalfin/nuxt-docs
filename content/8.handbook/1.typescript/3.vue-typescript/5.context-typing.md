# Context Typing (Provide / Inject)

In Vue, "Context" typically refers to two things:
1. The **Setup Context** (which contains `attrs`, `slots`, and `emit` in the traditional `setup()` function).
2. The **Dependency Injection System** (`provide` and `inject`), which acts exactly like React's Context API.

When building complex applications, sharing state deeply across the component tree without prop-drilling requires strict type safety to prevent runtime bugs.

---

## 1. Typing Provide / Inject (The Vue Context API)

By default, when you `inject` a value in Vue, its type is unknown (or `any` in loose configurations), because Vue has no way to trace where the `provide` came from.

```typescript
// ❌ Untyped Injection
const theme = inject('theme'); 
// TypeScript thinks 'theme' is `unknown`.
```

To achieve strict type safety across the application, Vue provides the **`InjectionKey`** utility.

### Step 1: Define the InjectionKey

Create a central file (e.g., `keys.ts`) where you define the specific symbol or string key, wrapped in an `InjectionKey`.

```typescript
// keys.ts
import type { InjectionKey, Ref } from 'vue';

// Define the exact shape of the data that will be provided
export interface ThemeContext {
  color: Ref<string>;
  toggleTheme: () => void;
}

// Create a strongly-typed key!
export const ThemeKey: InjectionKey<ThemeContext> = Symbol('ThemeContext');
```

### Step 2: Providing the Data

In your parent component (the Provider), use the key when calling `provide`.

```vue
<!-- Provider.vue -->
<script setup lang="ts">
import { provide, ref } from 'vue';
import { ThemeKey } from './keys';

const color = ref('dark');
const toggleTheme = () => {
  color.value = color.value === 'dark' ? 'light' : 'dark';
};

// ✅ Type-safe! If you try to provide a number instead of the 
// ThemeContext object, TypeScript will throw an error.
provide(ThemeKey, { color, toggleTheme });
</script>
```

### Step 3: Injecting the Data

In any deeply nested child component, use the same key. Vue automatically infers the return type of `inject` based on the `InjectionKey`!

```vue
<!-- Consumer.vue -->
<script setup lang="ts">
import { inject } from 'vue';
import { ThemeKey } from './keys';

// ✅ `theme` is strictly typed as `ThemeContext | undefined`
const theme = inject(ThemeKey);

if (!theme) {
  throw new Error("Consumer must be used within a ThemeProvider!");
}

// Autocomplete works perfectly here!
console.log(theme.color.value);
</script>
```

---

## 2. Default Injection Values

If you want to guarantee that `inject` never returns `undefined`, you can provide a default value.

```typescript
const theme = inject(ThemeKey, {
  // We provide a fallback implementation
  color: ref('light'),
  toggleTheme: () => console.warn("No ThemeProvider found")
});

// ✅ `theme` is strictly typed as `ThemeContext` (no undefined!)
```

## 3. Typing the Setup Context (For Non-`<script setup>` users)

If you are maintaining an older Vue 3 codebase, or writing a complex component where you opted out of `<script setup>` and are using the explicit `setup()` function, you need to type the Context object manually.

The context object contains `attrs`, `slots`, `emit`, and `expose`.

```typescript
import { defineComponent, type SetupContext } from 'vue';

export default defineComponent({
  props: {
    title: String
  },
  emits: ['update'],
  
  // Type the context!
  setup(props, context: SetupContext<['update']>) {
    
    // ✅ Autocompletes 'update'
    context.emit('update');
    
    // Fallthrough attributes (Record<string, unknown>)
    console.log(context.attrs);
    
    // Available slots
    console.log(context.slots.default?.());

    return () => {};
  }
})
```

*(Note: When using `<script setup>`, you do not need to type the context object. You use `useAttrs()`, `useSlots()`, and `defineEmits()` instead, which are all inherently type-safe without manual intervention).*

---

## Summary

- Use **`InjectionKey<T>`** to create strictly typed keys for your `provide` and `inject` flows (Vue's equivalent of React Context).
- Always export your `InjectionKey` from a central file so both the Provider and Consumer components can import it.
- If you aren't using a default value, remember that `inject` returns `T | undefined`. You should narrow this using an `if (!context)` check.
- If you are still using the explicit `setup(props, ctx)` function, use the `SetupContext<EmitTypes>` interface to strongly type the `ctx.emit` function.
