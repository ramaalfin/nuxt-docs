# ForwardRef with Generics (Template Refs)

In React, passing refs to child components requires a specific `forwardRef` API. In Vue 3, this concept is handled naturally via **Template Refs** (`ref="myChild"`) and the **`defineExpose`** macro.

However, when you combine Template Refs with **Generic Components** (`<script setup generic="T">`), extracting the correct type for the ref can become tricky.

---

## 1. The Standard Template Ref

Normally, to type a template ref for a child component, you import the component and use the `InstanceType` utility.

```vue
<!-- Parent.vue -->
<script setup lang="ts">
import { ref } from 'vue';
import MyChild from './MyChild.vue';

// 1. Type the ref using InstanceType
const childRef = ref<InstanceType<typeof MyChild> | null>(null);

const doSomething = () => {
  // Autocomplete works for exposed methods!
  childRef.value?.someExposedMethod();
};
</script>

<template>
  <MyChild ref="childRef" />
</template>
```

This works perfectly for standard components, but breaks down when `MyChild.vue` is generic.

---

## 2. The Problem with Generic Refs

Let's say `GenericList.vue` looks like this:

```vue
<!-- GenericList.vue -->
<script setup lang="ts" generic="T">
defineProps<{ items: T[] }>();

const getSelectedItem = (): T | null => {
  // ... implementation
  return null;
}

// Expose the method to the parent
defineExpose({ getSelectedItem });
</script>
```

If you try to use `InstanceType<typeof GenericList>` in the parent, TypeScript doesn't know what `T` is supposed to be. It will often fallback to `unknown` or throw a generic constraint error.

```typescript
// ❌ `T` is unknown here! getSelectedItem() will return `unknown | null`
const listRef = ref<InstanceType<typeof GenericList> | null>(null);
```

---

## 3. The Solution: Helper Types

Because `InstanceType` extracts the type of the constructor *without* providing generic arguments, we need to create a helper type to pass our specific generic argument into the component's type definition.

### Approach 1: The Vue 3.3+ Clean Approach

With recent updates to Volar (Vue's language server) and Vue 3.3+, you can often cast the component's type directly by creating an intersection or using `typeof` with generic arguments if your IDE supports it, but the most reliable cross-platform way is to extract the exposed type directly.

Instead of trying to type the entire component instance, you can type **just what is exposed**.

```vue
<!-- GenericList.vue -->
<script setup lang="ts" generic="T">
// ... props and logic

const getSelectedItem = (): T | null => null;
const clearSelection = () => {};

// 1. Group what you expose into an object
const exposed = {
  getSelectedItem,
  clearSelection
};

defineExpose(exposed);

// 2. Export the TYPE of the exposed object
export type GenericListExposed<T> = typeof exposed;
</script>
```

**Usage in Parent:**

```vue
<!-- Parent.vue -->
<script setup lang="ts">
import { ref } from 'vue';
import GenericList, { type GenericListExposed } from './GenericList.vue';

interface User { id: number; name: string }

// ✅ We strictly type the ref using our exported interface, providing `User` as `T`!
// (We union with ComponentPublicInstance to keep standard Vue ref properties like $el)
import type { ComponentPublicInstance } from 'vue';
type TypedListRef = ComponentPublicInstance & GenericListExposed<User>;

const listRef = ref<TypedListRef | null>(null);

const checkSelection = () => {
  // ✅ TypeScript KNOWS this returns a User | null!
  const user = listRef.value?.getSelectedItem();
  console.log(user?.name);
};
</script>

<template>
  <GenericList ref="listRef" :items="[{ id: 1, name: 'Alice' }]" />
</template>
```

---

## 4. Why this matters (Component Wrappers)

This pattern is especially critical when you are building **Wrapper Components**—for example, wrapping a complex third-party grid library or a UI framework's select component.

You want the parent to be able to call `wrapperRef.value.focus()` or `wrapperRef.value.getSelectedRow()`, but you need `getSelectedRow()` to return the exact generic type `T` that the parent passed into the wrapper.

By exporting the `Exposed<T>` interface from your generic component, you create a robust, type-safe API contract between the parent and the generic child.

---

## Summary

- Standard component refs are typed using `ref<InstanceType<typeof Component> | null>(null)`.
- **Generic Components** lose their generic type `T` when you use `InstanceType`.
- To fix this, group your `defineExpose` properties into an object, and `export type MyComponentExposed<T> = typeof exposed`.
- In the parent, import that type and pass your specific generic argument to it (`ref<MyComponentExposed<User> | null>(null)`).
