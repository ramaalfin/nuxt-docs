# Generic Components

In Vue 3, components often act as wrappers around data. Think of a `<DataTable>`, a `<CustomSelect>`, or a `<ListGroup>`. These components shouldn't care about the *exact* shape of the data they are rendering, as long as the data meets a minimum structural requirement.

Historically, typing these components in Vue was incredibly difficult and often required falling back to `any`. Vue 3.3 officially introduced the `<script setup generic="...">` attribute, making first-class generic components a reality.

---

## 1. What is a Generic Component?

A generic component is a component that accepts a Type Parameter (just like a generic TypeScript function) from its parent component. 

This type flows through the component, ensuring that the **props**, **emitted events**, and **scoped slots** all share the exact same strict typing.

### The Problem without Generics

```vue
<!-- ❌ Untyped CustomSelect.vue -->
<script setup lang="ts">
// We have to use 'any' or 'unknown'
const props = defineProps<{
  items: any[];
  modelValue: any;
}>();

const emit = defineEmits<{
  (e: 'update:modelValue', value: any): void;
}>();
</script>
```

If a parent passes an array of `User` objects into `items`, the parent has no guarantee that the `@update:modelValue` event will emit a `User`.

---

## 2. Basic Syntax (`generic="..."`)

To make a component generic, add the `generic` attribute to the `<script setup>` tag.

```vue
<!-- ✅ Generic CustomSelect.vue -->
<script setup lang="ts" generic="T">
// T represents the type of the item passed into the component

const props = defineProps<{
  items: T[];
  modelValue?: T;
}>();

const emit = defineEmits<{
  (e: 'update:modelValue', value: T): void;
}>();

const select = (item: T) => {
  emit('update:modelValue', item);
};
</script>
```

**How the parent uses it:**

Vue's compiler is incredibly smart. It looks at the `items` prop passed by the parent and **automatically infers** the type `T`!

```vue
<script setup lang="ts">
import CustomSelect from './CustomSelect.vue';

interface User { id: number; name: string }
const users: User[] = [{ id: 1, name: "Alice" }, { id: 2, name: "Bob" }];
</script>

<template>
  <!-- Vue infers T = User based on the :items array! -->
  <CustomSelect 
    :items="users" 
    @update:model-value="user => console.log(user.name)" 
  />
  <!-- The IDE knows `user` is strictly of type `User` -->
</template>
```

## 3. Extending/Constraining Generics

Sometimes, you need to guarantee that the generic type `T` has certain properties. For example, a `DataTable` might require that every row has an `id` property to act as a unique key.

You can constrain the generic type using the `extends` keyword, exactly like standard TypeScript.

```vue
<!-- DataTable.vue -->
<script setup lang="ts" generic="T extends { id: string | number }">
const props = defineProps<{
  data: T[];
}>();
</script>

<template>
  <table>
    <tbody>
      <!-- We know item.id exists because of the generic constraint! -->
      <tr v-for="item in data" :key="item.id">
        <!-- ... -->
      </tr>
    </tbody>
  </table>
</template>
```

If a parent tries to pass an array of objects that *don't* have an `id` property, TypeScript will throw a compile-time error.

---

## 4. Multiple Generics

You can define multiple generics by separating them with a comma.

A common use case is a component that takes a list of objects and a specific "key" to extract from those objects.

```vue
<!-- ExtractValue.vue -->
<script setup lang="ts" generic="T, K extends keyof T">
const props = defineProps<{
  items: T[];
  displayKey: K;
}>();
</script>

<template>
  <ul>
    <!-- TypeScript knows item[displayKey] is safe to access -->
    <li v-for="item in items">
      {{ item[displayKey] }}
    </li>
  </ul>
</template>
```

## 5. Typing Scoped Slots

One of the most powerful aspects of generic components is passing the generic type down into **Scoped Slots**. This allows the parent component to define how an item should be rendered while maintaining perfect type safety.

```vue
<!-- GenericList.vue -->
<script setup lang="ts" generic="T">
defineProps<{ items: T[] }>();
</script>

<template>
  <ul>
    <li v-for="(item, index) in items" :key="index">
      <!-- We pass the generically-typed item back to the parent -->
      <slot name="item" :item="item" :index="index">
        <!-- Default fallback -->
        {{ item }}
      </slot>
    </li>
  </ul>
</template>
```

**Usage in the Parent:**

```vue
<template>
  <GenericList :items="users">
    <!-- `slotProps.item` is perfectly typed as a User! -->
    <template #item="slotProps">
      <div class="user-card">
        <h3>{{ slotProps.item.name }}</h3>
        <p>ID: {{ slotProps.item.id }}</p>
      </div>
    </template>
  </GenericList>
</template>
```

---

## Summary

- Use `<script setup generic="T">` to create highly reusable components that adapt to the data they are given.
- Vue automatically infers the generic type based on the props passed by the parent.
- Use `extends` to constrain the generic type (e.g., `generic="T extends Record<string, any>"` or `generic="T extends { id: number }"`) to ensure the data meets minimum structural requirements.
- Generics flow perfectly from **Props** -> to **Emits** -> to **Scoped Slots**, providing 100% End-to-End type safety between parent and child components.
