# Composite (GPU Acceleration, Blend Modes)

The final step in the Critical Rendering Path is **Compositing**. 

By the time the browser reaches this stage, it has already calculated the geometry (Layout) and rasterized the pixels for individual elements into separate, independent bitmaps called **Layers** (Paint). 

The Compositor's job is to take all these separate layers, figure out how they overlap, apply any necessary visual effects (like opacity or blend modes), and draw the final merged image to the screen.

---

## 1. The Compositor Thread and GPU Acceleration

The most important thing to know about Compositing is that it usually happens on a dedicated **Compositor Thread**, which heavily utilizes the **GPU (Graphics Processing Unit)**.

Because the Compositor Thread runs independently of the Main Thread (where JavaScript, DOM construction, and Layout run), it can perform visual updates even if the Main Thread is completely frozen by a heavy JavaScript task.

### Why GPU Acceleration is Fast
The GPU is essentially a massive, highly optimized calculator built specifically for manipulating pixels in parallel grids. 
When the Compositor wants to move a layer `100px` to the right, it doesn't recalculate the layout or repaint the text. It simply tells the GPU: *"Take this existing bitmap in VRAM and shift its display coordinates by 100."*

This operation is mathematically trivial for a GPU, taking fractions of a millisecond, guaranteeing a silky-smooth 60 frames-per-second (fps).

## 2. Triggering Compositor-Only Animations

To achieve performant web animations, your goal is to create **Compositor-only animations**. This means the animation completely skips the Layout and Paint phases and runs exclusively on the Compositor Thread.

Currently, modern browsers can only animate two CSS properties exclusively on the Compositor:

1. **`transform`** (e.g., `translate`, `scale`, `rotate`)
2. **`opacity`**

```css
/* ❌ BAD: Animating 'left' triggers Layout + Paint + Composite on every frame */
.box {
  left: 0;
  transition: left 0.3s ease;
}
.box:hover {
  left: 100px; 
}

/* ✅ GOOD: Animating 'transform' triggers ONLY Composite on every frame */
.box {
  transform: translateX(0);
  transition: transform 0.3s ease;
}
.box:hover {
  transform: translateX(100px); 
}
```

### The `will-change` Property
You can explicitly tell the browser to promote an element to its own layer ahead of time using `will-change: transform` or `will-change: opacity`. This prepares the GPU for the animation, preventing visual stuttering when the animation begins. 

*Warning:* Overusing `will-change` consumes excessive Video RAM (VRAM) and can crash lower-end mobile devices. Use it sparingly.

## 3. CSS Blend Modes

The Compositor isn't just for moving layers around; it also handles how the pixels of overlapping layers interact with each other. This is where **CSS Blend Modes** come in.

Historically, blending two images required Photoshop. Today, the browser's GPU can calculate complex math between overlapping pixels in real-time.

### `mix-blend-mode`
This property dictates how an element's content should blend with the background of its parent or sibling elements.

```css
.text-overlay {
  color: white;
  mix-blend-mode: difference; /* Inverts the text color based on what is behind it */
}
```

Common Blend Modes:
- **`multiply`:** Multiplies the colors of the overlapping layers. Always results in a darker color (great for removing white backgrounds from logos).
- **`screen`:** Inverts, multiplies, and inverts again. Always results in a lighter color.
- **`difference`:** Subtracts the darker color from the lighter color. If the background is white, the text becomes black, and vice versa.

### `background-blend-mode`
This property dictates how a single element's `background-image` should blend with its own `background-color`.

```css
.hero-banner {
  background-image: url('photo.jpg');
  background-color: blue;
  background-blend-mode: multiply; /* Tints the entire photo blue */
}
```

### Performance Impact of Blend Modes
Blend modes force the Compositor to perform mathematical operations on every overlapping pixel. While GPUs are fast, applying complex blend modes (like `difference` or `overlay`) over massive, full-screen areas *can* cause performance hits on low-end mobile devices. Always test on real hardware.

---

## Summary

1. **The Compositor Thread:** Runs independently of the Main Thread and uses the GPU to merge painted layers into the final screen image.
2. **GPU Acceleration:** Animating `transform` and `opacity` skips the expensive Layout and Paint phases, running exclusively on the Compositor for buttery-smooth 60fps.
3. **Blend Modes:** `mix-blend-mode` and `background-blend-mode` allow the GPU to perform Photoshop-like pixel math (like `multiply` and `difference`) between overlapping layers in real-time.
