# Hoisting & The Temporal Dead Zone (TDZ)

When learning JavaScript, one of the most confusing behaviors is the ability to use functions and variables *before* they are defined in the code. This phenomenon is known as **Hoisting**. 

However, with the introduction of ES6 (`let` and `const`), JavaScript introduced a new concept to make hoisting safer and more predictable: **The Temporal Dead Zone (TDZ)**.

---

## 1. What is Hoisting?

As discussed in the *Call Stack & Execution Context* chapter, whenever an Execution Context is created, it goes through a **Creation Phase** before the code is actually executed line-by-line.

During this Creation Phase, the JavaScript engine scans the code for variable and function declarations and allocates memory for them. This makes it *appear* as though these declarations have been physically moved (hoisted) to the top of the file.

### Function Hoisting
Function *declarations* are fully hoisted. The engine places the entire function into memory, meaning you can safely call the function before its definition appears in the code.

```javascript
// ✅ Works perfectly!
greet(); 

function greet() {
  console.log("Hello, World!");
}
```

### Variable Hoisting (`var`)
Variables declared with `var` are also hoisted, but differently. The engine allocates memory for the variable, but it initializes it with a default value of `undefined`. The actual assignment happens during the Execution Phase.

```javascript
// ❌ Does not throw an error, but logs 'undefined'
console.log(playerName); 

var playerName = "Alice";

// ✅ Now it logs 'Alice'
console.log(playerName); 
```

### Function Expressions vs Declarations
It is critical to note that Function *Expressions* (including Arrow Functions) assigned to a `var` are hoisted as variables, not as functions.

```javascript
// ❌ TypeError: sayHi is not a function (it is currently 'undefined')
sayHi(); 

var sayHi = function() {
  console.log("Hi!");
};
```

---

## 2. The Temporal Dead Zone (TDZ)

The behavior of `var` (returning `undefined` when accessed early) was a massive source of bugs in early JavaScript development. To fix this, ES6 introduced `let` and `const`.

Variables declared with `let` and `const` **are still hoisted** to the top of their block scope. However, unlike `var`, they are *not* initialized with `undefined`. 

Instead, they are placed into the **Temporal Dead Zone (TDZ)**.

### What is the TDZ?
The Temporal Dead Zone is the period of time from the start of the block scope until the exact line where the variable is declared and initialized. 

If you try to access a variable while it is in the TDZ, the engine will throw a `ReferenceError`.

```javascript
{
  // 🔴 TDZ starts here (start of block)
  
  // ❌ ReferenceError: Cannot access 'score' before initialization
  console.log(score); 
  
  let score = 100; // 🟢 TDZ ends here
  
  console.log(score); // 100
}
```

### Proving `let` and `const` are actually hoisted
Some developers mistakenly believe that `let` and `const` are simply not hoisted at all. We can prove they *are* hoisted by observing how they shadow outer variables:

```javascript
const name = "Alice";

{
  console.log(name); // ❌ ReferenceError!
  const name = "Bob"; 
}
```
If `const name = "Bob"` wasn't hoisted, `console.log(name)` would simply look up the scope chain and print `"Alice"`. But because it *is* hoisted, it shadows the outer variable. However, because it is in the TDZ, it throws an error.

---

## 3. Best Practices

Because of the confusion surrounding hoisting and the TDZ, modern JavaScript development follows these strict rules:

1. **Never use `var`.** Use `const` by default, and `let` only if the variable must be reassigned. This ensures that accessing a variable too early results in a loud, immediate error (TDZ) rather than a silent bug (`undefined`).
2. **Declare variables at the top of their scope.** Don't bury declarations deep inside massive blocks of code.
3. **Use Arrow Functions or Function Expressions assigned to `const`.** This forces you to define your functions before you use them, creating a linear, predictable, top-down reading experience.

---

## Summary

1. **Hoisting:** The JavaScript engine's behavior of allocating memory for variable and function declarations during the Creation Phase, before code execution begins.
2. **`var` Hoisting:** Variables are hoisted and initialized as `undefined`.
3. **Function Hoisting:** Function *declarations* are hoisted in their entirety and can be called early. Function *expressions* behave like variables.
4. **Temporal Dead Zone (TDZ):** The period between the start of a block and the declaration of a `let` or `const` variable. Accessing the variable during this time throws a `ReferenceError`.
