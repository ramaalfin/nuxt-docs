# Composable Hooks (Nesting and Composition)

One of the most powerful features of Vue 3's Composition API is the ability to compose logic. Unlike Vue 2 Mixins, which suffered from implicit dependencies and namespace collisions, Custom Hooks (Composables) can safely use other Composables to build complex, layered logic trees.

This pattern is known as **Composable Hooks**.

## The Concept of Composition

Composition means combining simple, single-responsibility functions to build more complex behaviors. A Composable does not have to be a completely isolated island; it can consume other Composables to do its job.

### Example: A Complex "Checkout" Hook
Imagine an e-commerce application. A checkout process requires checking authentication, interacting with the shopping cart, and making a network request.

Instead of writing all this logic from scratch inside a monolithic `useCheckout.ts`, we compose it using existing hooks.

```typescript
// composables/useCheckout.ts

// 1. We import (or auto-import in Nuxt) our primitive hooks
import { useAuth } from './useAuth'
import { useCart } from './useCart'
import { useNotification } from './useNotification'

export function useCheckout() {
  // 2. We initialize the hooks we depend on
  const { user, isAuthenticated } = useAuth()
  const { items, clearCart, cartTotal } = useCart()
  const { notifySuccess, notifyError } = useNotification()
  
  const isProcessing = ref(false)

  // 3. We build complex logic by coordinating the primitive hooks
  const submitOrder = async () => {
    if (!isAuthenticated.value) {
      notifyError('You must be logged in to checkout.')
      return
    }

    if (items.value.length === 0) {
      notifyError('Your cart is empty.')
      return
    }

    isProcessing.value = true

    try {
      await $fetch('/api/checkout', {
        method: 'POST',
        body: { userId: user.value.id, items: items.value, total: cartTotal.value }
      })
      
      clearCart()
      notifySuccess('Order placed successfully!')
    } catch (error) {
      notifyError('Checkout failed. Please try again.')
    } finally {
      isProcessing.value = false
    }
  }

  return {
    isProcessing,
    submitOrder
  }
}
```

## Best Practices for Composing Hooks

### 1. Avoid Deep Nesting
While composing hooks is encouraged, nesting them too deeply (Hook A uses Hook B, which uses Hook C, which uses Hook D) creates implicit coupling and makes debugging difficult. Try to keep the composition shallow. If you find yourself nesting 4 levels deep, consider flattening the architecture by orchestrating them at the component level instead.

### 2. Pass Reactive State as Arguments
Sometimes, it's better to pass reactive references into a hook rather than having the hook instantiate them. This is known as **Dependency Injection via Arguments**.

**Bad (Tightly Coupled):**
```javascript
export function useFilteredUsers() {
  // Tightly coupled to the global user store
  const { users } = useUserStore() 
  const searchQuery = ref('')
  
  const filtered = computed(() => /* filter logic */)
  return { searchQuery, filtered }
}
```

**Good (Composable & Reusable):**
```javascript
import { isRef, unref } from 'vue'

// Accepts users as an argument (can be a ref or raw array)
export function useFilteredUsers(usersList) {
  const searchQuery = ref('')
  
  const filtered = computed(() => {
    const rawUsers = unref(usersList)
    return rawUsers.filter(u => u.name.includes(searchQuery.value))
  })
  
  return { searchQuery, filtered }
}
```
Now, `useFilteredUsers` can be used with a Pinia store, a local API fetch, or a hardcoded array.

### 3. Lifecycle Hooks Awareness
If Composable A registers an `onMounted` or `onUnmounted` hook, and Composable B consumes Composable A, those lifecycle hooks will run when the Vue component using Composable B mounts/unmounts.
Always be aware of hidden lifecycle events when composing hooks. A composable should rarely hide destructive or heavy operations inside lifecycle hooks without explicitly documenting it.

### 4. Managing Nuxt Context
In Nuxt 3, Composables have access to the Nuxt Context (`useNuxtApp()`, `useRoute()`, `useFetch()`). However, this context is only available during the setup phase of a component or plugin. If you invoke a Composable inside an async callback or setTimeout, the Nuxt Context might be lost.

Always call composed hooks synchronously at the top level of your setup function or parent hook to ensure the Nuxt instance is properly passed down the composition chain.
