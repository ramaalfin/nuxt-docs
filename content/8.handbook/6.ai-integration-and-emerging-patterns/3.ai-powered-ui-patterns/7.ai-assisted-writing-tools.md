# AI-Assisted Writing Tools

Integrating AI into rich text editors (like TipTap or ProseMirror) creates a superpower for users. Whether they are drafting emails, writing documentation, or creating marketing copy, AI writing assistants require specific UX patterns to feel collaborative rather than intrusive.

## 1. The "Magic Wand" Highlight Menu

When a user selects text, provide contextual AI actions.

```
[User highlights: "The meeting is on Tuesday"]
  ↳ 🪄 AI Actions:
      - Expand / Elaborate
      - Make more professional
      - Fix grammar & spelling
      - Summarize
```

### Implementation with TipTap Bubble Menu

```vue
<!-- components/Editor.vue -->
<script setup lang="ts">
import { useEditor, BubbleMenu, EditorContent } from '@tiptap/vue-3'
import StarterKit from '@tiptap/starter-kit'

const editor = useEditor({ extensions: [StarterKit], content: '<p>Start writing...</p>' })
const isProcessing = ref(false)

const applyAiTransform = async (action: 'expand' | 'professional' | 'grammar') => {
  if (!editor.value) return
  
  const selectedText = editor.value.state.doc.textBetween(
    editor.value.state.selection.from,
    editor.value.state.selection.to
  )
  
  if (!selectedText) return

  isProcessing.value = true
  try {
    const { text } = await $fetch('/api/ai-edit', {
      method: 'POST',
      body: { text: selectedText, action }
    })
    
    // Replace the selected text with the AI's result
    editor.value.chain().focus().insertContent(text).run()
  } finally {
    isProcessing.value = false
  }
}
</script>

<template>
  <div class="editor-wrapper">
    <BubbleMenu v-if="editor" :editor="editor" :tippy-options="{ duration: 100 }">
      <div class="ai-menu bg-white shadow-lg rounded-lg border p-1 flex gap-1">
        <button 
          class="px-2 py-1 hover:bg-gray-100 rounded text-sm" 
          @click="applyAiTransform('professional')"
          :disabled="isProcessing"
        >
          👔 Professional
        </button>
        <button 
          class="px-2 py-1 hover:bg-gray-100 rounded text-sm" 
          @click="applyAiTransform('grammar')"
          :disabled="isProcessing"
        >
          ✨ Fix Grammar
        </button>
      </div>
    </BubbleMenu>
    
    <EditorContent :editor="editor" class="prose max-w-none" />
  </div>
</template>
```

## 2. In-Line Generation (Notion Style)

Allow users to type `/ai` or press `Space` on an empty line to summon an AI generator that streams text directly into the document.

```typescript
// Use TipTap's NodeView or a custom extension to handle the streaming insertion
const generateInline = async (prompt: string) => {
  const stream = await $fetch('/api/generate-stream', {
    method: 'POST', body: { prompt }, responseType: 'stream'
  })
  
  // Create a placeholder node where the cursor is
  let currentPos = editor.value.state.selection.from
  
  // As chunks arrive, insert them into the editor
  for await (const chunk of stream) {
    editor.value.chain().focus().insertContentAt(currentPos, chunk).run()
    currentPos += chunk.length
  }
}
```

## 3. "Accept/Reject" Diff Views

Never silently overwrite user text without their approval. When AI suggests an edit, show the difference and allow the user to accept, reject, or regenerate.

```
Original: "I think we should probably launch it next week."
AI Edit:  "I recommend launching the product next week."

[ ✓ Accept ] [ ✗ Reject ] [ ↻ Try Again ]
```

## 4. Context-Aware Generation

An AI writing tool is only as good as its context. When generating text, always feed the LLM the surrounding document.

```typescript
// server/api/ai-edit.post.ts
export default defineEventHandler(async (event) => {
  const { text, action, documentContext } = await readBody(event)
  
  const prompts = {
    professional: 'Rewrite this text to be professional and formal.',
    expand: 'Expand on this thought, adding more detail and context.',
    grammar: 'Fix all grammar and spelling errors. Do not change the meaning or tone.'
  }

  const { text: result } = await generateText({
    model: openai('gpt-4o'),
    system: `You are an expert editor. Complete the requested action on the target text.
    Output ONLY the revised text.
    
    Overall Document Context (for reference only):
    """
    ${documentContext}
    """`,
    prompt: `Action: ${prompts[action]}\n\nTarget Text: "${text}"`,
    maxTokens: 1000
  })

  return { text: result }
})
```

## 5. Brand Voice & Tone Settings

Provide users with a control panel to set the AI's default tone (e.g., "Friendly", "Direct", "Academic"). Store this preference in their settings and inject it into the system prompt.

```typescript
const systemPrompt = `
You are a writing assistant.
The user has configured your brand voice as: ${userSettings.brandVoice}
Tone: ${userSettings.tonePreference}
Avoid using these words: ${userSettings.bannedWords.join(', ')}
`
```
