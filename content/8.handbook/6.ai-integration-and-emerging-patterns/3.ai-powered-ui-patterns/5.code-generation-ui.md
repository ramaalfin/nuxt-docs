# Code Generation UI

Building UIs for AI code generation requires handling complex formatting, syntax highlighting, diff viewing, and one-click actions (like "Copy" or "Apply to file"). Code is structurally different from conversational text and needs specialized presentation.

## 1. Code Block Architecture

A good code generation block needs:
1. Language detection and syntax highlighting
2. Copy to clipboard button
3. "Apply/Insert" button (if integrated into an editor)
4. Streaming support (highlighting code as it streams in)

## 2. Syntax Highlighting with Shiki

Shiki is the modern standard for syntax highlighting. Unlike Prism or Highlight.js, it uses TextMate grammars (the same engine as VS Code) for perfectly accurate highlighting.

```bash
pnpm add shiki
```

```vue
<!-- components/CodeBlock.vue -->
<script setup lang="ts">
import { getHighlighter, type Highlighter } from 'shiki'

const props = defineProps<{
  code: string
  language: string
}>()

const highlightedHtml = ref('')
const isCopied = ref(false)
let highlighter: Highlighter | null = null

// Initialize Shiki (do this once globally in a real app)
onMounted(async () => {
  highlighter = await getHighlighter({
    themes: ['vitesse-dark', 'vitesse-light'],
    langs: ['javascript', 'typescript', 'vue', 'html', 'css', 'json', 'bash']
  })
  renderCode()
})

const renderCode = () => {
  if (!highlighter || !props.code) return
  highlightedHtml.value = highlighter.codeToHtml(props.code, {
    lang: props.language || 'text',
    theme: 'vitesse-dark'
  })
}

// Re-render when code streams in (debounced for performance)
watch(() => props.code, useDebounceFn(renderCode, 100))

const copyToClipboard = async () => {
  await navigator.clipboard.writeText(props.code)
  isCopied.value = true
  setTimeout(() => isCopied.value = false, 2000)
}
</script>

<template>
  <div class="code-block-container">
    <div class="code-header">
      <span class="language-badge">{{ language }}</span>
      <button class="copy-button" @click="copyToClipboard">
        <CheckIcon v-if="isCopied" class="text-green-500" />
        <CopyIcon v-else />
        {{ isCopied ? 'Copied!' : 'Copy' }}
      </button>
    </div>
    
    <!-- Rendered syntax highlighting -->
    <div class="code-content" v-html="highlightedHtml" />
  </div>
</template>

<style scoped>
.code-block-container {
  border-radius: 8px;
  overflow: hidden;
  background: #121212;
  margin: 1rem 0;
}
.code-header {
  display: flex;
  justify-content: space-between;
  padding: 8px 16px;
  background: #222;
  color: #888;
  font-size: 0.85rem;
}
.code-content { padding: 16px; overflow-x: auto; }
</style>
```

## 3. Streaming Code Blocks

When an LLM streams markdown containing code blocks, you need a robust parser that can identify partial code blocks and extract the language.

```typescript
// utils/markdown-parser.ts
export const extractCodeBlocks = (markdown: string) => {
  const blocks = []
  const regex = /```([\w-]*)\n([\s\S]*?)(```|$)/g
  let match

  while ((match = regex.exec(markdown)) !== null) {
    blocks.push({
      language: match[1] || 'text',
      code: match[2].trim(),
      isComplete: !!match[3] // True if the closing backticks were found
    })
  }

  return blocks
}
```

## 4. Diff Views (Inline and Side-by-Side)

When the AI suggests changes to existing code, showing a diff is crucial.

```vue
<!-- components/CodeDiff.vue -->
<script setup lang="ts">
import * as Diff from 'diff'

const props = defineProps<{
  original: string
  suggested: string
}>()

const diffLines = computed(() => {
  const changes = Diff.diffLines(props.original, props.suggested)
  
  // Flatten changes into individual lines with types
  return changes.flatMap(change => {
    const lines = change.value.replace(/\n$/, '').split('\n')
    return lines.map(line => ({
      text: line,
      type: change.added ? 'added' : change.removed ? 'removed' : 'unchanged'
    }))
  })
})
</script>

<template>
  <div class="diff-viewer font-mono text-sm">
    <div 
      v-for="(line, idx) in diffLines" 
      :key="idx"
      class="diff-line flex"
      :class="{
        'bg-green-900/30 text-green-300': line.type === 'added',
        'bg-red-900/30 text-red-300 line-through opacity-70': line.type === 'removed'
      }"
    >
      <span class="w-6 text-right select-none opacity-50 mr-4">
        {{ line.type === 'added' ? '+' : line.type === 'removed' ? '-' : ' ' }}
      </span>
      <span class="whitespace-pre">{{ line.text }}</span>
    </div>
  </div>
</template>
```

## 5. IDE Integration Pattern ("Apply to File")

If you are building an AI tool for developers, "Copy" isn't enough. Provide one-click injection into their workspace.

```vue
<!-- Inside CodeBlock.vue -->
<script setup lang="ts">
const applyCode = async () => {
  // If running in a web IDE environment with filesystem access
  if (window.fs && currentActiveFile) {
    await window.fs.writeFile(currentActiveFile.path, props.code)
    toast.success('Code applied to file')
  }
}
</script>
```

## 6. Prompt Engineering for Code Generation

Always enforce strict code generation rules in your system prompt to ensure parsable outputs.

```typescript
const CODE_GEN_SYSTEM_PROMPT = `
You are an expert software engineer.
When asked to write code, adhere to these rules:
1. Wrap all code in standard markdown code blocks with the correct language identifier.
2. Output ONLY the code block. Do NOT include explanatory text before or after the code.
3. Include all necessary imports.
4. Provide complete, working code (no "rest of code here" comments).
`
```
