# Batching Requests

Batching combines multiple AI requests into fewer API calls, reducing per-request overhead and enabling the use of batch-pricing APIs that cut costs by 50%. It is one of the highest-leverage cost and performance optimizations available for high-volume AI workloads.

## 1. When to Batch

| Scenario | Batchable? | Notes |
|---|---|---|
| Classifying 1,000 user messages | ✅ Yes | Perfect for batch |
| Generating product descriptions | ✅ Yes | Async, high volume |
| Real-time chat response | ❌ No | Latency-sensitive |
| Auto-complete suggestions | ❌ No | User waits for response |
| Nightly data enrichment | ✅ Yes | Ideal for OpenAI Batch API |
| Embedding large document sets | ✅ Yes | `embedMany` is built for this |

## 2. Embedding Batching with `embedMany`

Generating embeddings for many documents individually wastes HTTP round trips. Use `embedMany` to batch them.

```typescript
// ❌ SLOW: Sequential individual embeddings (N round trips)
const embeddings = []
for (const doc of documents) {
  const { embedding } = await embed({ model, value: doc.content })
  embeddings.push(embedding)
}

// ✅ FAST: Batch all at once (1 round trip, or a few for large datasets)
import { embedMany } from 'ai'

const BATCH_SIZE = 100 // OpenAI supports up to 2048 per batch

const batchEmbed = async (texts: string[]): Promise<number[][]> => {
  const allEmbeddings: number[][] = []
  
  for (let i = 0; i < texts.length; i += BATCH_SIZE) {
    const batch = texts.slice(i, i + BATCH_SIZE)
    const { embeddings } = await embedMany({
      model: openai.embedding('text-embedding-3-small'),
      values: batch
    })
    allEmbeddings.push(...embeddings)
  }
  
  return allEmbeddings
}
```

## 3. OpenAI Batch API (50% Cost Discount)

OpenAI's Batch API processes large jobs asynchronously within 24 hours at half price. Perfect for nightly jobs, bulk content generation, and large-scale analysis.

```typescript
// server/jobs/nightly-enrichment.ts
import OpenAI from 'openai'
import fs from 'fs/promises'

const openai = new OpenAI()

export const runNightlyEnrichment = async () => {
  // 1. Fetch items that need enrichment
  const products = await db.products
    .where('aiDescription', null)
    .limit(5000)

  // 2. Build JSONL batch file
  const requests = products.map(product => ({
    custom_id: product.id,
    method: 'POST',
    url: '/v1/chat/completions',
    body: {
      model: 'gpt-4o',  // Use best model — 50% off!
      messages: [
        { role: 'system', content: 'Write a compelling 50-word product description.' },
        { role: 'user', content: `Product: ${product.name}\nCategory: ${product.category}\nFeatures: ${product.features}` }
      ],
      max_tokens: 120
    }
  }))

  const jsonl = requests.map(r => JSON.stringify(r)).join('\n')
  
  // 3. Upload batch file
  const file = await openai.files.create({
    file: new File([jsonl], 'batch.jsonl', { type: 'application/jsonl' }),
    purpose: 'batch'
  })

  // 4. Submit batch job
  const batch = await openai.batches.create({
    input_file_id: file.id,
    endpoint: '/v1/chat/completions',
    completion_window: '24h'
  })

  // 5. Store batch ID for later retrieval
  await db.batchJobs.create({ batchId: batch.id, status: 'submitted', submittedAt: new Date() })
  
  console.log(`[Batch] Submitted ${requests.length} requests. Batch ID: ${batch.id}`)
  return batch.id
}

// 6. Retrieve results (run next morning)
export const processBatchResults = async (batchId: string) => {
  const batch = await openai.batches.retrieve(batchId)
  
  if (batch.status !== 'completed') {
    console.log(`[Batch] Status: ${batch.status}`)
    return
  }

  const fileContent = await openai.files.content(batch.output_file_id!)
  const lines = (await fileContent.text()).split('\n').filter(Boolean)
  
  for (const line of lines) {
    const result = JSON.parse(line)
    if (result.error) continue
    
    const description = result.response.body.choices[0].message.content
    await db.products.update(result.custom_id, { aiDescription: description })
  }
}
```

## 4. Client-Side Request Batching (DataLoader Pattern)

Batch multiple individual component-level requests into a single API call.

```typescript
// utils/request-batcher.ts
class RequestBatcher<K, V> {
  private pending: Map<K, Array<(v: V) => void>> = new Map()
  private timer: ReturnType<typeof setTimeout> | null = null

  constructor(
    private batchFn: (keys: K[]) => Promise<Map<K, V>>,
    private delayMs = 10
  ) {}

  load(key: K): Promise<V> {
    return new Promise((resolve) => {
      if (!this.pending.has(key)) this.pending.set(key, [])
      this.pending.get(key)!.push(resolve)

      if (!this.timer) {
        this.timer = setTimeout(async () => {
          const keys = [...this.pending.keys()]
          const resolvers = new Map(this.pending)
          this.pending.clear()
          this.timer = null

          const results = await this.batchFn(keys)
          keys.forEach(key => resolvers.get(key)?.forEach(r => r(results.get(key)!)))
        }, this.delayMs)
      }
    })
  }
}

// Usage: batch sentiment analysis across many components
const sentimentBatcher = new RequestBatcher<string, string>(async (texts) => {
  const results = await $fetch('/api/batch-sentiment', { method: 'POST', body: { texts } })
  return new Map(Object.entries(results))
})

// Each component calls this — they're automatically batched
export const getSentiment = (text: string) => sentimentBatcher.load(text)
```
