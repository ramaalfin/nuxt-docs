# Model Selection (Cost vs Quality)

Choosing the right foundation model for a specific task is the most critical decision in optimizing your AI integration. Using a frontier model (like GPT-4o or Claude 3.5 Sonnet) for every task is financial negligence. A production system routes requests dynamically to the cheapest model capable of completing the task.

## 1. The Model Tiers (as of late 2024 / early 2025)

| Tier | Models | Cost per 1M (In/Out) | Best For |
|---|---|---|---|
| **Frontier (Highest Quality)** | GPT-4o, Claude 3.5 Sonnet | ~$3.00 / ~$15.00 | Complex reasoning, coding, long context synthesis, formatting adherence |
| **Reasoning (Deep Logic)** | OpenAI o1, o3 | ~$15.00 / ~$60.00 | Advanced math, architectural design, complex bug fixing |
| **Fast / Efficient** | GPT-4o-mini, Claude 3 Haiku | ~$0.15 / ~$0.60 | Routing, classification, summarization, simple chat, auto-complete |
| **Open Weight** | LLaMA 3.1 8B, Mistral | Free (compute only) | Edge inference, extreme cost sensitivity, privacy |

**Rule of thumb:** The efficient tier (`gpt-4o-mini`) is **100x cheaper** than the frontier tier. Start all tasks on the efficient tier. Only upgrade to the frontier tier if the efficient model fails.

## 2. Dynamic Model Routing

Build a router that selects models based on the task's complexity, urgency, and the user's subscription tier.

```typescript
// server/utils/model-router.ts
import { openai } from '@ai-sdk/openai'
import { anthropic } from '@ai-sdk/anthropic'
import type { LanguageModel } from 'ai'

interface RoutingContext {
  taskComplexity: 'low' | 'medium' | 'high'
  isAsync: boolean
  userTier: 'free' | 'pro' | 'enterprise'
}

export const routeModel = (context: RoutingContext): LanguageModel => {
  const { taskComplexity, isAsync, userTier } = context

  // 1. Business Logic Rules
  // Free users always get the efficient model to control costs
  if (userTier === 'free') {
    return openai('gpt-4o-mini')
  }

  // 2. Task Complexity Rules
  if (taskComplexity === 'low') {
    return openai('gpt-4o-mini')
  }

  // 3. Modality Rules
  if (taskComplexity === 'high') {
    // Claude 3.5 Sonnet is currently the gold standard for complex coding tasks
    return anthropic('claude-3-5-sonnet-20241022')
  }

  // Default balanced choice
  return openai('gpt-4o')
}
```

## 3. LLM Router Pattern (AI judging AI)

Use a cheap, fast model to evaluate the user's prompt and decide which expensive model to route it to.

```typescript
// Use a cheap model to classify the intent
export const intelligentRoute = async (prompt: string): Promise<LanguageModel> => {
  const { object } = await generateObject({
    model: openai('gpt-4o-mini'), // Costs ~$0.0001 per check
    schema: z.object({
      complexity: z.enum(['simple', 'complex', 'coding']),
    }),
    prompt: `Analyze this request: "${prompt}". Is it a simple question, complex reasoning, or coding?`
  })

  switch (object.complexity) {
    case 'simple': return openai('gpt-4o-mini')
    case 'coding': return anthropic('claude-3-5-sonnet-20241022')
    case 'complex': return openai('gpt-4o')
  }
}
```

## 4. Fallback Routing (Resilience)

Cloud providers go down. Your application shouldn't. If OpenAI fails, instantly fallback to Anthropic.

```typescript
import { streamText } from 'ai'

export const resilientStreamText = async (messages: any[]) => {
  try {
    // Try primary provider
    return await streamText({
      model: openai('gpt-4o'),
      messages,
      timeout: 5000 // 5s timeout
    })
  } catch (error) {
    console.error('Primary LLM failed, falling back to secondary:', error)
    
    // Seamless fallback to secondary provider
    return await streamText({
      model: anthropic('claude-3-5-sonnet-20241022'),
      messages
    })
  }
}
```

## 5. Cost Auditing by Task

Always tag your LLM requests with the feature name. This allows you to audit which features are burning cash.

```typescript

    feature: featureName,
    inputTokens: usage.promptTokens,
```

## 6. The "Bring Your Own Key" (BYOK) Pattern

For enterprise B2B apps or heavy power users, allow them to provide their own OpenAI/Anthropic API keys. This shifts the compute cost entirely off your balance sheet.

```typescript
// server/utils/llm.ts
export const getModelForUser = async (userId: string) => {
  // Check if user provided their own key
  const userSettings = await db.userSettings.find(userId)
  
  if (userSettings?.customOpenAiKey) {
    // Use their key, they pay for it
    const customOpenAI = createOpenAI({ apiKey: userSettings.customOpenAiKey })
    return customOpenAI('gpt-4o')
  }
  
  // Use our key, we pay for it (subject to rate limits)
  return openai('gpt-4o-mini') 
}
```
