# Streaming Response Optimization

Streaming is the foundation of a responsive AI UX. But naive streaming implementations waste bandwidth, create visual jitter, and make slow models feel even slower. This chapter covers optimizations that make streaming feel fast, smooth, and professional.

## 1. Time-to-First-Token (TTFT) Optimization

TTFT — the delay between submitting a request and seeing the first character — is the most important latency metric for streaming UX. Users perceive waiting as "slowness" even if the total generation time is the same.

```typescript
// Optimize TTFT by reducing processing before streaming starts
export default defineEventHandler(async (event) => {
  const { messages } = await readBody(event)

  // ✅ DO: Start streaming immediately without blocking preprocessing
  const result = await streamText({
    model: openai('gpt-4o'),
    messages,
    
    // ✅ Keep system prompt short (fewer input tokens = faster first token)
    system: 'Helpful assistant. Be concise.',
    
    // ✅ Set a reasonable maxTokens (prevents generating forever)
    maxTokens: 1000,
    
    // ✅ Use temperature that doesn't require excessive sampling
    temperature: 0.7,
  })

  // Return stream immediately — don't await the full response
  return result.toDataStreamResponse()
})
```

**TTFT reduction checklist:**
- Shorten system prompts (fewer input tokens → faster processing)
- Use `gpt-4o-mini` for simple tasks (~2x faster TTFT vs `gpt-4o`)
- Deploy server close to users (use Edge Runtime on Vercel/Cloudflare)
- Avoid `await`-ing expensive operations before starting the stream

## 2. Smooth Token Rendering

Raw token streaming can cause visual jitter if each token triggers a full re-render. Buffer tokens and render in batches.

```typescript
// composables/useSmoothedStream.ts
export function useSmoothedStream(targetFPS = 60) {
  const displayedText = ref('')
  let buffer = ''
  let animationFrame: number | null = null

  const flushBuffer = () => {
    if (buffer) {
      // Add next chunk from buffer each frame
      const chunkSize = Math.max(1, Math.floor(buffer.length / 3))
      displayedText.value += buffer.slice(0, chunkSize)
      buffer = buffer.slice(chunkSize)
    }
    if (buffer) animationFrame = requestAnimationFrame(flushBuffer)
    else animationFrame = null
  }

  const addTokens = (tokens: string) => {
    buffer += tokens
    if (!animationFrame) animationFrame = requestAnimationFrame(flushBuffer)
  }

  const reset = () => {
    buffer = ''
    displayedText.value = ''
    if (animationFrame) cancelAnimationFrame(animationFrame)
  }

  return { displayedText, addTokens, reset }
}
```

## 3. Progressive Markdown Rendering

Partially-formed Markdown during streaming can render broken UI (an unclosed `**` renders as raw asterisks). Use streaming-aware markdown rendering.

```typescript
// utils/streaming-markdown.ts
// Only render complete markdown blocks — hold partial blocks in buffer
export const streamingMarkdown = (raw: string): string => {
  // Don't render if we're in the middle of a code block
  const codeBlockCount = (raw.match(/```/g) ?? []).length
  if (codeBlockCount % 2 !== 0) {
    // Incomplete code block — render up to the last complete block
    const lastComplete = raw.lastIndexOf('\n```\n')
    return lastComplete > -1
      ? marked.parse(raw.slice(0, lastComplete + 5))
      : DOMPurify.sanitize(raw) // Fallback: render as plain text
  }
  return DOMPurify.sanitize(marked.parse(raw) as string)
}
```

## 4. Streaming with Abort Control

Allow users to stop generation mid-stream and handle cleanup properly.

```typescript
// The AI SDK's useChat handles abort automatically via the `stop` function
const { stop, isLoading } = useChat({ api: '/api/chat' })

// Server-side: respect abort signals
export default defineEventHandler(async (event) => {
  const abortController = new AbortController()
  
  // Clean up when client disconnects
  event.node.req.on('close', () => abortController.abort())
  
  const result = await streamText({
    model: getModel(),
    messages,
    abortSignal: abortController.signal,
  })

  return result.toDataStreamResponse()
})
```

## 5. Streaming Performance Metrics

```typescript
// Track streaming performance for monitoring
const streamMetrics = {
  startTime: 0,
  firstTokenTime: 0,
  tokenCount: 0,

  start() { this.startTime = performance.now() },
  
  onFirstToken() {
    this.firstTokenTime = performance.now()
    // Track TTFT
    analytics.track('llm_ttft', { ms: this.firstTokenTime - this.startTime })
  },

  onToken() { this.tokenCount++ },

  onComplete() {
    const totalMs = performance.now() - this.startTime
    analytics.track('llm_generation_complete', {
      totalMs,
      tokensPerSecond: this.tokenCount / (totalMs / 1000),
      ttft: this.firstTokenTime - this.startTime
    })
  }
}
```
