# Token Counting & Efficiency

Every token costs money and adds latency. Token counting and optimization is not a premature optimization — in production AI applications, small inefficiencies compound across millions of requests into significant costs. This chapter provides practical techniques to maximize token efficiency.

## 1. Counting Tokens Before Sending

Count tokens client-side or server-side to validate inputs and prevent oversized requests.

```bash
pnpm add gpt-tokenizer  # For GPT models (tiktoken equivalent)
```

```typescript
// utils/token-counter.ts
import { encode, decode } from 'gpt-tokenizer'

export const countTokens = (text: string): number => encode(text).length

export const countMessageTokens = (messages: Array<{ role: string; content: string }>): number => {
  // GPT-4 token overhead: 3 tokens per message, 1 token for reply primer
  const messageOverhead = messages.length * 3 + 1
  const contentTokens = messages.reduce((sum, m) => sum + countTokens(m.content), 0)
  return messageOverhead + contentTokens
}

// Server-side: reject requests that are too large
export default defineEventHandler(async (event) => {
  const { messages } = await readBody(event)
  
  const totalTokens = countMessageTokens(messages)
  const MAX_INPUT_TOKENS = 8000
  
  if (totalTokens > MAX_INPUT_TOKENS) {
    throw createError({
      statusCode: 400,
      statusMessage: `Request too large: ${totalTokens} tokens exceeds limit of ${MAX_INPUT_TOKENS}`
    })
  }
  
  // Continue with request...
})
```

## 2. System Prompt Efficiency

System prompts are sent with every request. A 200-token reduction multiplied by 1 million requests per month = 200M fewer tokens charged.

```typescript
// ❌ VERBOSE system prompt (387 tokens)
const verbosePrompt = `
You are an extremely helpful, friendly, and professional customer service representative 
working for our company. You should always be very polite and courteous in your responses.
Please make sure to answer all customer questions thoroughly and completely.
If you don't know the answer to something, please make sure to let the customer know 
that you don't have that information and direct them to contact us via email or phone.
Our support email is support@example.com and our phone number is 1-800-EXAMPLE.
Please always maintain a professional tone throughout all of your interactions.
`

// ✅ OPTIMIZED system prompt (87 tokens — 78% reduction, same behavior)
const optimizedPrompt = `Customer service rep for Example Corp.
Rules: Be concise, professional, helpful.
If unknown: "Contact support@example.com or 1-800-EXAMPLE."
`

// Savings: 300 tokens × $0.000005/token × 1M requests/month = $1,500/month saved
```

## 3. Response Length Control

Output tokens are typically 3-5x more expensive than input tokens. Enforce response length constraints.

```typescript
// Force concise responses
const compactChat = await streamText({
```

## 4. Token Budget Manager

```typescript
// composables/useTokenBudget.ts
export function useTokenBudget(maxTokensPerSession = 10_000) {
  const tokensUsed = ref(0)
  const isNearLimit = computed(() => tokensUsed.value > maxTokensPerSession * 0.8)
  const isAtLimit = computed(() => tokensUsed.value >= maxTokensPerSession)
  const remaining = computed(() => maxTokensPerSession - tokensUsed.value)

  const track = (inputTokens: number, outputTokens: number) => {
    tokensUsed.value += inputTokens + outputTokens
  }

  return { tokensUsed, isNearLimit, isAtLimit, remaining, track }
}
```

```vue
<template>
  <!-- Warn user when approaching token limit -->
  <div v-if="isNearLimit" class="token-warning">
    ⚠️ Approaching session limit. {{ remaining }} tokens remaining.
  </div>
  <div v-if="isAtLimit" class="token-exceeded">
    Session limit reached. Start a new conversation.
  </div>
</template>
```

## 5. Token Usage Dashboard

```typescript
// server/api/usage/daily.get.ts — For admin monitoring
export default defineEventHandler(async (event) => {
  const { startDate, endDate } = getQuery(event)
  
  const usage = await db.aiUsageLogs
    .where('createdAt', '>=', new Date(startDate as string))
    .where('createdAt', '<=', new Date(endDate as string))
    .groupBy('model')
    .select([
      'model',
      db.raw('SUM(input_tokens) as total_input'),
      db.raw('SUM(output_tokens) as total_output'),
      db.raw('SUM(cost_usd) as total_cost'),
      db.raw('COUNT(*) as request_count'),
      db.raw('AVG(input_tokens) as avg_input'),
      db.raw('AVG(output_tokens) as avg_output'),
    ])

  return usage
})
```
