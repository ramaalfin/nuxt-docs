# Chain of Thought Reasoning

Chain of Thought (CoT) prompting is a technique that instructs the LLM to "think out loud" by generating intermediate reasoning steps before producing a final answer. This dramatically improves accuracy on complex tasks involving multi-step reasoning, mathematics, logical deduction, and code analysis.

## 1. Why CoT Works

Without CoT, the model compresses multi-step reasoning into a single forward pass — prone to errors. With CoT, the model generates each reasoning step as output tokens, which then become input for subsequent steps in the same generation — effectively giving the model a "scratchpad" to work through complexity.

```
Without CoT:
User: "If a store buys 50 items at $8 each and sells them at $12 each, 
       but 5 items are damaged and unsellable, what's the profit?"
Model: "$160" ❌ (wrong — skipped the damaged items calculation)

With CoT:
Model: "Let me work through this step by step:
1. Cost: 50 × $8 = $400
2. Sellable items: 50 - 5 = 45
3. Revenue: 45 × $12 = $540
4. Profit: $540 - $400 = $140" ✅
```

## 2. Basic CoT Trigger Phrases

The simplest CoT technique is adding a reasoning instruction to your prompt.

```typescript
// Magic phrases that activate CoT reasoning
const COT_TRIGGERS = [
  "Let's think step by step.",
  "Think through this carefully before answering.",
  "Work through each step of your reasoning before giving a final answer.",
  "First, let's break down the problem..."
]

// Usage
const prompt = `
You are a senior software architect.

Analyze the following system design and identify potential bottlenecks.
Think through this carefully, considering each component before giving your assessment.

System Design:
${systemDescription}
`
```

## 3. Structured CoT with Scratchpad

For programmatic use, separate the reasoning from the final answer.

```typescript
import { generateObject } from 'ai'
import { z } from 'zod'

const CoTResponseSchema = z.object({
  reasoning: z.string().describe('Step-by-step thinking process'),
  answer: z.string().describe('Final concise answer'),
  confidence: z.number().min(0).max(1)
})

export const analyzeCodeWithCoT = async (code: string, question: string) => {
  const { object } = await generateObject({
    model: openai('gpt-4o'), // CoT benefits more from capable models
    schema: CoTResponseSchema,
    messages: [
      {
        role: 'system',
        content: `You are a code analysis expert. When answering questions about code:
1. First, trace through the code execution mentally in your "reasoning" field.
2. Identify any issues, patterns, or relevant behaviors.
3. Then provide your final answer concisely in the "answer" field.`
      },
      { role: 'user', content: `Code:\n\`\`\`\n${code}\n\`\`\`\n\nQuestion: ${question}` }
    ]
  })

  return object
}
```

## 4. ReAct Pattern (Reasoning + Acting)

ReAct interleaves CoT reasoning with tool calls — the model thinks, acts (calls a function), observes the result, and reasons again.

```typescript
import { generateText, tool } from 'ai'
import { z } from 'zod'

export const reactAgent = async (question: string) => {
  const { text, steps } = await generateText({
    model: openai('gpt-4o'),
    prompt: question,
    maxSteps: 5, // Allow up to 5 reasoning + action cycles
    tools: {
      searchDocs: tool({
        description: 'Search the documentation for relevant information',
        parameters: z.object({ query: z.string() }),
        execute: async ({ query }) => {
          return searchVectorDB(query) // Real tool call
        }
      }),
      calculateMetric: tool({
        description: 'Calculate a specific business metric',
        parameters: z.object({ metric: z.string(), period: z.string() }),
        execute: async ({ metric, period }) => {
          return fetchMetricFromDB(metric, period)
        }
      })
    }
  })

  // steps contains the full reasoning trace: thought → action → observation cycles
  return { answer: text, trace: steps }
}
```

## 5. o1 and "Reasoning Models"

OpenAI's `o1` and `o3` models have CoT built-in at the model level. They spend tokens "thinking" internally before producing a response — you pay for the thinking tokens but don't see them.

```typescript
// o1 is better for: math, complex code, multi-step logic
// GPT-4o with CoT prompting is better for: tasks where you want to see the reasoning
const model = taskComplexity === 'very-high'
  ? openai('o1-mini')          // Built-in deep reasoning
  : openai('gpt-4o')           // Use CoT prompting + cheaper

const result = await streamText({
  model,
  messages,
  // o1 requires these settings
  ...(model.modelId?.startsWith('o1') ? {
    temperature: 1,    // o1 only supports temperature=1
    maxTokens: 16_000  // o1 generates more tokens due to internal reasoning
  } : {})
})
```

## 6. When to Use CoT

| Task Type | CoT Benefit | Model Recommendation |
|---|---|---|
| Math / calculations | ✅ High | GPT-4o with CoT or o1 |
| Code debugging | ✅ High | GPT-4o with CoT |
| Logical reasoning | ✅ High | o1 or GPT-4o with CoT |
| Classification | ⚠️ Low | GPT-4o-mini (simpler) |
| Summarization | ⚠️ Low | GPT-4o-mini (simpler) |
| Creative writing | ❌ None | GPT-4o without CoT |
