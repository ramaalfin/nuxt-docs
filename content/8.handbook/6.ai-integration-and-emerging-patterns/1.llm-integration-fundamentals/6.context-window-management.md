# Context Window Management

The context window is the total amount of text (measured in tokens) that an LLM can process in a single request — including your system prompt, conversation history, retrieved documents, and the model's response. Managing the context window efficiently is crucial for both cost control and output quality.

## 1. Context Window Sizes (2024-2025)

| Model | Context Window | Practical Useful Limit |
|---|---|---|
| **GPT-4o** | 128,000 tokens | ~80,000 (performance degrades near limits) |
| **Claude 3.5 Sonnet** | 200,000 tokens | ~150,000 |
| **GPT-4o-mini** | 128,000 tokens | ~80,000 |
| **LLaMA 3.1 8B** | 128,000 tokens | ~60,000 |

While large windows seem to solve all problems, research ("Lost in the Middle") shows that LLMs perform **worst on information in the middle** of the context — they're better at using info at the beginning and end.

## 2. Context Composition

For a typical chat application, the context budget is divided:

```
System Prompt:      ~500 tokens  (fixed per request)
Conversation History: ~4,000 tokens (managed)
RAG Documents:      ~3,000 tokens (when applicable)
User Message:         ~200 tokens  (variable)
Reserved for Response: 1,000 tokens (maxTokens)
─────────────────────────────────────────────────
Total:              ~8,700 tokens (well within limits)
```

## 3. Conversation History Strategies

### Strategy 1: Fixed Window (Sliding Window)
Keep only the N most recent messages.

```typescript
// utils/context.ts
export const slidingWindow = (messages: Message[], maxMessages = 10): Message[] => {
  // Always keep the system message (index 0) if present
  const systemMessages = messages.filter(m => m.role === 'system')
  const conversationMessages = messages.filter(m => m.role !== 'system')
  
  return [
    ...systemMessages,
    ...conversationMessages.slice(-maxMessages)
  ]
}
```

### Strategy 2: Token-Aware Trimming
Trim messages based on actual token count, not message count.

```typescript
// utils/context.ts
import { encode } from 'gpt-tokenizer'

export const trimToTokenBudget = (
  messages: Message[],
  maxTokens = 6000
): Message[] => {
  const systemMsgs = messages.filter(m => m.role === 'system')
  const convoMsgs = messages.filter(m => m.role !== 'system')
  
  let budget = maxTokens - systemMsgs.reduce((sum, m) => sum + encode(m.content).length, 0)
  const trimmed: Message[] = []
  
  // Walk backwards — keep most recent messages
  for (const msg of [...convoMsgs].reverse()) {
    const tokens = encode(msg.content).length + 4
    if (budget - tokens < 0) break
    trimmed.unshift(msg)
    budget -= tokens
  }
  
  return [...systemMsgs, ...trimmed]
}
```

### Strategy 3: Summarization
When conversation history grows too long, summarize older messages.

```typescript
// server/api/summarize-history.ts
export const summarizeOldMessages = async (messages: Message[]): Promise<Message[]> => {
  const KEEP_RECENT = 6 // Keep last 6 messages verbatim
  
  if (messages.length <= KEEP_RECENT) return messages

  const toSummarize = messages.slice(0, -KEEP_RECENT)
  const toKeep = messages.slice(-KEEP_RECENT)
  
  const { text: summary } = await generateText({
    model: openai('gpt-4o-mini'), // Use cheap model for summarization
    messages: [
      { role: 'system', content: 'Summarize the following conversation in 3-5 sentences, preserving key decisions, facts, and context.' },
      { role: 'user', content: toSummarize.map(m => `${m.role}: ${m.content}`).join('\n') }
    ],
    maxTokens: 300
  })
  
  return [
    { role: 'system', content: `[Earlier conversation summary]: ${summary}` },
    ...toKeep
  ]
}
```

## 4. Context Injection Order

The order of information in the context affects quality. Follow this pattern:

```typescript
const buildContext = (params: {
  systemPrompt: string
  retrievedDocuments?: string[]
  conversationHistory: Message[]
  userMessage: string
}): Message[] => [
  // 1. System instructions (always first)
  { role: 'system', content: params.systemPrompt },
  
  // 2. Retrieved documents (before history — high relevance)
  ...(params.retrievedDocuments?.length ? [{
    role: 'system' as const,
    content: `Relevant context:\n${params.retrievedDocuments.join('\n\n')}`
  }] : []),
  
  // 3. Conversation history (chronological)
  ...params.conversationHistory,
  
  // 4. Current user message (always last)
  { role: 'user', content: params.userMessage }
]
```

## 5. Monitoring Context Size

```typescript
// Add context size logging to every AI request
export default defineEventHandler(async (event) => {
  const { messages } = await readBody(event)
  
  const totalTokens = messages.reduce((sum: number, m: Message) =>
    sum + encode(m.content).length + 4, 0
  )
  
  if (totalTokens > 100_000) {
    console.warn(`[LLM] Large context: ${totalTokens} tokens`)
  }
  
  const result = await streamText({ model: getModel(), messages })
  return result.toDataStreamResponse()
})
```
