# Prompt Engineering Basics

Prompt engineering is the practice of crafting LLM inputs to reliably produce high-quality, predictable outputs. A well-engineered prompt is often more impactful than switching to a more expensive model. It is a core skill for any developer integrating LLMs into production applications.

## 1. The Anatomy of a Good Prompt

A production prompt has four components:

```
1. ROLE     → Who the model is (persona, expertise)
2. CONTEXT  → Background information relevant to the task
3. TASK     → The specific thing you want the model to do
4. FORMAT   → How you want the output structured
```

```typescript
const systemPrompt = `
# Role
You are a senior code reviewer specializing in TypeScript and Vue 3.

# Context
You are reviewing code submitted as part of a pull request for a production Nuxt 3 application.
The codebase follows the Composition API and uses Pinia for state management.

# Task
Review the provided code snippet and identify:
1. Security vulnerabilities
2. Performance issues
3. Violations of Vue best practices
4. TypeScript type safety issues

# Format
Respond in JSON with this exact structure:
{
  "severity": "critical" | "major" | "minor" | "none",
  "issues": [{ "line": number, "category": string, "description": string, "suggestion": string }],
  "summary": string
}
`
```

## 2. System Prompts vs User Messages

| Message Type | Purpose | Best Practices |
|---|---|---|
| `system` | Permanent instructions, persona, format rules | Keep concise — costs tokens on every request |
| `user` | The actual request or question | Include all context needed for the specific task |
| `assistant` | Prefill to steer output | Used for few-shot examples |

```typescript
const messages = [
  { role: 'system', content: 'You are a helpful coding assistant. Respond in markdown.' },
  { role: 'user', content: 'Explain what a Vue composable is.' }
]
```

## 3. Be Explicit, Not Implicit

LLMs follow instructions literally. Vague prompts produce vague outputs.

```typescript
// ❌ VAGUE: "Summarize this article"
// LLM doesn't know: Length? Audience? Format? Tone?

// ✅ EXPLICIT:
const prompt = `
Summarize the following article in exactly 3 bullet points.
Each bullet point must be under 20 words.
Target audience: non-technical business executives.
Tone: professional and neutral.

Article:
${article}
`
```

## 4. Output Format Control

For programmatic use, enforce structured output (JSON) to make responses parseable.

```typescript
// With Vercel AI SDK — structured output with Zod validation
import { generateObject } from 'ai'
import { z } from 'zod'

const SentimentSchema = z.object({
  sentiment: z.enum(['positive', 'neutral', 'negative']),
  confidence: z.number().min(0).max(1),
  keyPhrases: z.array(z.string()).max(5),
  summary: z.string().max(100)
})

export default defineEventHandler(async (event) => {
  const { text } = await readBody(event)

  const { object } = await generateObject({
    model: openai('gpt-4o-mini'),
    schema: SentimentSchema,
    prompt: `Analyze the sentiment of the following customer review:\n\n${text}`
  })

  // object is fully typed as { sentiment: 'positive'|'neutral'|'negative', ... }
  return object
})
```

## 5. Persona & Tone Instructions

```typescript
// Consistent persona across all app interactions
const ASSISTANT_PERSONA = `
You are Aria, a friendly and knowledgeable assistant for Acme HR Platform.

Personality:
- Warm but professional
- Concise — prefer short answers unless asked for details
- Honest about limitations — say "I don't know" rather than guessing

Boundaries:
- Only answer questions about HR processes, company policies, and benefits
- Never provide legal, medical, or financial advice
- For anything outside your scope, direct users to: support@acme.com
`
```

## 6. Common Anti-Patterns

| Anti-Pattern | Problem | Fix |
|---|---|---|
| `"Don't say..."` | LLMs struggle with negations | Use positive instructions: `"Always respond in English"` |
| Overloaded prompts | Too many instructions cause random skipping | Break into multiple focused prompts |
| No output constraints | Response length varies wildly | Specify `"Respond in under 100 words"` |
| Missing context | Model hallucinates to fill gaps | Provide all relevant context explicitly |
| `"Be creative"` | Too open-ended for programmatic use | Specify constraints: `"Generate 5 different options, each under 50 chars"` |
