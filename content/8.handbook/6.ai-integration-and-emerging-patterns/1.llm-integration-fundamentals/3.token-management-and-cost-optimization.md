# Token Management & Cost Optimization

LLM costs are directly proportional to token usage. A single production feature left unoptimized can generate thousands of dollars in API bills. Understanding token management and implementing cost controls is not optional—it is a fundamental engineering responsibility.

## 1. Understanding Tokens

Tokens are the fundamental unit of LLM processing. A token is approximately 4 characters of English text, or ¾ of a word.

```
"Hello, world!" = 4 tokens
"The quick brown fox jumps over the lazy dog" = 9 tokens
A 1,000-word essay ≈ 1,300 tokens
GPT-4o: $5 per 1M input tokens, $15 per 1M output tokens
Claude 3.5 Sonnet: $3 per 1M input, $15 per 1M output
```

**Key insight:** Input tokens (your prompt + context) and output tokens (model response) are priced differently. Output tokens are typically 3–5x more expensive.

## 2. Tracking Token Usage

Always track token usage per request in production.

```typescript
// server/api/chat.post.ts
import { streamText } from 'ai'
import { getModel } from '../utils/llm'

export default defineEventHandler(async (event) => {
  const { messages } = await readBody(event)
  const userId = event.context.userId

  const result = await streamText({
    model: getModel(),
    messages,
    maxTokens: 1000, // Hard cap on output tokens

    onFinish: async ({ usage, finishReason }) => {
      // Log every request for cost monitoring
      await db.aiUsageLogs.create({
        userId,
        model: process.env.LLM_MODEL,
        inputTokens: usage.promptTokens,
        outputTokens: usage.completionTokens,
        totalTokens: usage.totalTokens,
        cost: calculateCost(usage, process.env.LLM_MODEL!),
        finishReason,
        timestamp: new Date()
      })
    }
  })

  return result.toDataStreamResponse()
})

function calculateCost(usage: { promptTokens: number; completionTokens: number }, model: string) {
  const pricing: Record<string, { input: number; output: number }> = {
    'gpt-4o':       { input: 0.000005, output: 0.000015 },  // Per token
    'gpt-4o-mini':  { input: 0.00000015, output: 0.0000006 },
    'claude-3-5-sonnet-20241022': { input: 0.000003, output: 0.000015 },
  }
  const p = pricing[model] ?? { input: 0, output: 0 }
  return (usage.promptTokens * p.input) + (usage.completionTokens * p.output)
}
```

## 3. Token Optimization Strategies

### 1. Use the Smallest Sufficient Model

```typescript
// Route different tasks to appropriately-sized models
const selectModel = (taskType: 'classification' | 'summary' | 'complex-reasoning') => {
  switch (taskType) {
    case 'classification': return openai('gpt-4o-mini')      // 100x cheaper
    case 'summary':        return openai('gpt-4o-mini')      // Sufficient quality
    case 'complex-reasoning': return openai('gpt-4o')        // Worth the cost
  }
}
```

### 2. Trim Conversation History

Sending the entire conversation history grows linearly with conversation length. Trim strategically.

```typescript
// utils/token-trimmer.ts
import { encode } from 'gpt-tokenizer'

const MAX_CONTEXT_TOKENS = 8000

export function trimMessages(messages: Message[], systemTokens: number): Message[] {
  const budgetTokens = MAX_CONTEXT_TOKENS - systemTokens - 500 // Reserve for response
  let usedTokens = 0
  const result: Message[] = []
  
  // Always keep the most recent messages, trim oldest first
  for (const msg of [...messages].reverse()) {
    const msgTokens = encode(msg.content).length + 4 // +4 per message overhead
    if (usedTokens + msgTokens > budgetTokens) break
    
    result.unshift(msg)
    usedTokens += msgTokens
  }
  
  return result
}
```

### 3. Cache Identical Requests

```typescript
// server/api/completion.post.ts
import { createHash } from 'crypto'

const cache = new Map<string, string>() // In production: use Redis

export default defineEventHandler(async (event) => {
  const { prompt, systemPrompt } = await readBody(event)
  
  // Hash the prompt for cache key
  const cacheKey = createHash('sha256')
    .update(`${systemPrompt}::${prompt}`)
    .digest('hex')
  
  if (cache.has(cacheKey)) {
    return { text: cache.get(cacheKey), cached: true }
  }
  
  const result = await generateText({ model: getModel(), prompt })
  cache.set(cacheKey, result.text)
  
  return { text: result.text, cached: false }
})
```

### 4. Compress System Prompts

Verbose system prompts waste tokens on every request. Every word in your system prompt costs money on every single API call.

```
❌ VERBOSE: 
"You are an extremely helpful, friendly, and professional customer service agent 
who works for our amazing company. Please be very polite and courteous at all times 
when responding to customer inquiries..."

✅ COMPRESSED:
"Customer service agent. Be concise, helpful, professional. Company: Acme Corp."
```

## 4. User-Level Rate Limiting & Spending Caps

```typescript
// server/middleware/ai-rate-limiter.ts
export default defineEventHandler(async (event) => {
  if (!event.path.startsWith('/api/ai/')) return

  const userId = event.context.userId
  if (!userId) throw createError({ statusCode: 401 })

  const today = new Date().toISOString().split('T')[0]
  const dailyUsage = await db.aiUsageLogs
    .where({ userId, date: today })
    .sum('totalTokens')

  const FREE_TIER_DAILY_LIMIT = 50_000 // tokens per day

  if (dailyUsage > FREE_TIER_DAILY_LIMIT) {
    throw createError({
      statusCode: 429,
      statusMessage: 'Daily AI usage limit reached. Upgrade for more.',
      data: { dailyUsage, limit: FREE_TIER_DAILY_LIMIT }
    })
  }
})
```

## 5. Cost Monitoring Dashboard

Set up budget alerts before you need them:

- **OpenAI:** Usage → Billing → Set usage limits at $X/day and $Y/month.
- **Anthropic:** Console → Settings → Billing limits.
- Alert at 50%, 80%, and 100% of your monthly budget.
- Set a hard limit that blocks API calls to prevent runaway costs.
