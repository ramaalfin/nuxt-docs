# Streaming Response Handling (SSE)

LLM APIs generate text token by token. Waiting for the full response before displaying anything creates a poor user experience—especially for long responses that take 5–20 seconds. **Streaming** delivers each token to the client as it is generated, making the response feel instant and interactive.

LLM streaming is built on **Server-Sent Events (SSE)**, the same technology used for real-time feeds.

## 1. The Streaming Architecture

```
Client                 Nuxt (Nitro)              OpenAI
  │                       │                        │
  │── POST /api/chat ────►│                        │
  │                       │── POST (streaming) ───►│
  │                       │                        │
  │◄── SSE stream ────────│◄── token by token ─────│
  │   "The" "quick" "brown"...                     │
  │                       │                        │
  │◄── [DONE] ────────────│◄── stream end ──────────│
```

## 2. Server: Streaming with Vercel AI SDK

```typescript
// server/api/chat.post.ts
import { openai } from '@ai-sdk/openai'
import { streamText } from 'ai'
import { z } from 'zod'

const ChatSchema = z.object({
  messages: z.array(z.object({
    role: z.enum(['user', 'assistant', 'system']),
    content: z.string().max(10000)
  })).max(50) // Limit history depth
})

export default defineEventHandler(async (event) => {
  // Validate input
  const body = await readValidatedBody(event, ChatSchema.parse)

  const result = await streamText({
    model: openai('gpt-4o'),
    messages: body.messages,
    system: 'You are a helpful assistant.',
    maxTokens: 2000,
    
    // Stream lifecycle callbacks
    onFinish: async ({ text, usage, finishReason }) => {
      // Log usage for cost tracking
      console.log(`Tokens used: ${usage.totalTokens}, Finish: ${finishReason}`)
      await logUsage({ userId: event.context.userId, tokens: usage.totalTokens })
    }
  })

  // Returns a Response with proper SSE headers
  return result.toDataStreamResponse()
})
```

## 3. Client: Consuming the Stream

### Using Vercel AI SDK's Vue Composable

```bash
pnpm add @ai-sdk/vue
```

```vue
<!-- components/ChatInterface.vue -->
<script setup lang="ts">
import { useChat } from '@ai-sdk/vue'

const { messages, input, handleSubmit, isLoading, error, stop } = useChat({
  api: '/api/chat',
  
  onError: (error) => {
    console.error('Chat error:', error)
    toast.error('Something went wrong. Please try again.')
  },
  
  onFinish: (message) => {
    // Called when the full response is received
    console.log('Response complete:', message.content)
  }
})
</script>

<template>
  <div class="chat-container">
    <!-- Message history -->
    <div class="messages">
      <div
        v-for="message in messages"
        :key="message.id"
        :class="['message', `message--${message.role}`]"
      >
        <!-- Render markdown for assistant messages -->
        <MarkdownRenderer v-if="message.role === 'assistant'" :content="message.content" />
        <p v-else>{{ message.content }}</p>
      </div>
      
      <!-- Streaming indicator -->
      <div v-if="isLoading" class="typing-indicator">
        <span /><span /><span />
      </div>
    </div>

    <!-- Input form -->
    <form @submit="handleSubmit">
      <input
        v-model="input"
        placeholder="Ask anything..."
        :disabled="isLoading"
      />
      <button v-if="isLoading" type="button" @click="stop">Stop</button>
      <button v-else type="submit" :disabled="!input.trim()">Send</button>
    </form>
  </div>
</template>
```

## 4. Manual Streaming (Without AI SDK)

When you need more control, consume the SSE stream manually using the Fetch API.

```typescript
// composables/useStreamingCompletion.ts
export function useStreamingCompletion() {
  const output = ref('')
  const isStreaming = ref(false)
  const controller = ref<AbortController | null>(null)

  const stream = async (prompt: string) => {
    output.value = ''
    isStreaming.value = true
    controller.value = new AbortController()

    const response = await fetch('/api/completion', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ prompt }),
      signal: controller.value.signal
    })

    if (!response.body) throw new Error('No response body')

    const reader = response.body.getReader()
    const decoder = new TextDecoder()

    try {
      while (true) {
        const { done, value } = await reader.read()
        if (done) break

        const chunk = decoder.decode(value, { stream: true })
        
        // Parse SSE data lines
        const lines = chunk.split('\n').filter(l => l.startsWith('data: '))
        for (const line of lines) {
          const data = line.replace('data: ', '').trim()
          if (data === '[DONE]') break
          
          try {
            const parsed = JSON.parse(data)
            output.value += parsed.text ?? ''
          } catch { /* Ignore malformed chunks */ }
        }
      }
    } finally {
      isStreaming.value = false
    }
  }

  const stop = () => {
    controller.value?.abort()
    isStreaming.value = false
  }

  return { output, isStreaming, stream, stop }
}
```

## 5. Displaying Streamed Markdown

LLM responses often contain Markdown. Render it progressively as tokens arrive.

```vue
<script setup lang="ts">
import { marked } from 'marked'
import DOMPurify from 'dompurify'

const props = defineProps<{ content: string }>()

const renderedHtml = computed(() =>
  DOMPurify.sanitize(marked.parse(props.content) as string)
)
</script>

<template>
  <!-- Safe: sanitized before rendering with v-html -->
  <div class="prose" v-html="renderedHtml" />
</template>
```
