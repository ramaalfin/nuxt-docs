# OpenAI API / Claude API / LLaMA

The LLM landscape has three dominant categories: proprietary cloud APIs (OpenAI, Anthropic Claude), open-weight models (LLaMA, Mistral), and provider-agnostic SDKs (Vercel AI SDK). Understanding their differences helps you choose the right tool and architect a flexible integration.

## 1. Provider Comparison

| Provider | Best Models | Strengths | Weaknesses |
|---|---|---|---|
| **OpenAI** | GPT-4o, o1, o3 | Best overall quality, richest ecosystem, function calling | Most expensive, US data residency |
| **Anthropic Claude** | Claude 3.5 Sonnet, Claude 3 Opus | Long context (200k), nuanced instruction-following, safer outputs | No fine-tuning, fewer integrations |
| **Google Gemini** | Gemini 1.5 Pro, Gemini Flash | Massive context window, multimodal | Inconsistent quality across tasks |
| **LLaMA (Meta)** | LLaMA 3.1, 3.2 | Open-weight, self-hostable, no API costs | Requires infrastructure, smaller ecosystem |
| **Mistral** | Mistral Large, Mixtral | Open-weight, EU-hosted option, competitive quality | Smaller community |

## 2. Vercel AI SDK (Recommended for Nuxt)

The **Vercel AI SDK** is the best way to integrate LLMs into Nuxt. It provides a unified API across all major providers, streaming support, and first-class Vue/Nuxt composables.

```bash
pnpm add ai @ai-sdk/openai @ai-sdk/anthropic
```

### OpenAI Integration

```typescript
// server/api/chat.post.ts
import { openai } from '@ai-sdk/openai'
import { streamText } from 'ai'

export default defineEventHandler(async (event) => {
  const { messages } = await readBody(event)

  const result = await streamText({
    model: openai('gpt-4o'),
    messages,
    system: 'You are a helpful assistant.',
    maxTokens: 2000,
    temperature: 0.7,
  })

  // Return a streaming response (ReadableStream)
  return result.toDataStreamResponse()
})
```

### Claude Integration (Drop-In Swap)

```typescript
// server/api/chat.post.ts
import { anthropic } from '@ai-sdk/anthropic'
import { streamText } from 'ai'

export default defineEventHandler(async (event) => {
  const { messages } = await readBody(event)

  // ✅ Same API — just swap the model
  const result = await streamText({
    model: anthropic('claude-3-5-sonnet-20241022'),
    messages,
    maxTokens: 4096,
  })

  return result.toDataStreamResponse()
})
```

## 3. Self-Hosted LLaMA with Ollama

Run LLaMA 3.1 locally for development or in your own infrastructure for production.

```bash
# Install Ollama
brew install ollama

# Pull and run LLaMA 3.1 8B
ollama pull llama3.1
ollama serve
```

```typescript
// server/api/chat.post.ts
import { createOpenAI } from '@ai-sdk/openai'

// Ollama exposes an OpenAI-compatible API
const ollama = createOpenAI({
  baseURL: 'http://localhost:11434/v1',
  apiKey: 'ollama', // Required but ignored by Ollama
})

export default defineEventHandler(async (event) => {
  const { messages } = await readBody(event)
  
  const result = await streamText({
    model: ollama('llama3.1:8b'),
    messages
  })

  return result.toDataStreamResponse()
})
```

## 4. Provider Abstraction Pattern

Design your integration so you can switch providers via environment variable—without code changes.

```typescript
// server/utils/llm.ts
import { openai } from '@ai-sdk/openai'
import { anthropic } from '@ai-sdk/anthropic'
import { createOpenAI } from '@ai-sdk/openai'
import type { LanguageModel } from 'ai'

const ollama = createOpenAI({ baseURL: 'http://localhost:11434/v1', apiKey: 'ollama' })

export function getModel(): LanguageModel {
  const provider = process.env.LLM_PROVIDER ?? 'openai'
  
  switch (provider) {
    case 'openai':    return openai(process.env.OPENAI_MODEL ?? 'gpt-4o-mini')
    case 'anthropic': return anthropic(process.env.ANTHROPIC_MODEL ?? 'claude-3-5-sonnet-20241022')
    case 'ollama':    return ollama(process.env.OLLAMA_MODEL ?? 'llama3.1:8b')
    default: throw new Error(`Unknown LLM provider: ${provider}`)
  }
}
```

```typescript
// server/api/chat.post.ts
import { streamText } from 'ai'
import { getModel } from '../utils/llm'

export default defineEventHandler(async (event) => {
  const { messages } = await readBody(event)
  const result = await streamText({ model: getModel(), messages })
  return result.toDataStreamResponse()
})
```

## 5. Authentication & API Key Management

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  runtimeConfig: {
    // Server-only — never exposed to client
    openaiApiKey: '',       // NUXT_OPENAI_API_KEY
    anthropicApiKey: '',    // NUXT_ANTHROPIC_API_KEY
    llmProvider: 'openai',  // NUXT_LLM_PROVIDER
  }
})
```

```bash
# .env (never commit this)
NUXT_OPENAI_API_KEY=sk-proj-...
NUXT_ANTHROPIC_API_KEY=sk-ant-...
NUXT_LLM_PROVIDER=openai
```

**Critical:** LLM API keys must NEVER be exposed to the client. All LLM calls must go through your Nitro server routes—the client never has direct API access.
