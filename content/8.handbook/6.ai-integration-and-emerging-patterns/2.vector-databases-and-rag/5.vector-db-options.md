# Vector DB Options

Choosing the right vector database is a foundational architectural decision. The wrong choice can mean expensive migrations later. This chapter compares the major options and provides a decision framework.

## 1. Comparison Table

| Database | Type | Scalability | Cost | Filtering | Best For |
|---|---|---|---|---|---|
| **Supabase pgvector** | PostgreSQL extension | Medium (100M vectors) | Cheap (existing DB) | ✅ Full SQL | Teams on Supabase/Postgres |
| **Pinecone** | Managed SaaS | Very High (billions) | $$$ | ✅ Metadata filters | Production at scale, fast setup |
| **Qdrant** | Self-hosted/Cloud | Very High | $ (self-hosted) | ✅ Complex filters | Cost-conscious, EU data |
| **Weaviate** | Self-hosted/Cloud | Very High | $$ | ✅ GraphQL | Multi-modal, hybrid built-in |
| **Chroma** | Self-hosted | Low-Medium | Free | ✅ Metadata | Local dev, prototyping |
| **Turbopuffer** | Managed SaaS | High | $$ | ✅ SQL-like | Cost-efficient at scale |
| **LanceDB** | Embedded/Cloud | Medium | Free (embedded) | ✅ SQL | Edge, serverless functions |

## 2. Supabase pgvector (Recommended Starting Point)

**Best for:** Teams already on Supabase, starting projects, budgets < $500/month.

```typescript
// No separate DB needed — just add the extension
// Full SQL filtering, JOINs with your existing data
const results = await supabase.rpc('match_documents', {
  query_embedding: embedding,
  match_threshold: 0.75,
  match_count: 5,
  // Native SQL filter: only search within a specific category
  filter_category: 'policies'
})
```

**Limitations:** Performance degrades above ~500k vectors on shared plans. Use dedicated Supabase instances or upgrade to Pinecone for millions of vectors.

## 3. Pinecone

**Best for:** Production apps needing > 1M vectors, fastest query speeds.

```typescript
import { Pinecone } from '@pinecone-database/pinecone'

const pc = new Pinecone({ apiKey: process.env.PINECONE_API_KEY! })

// Create index (serverless - pay per query)
await pc.createIndex({
  name: 'my-docs',
  dimension: 1536,        // Match OpenAI embedding dimensions
  metric: 'cosine',
  spec: { serverless: { cloud: 'aws', region: 'us-east-1' } }
})

const index = pc.index('my-docs')

// Upsert with rich metadata for filtering
await index.upsert([{
  id: 'doc-123',
  values: embedding,
  metadata: {
    content: 'The refund policy states...',
    category: 'policies',
    locale: 'en-US',
    version: 2
  }
}])

// Query with metadata filter
const results = await index.query({
  vector: queryEmbedding,
  topK: 5,
  filter: { category: { $eq: 'policies' }, locale: { $in: ['en-US', 'en-GB'] } },
  includeMetadata: true
})
```

## 4. Qdrant (Best Open-Source Option)

**Best for:** Teams that need self-hosting for data sovereignty, EU/GDPR compliance, or cost control.

```bash
# Local development with Docker
docker run -p 6333:6333 qdrant/qdrant
```

```typescript
import { QdrantClient } from '@qdrant/js-client-rest'

const client = new QdrantClient({ url: process.env.QDRANT_URL })

// Create collection
await client.createCollection('documents', {
  vectors: { size: 1536, distance: 'Cosine' }
})

// Upsert
await client.upsert('documents', {
  points: [{ id: 1, vector: embedding, payload: { content, category } }]
})

// Search with filter
const results = await client.search('documents', {
  vector: queryEmbedding,
  limit: 5,
  filter: { must: [{ key: 'category', match: { value: 'policies' } }] }
})
```

## 5. LanceDB (Embedded — Great for Serverless/Edge)

**Best for:** Nuxt apps deployed on Vercel/Cloudflare with small-to-medium vector datasets.

```typescript
import * as lancedb from '@lancedb/lancedb'

// No server needed — database is a directory on disk/S3
const db = await lancedb.connect('./data/lancedb')
const table = await db.openTable('documents')

const results = await table.search(queryEmbedding)
  .where("category = 'policies'")  // SQL-like filtering
  .limit(5)
  .toArray()
```

## 6. Decision Framework

```
Do you already use Supabase/PostgreSQL?
  └── YES → Use pgvector (zero extra cost, zero extra infra)

Do you need > 1M vectors or very low latency?
  └── YES → Use Pinecone (managed, instant scaling)

Do you need self-hosted for compliance/cost?
  └── YES → Use Qdrant (best open-source option)

Are you on a serverless/edge platform?
  └── YES → Use LanceDB (embedded, no server)

Are you prototyping locally?
  └── YES → Use Chroma (simplest setup, zero config)
```
