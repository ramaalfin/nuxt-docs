# Embeddings Basics (Conceptual)

Embeddings are numerical representations of text (or other data) as vectors of floating-point numbers. They are the mathematical foundation of semantic search, RAG (Retrieval Augmented Generation), and recommendation systems. Understanding embeddings conceptually is essential before implementing vector search.

## 1. What is an Embedding?

An embedding model converts text into a fixed-length vector — an array of numbers that captures the *semantic meaning* of the text.

```
"The cat sat on the mat"  → [0.12, -0.34, 0.78, ..., 0.45]  (1536 numbers)
"A feline rested on a rug" → [0.13, -0.31, 0.79, ..., 0.44]  (1536 numbers)
"JavaScript is a language" → [-0.82, 0.14, -0.23, ..., 0.91] (1536 numbers)
```

The key insight: **semantically similar texts produce numerically similar vectors**. The first two sentences (same meaning, different words) will be very close in vector space. The third sentence is semantically different and far away.

## 2. Measuring Similarity: Cosine Similarity

The standard similarity metric for embeddings is **cosine similarity** — the angle between two vectors, ranging from -1 (opposite meaning) to 1 (identical meaning).

```typescript
// utils/embeddings.ts
export function cosineSimilarity(a: number[], b: number[]): number {
  if (a.length !== b.length) throw new Error('Vectors must have the same dimension')
  
  const dotProduct = a.reduce((sum, ai, i) => sum + ai * b[i], 0)
  const magnitudeA = Math.sqrt(a.reduce((sum, ai) => sum + ai * ai, 0))
  const magnitudeB = Math.sqrt(b.reduce((sum, bi) => sum + bi * bi, 0))
  
  return dotProduct / (magnitudeA * magnitudeB)
}

// Example:
const catEmbedding = await embed('cat')
const feline​Embedding = await embed('feline')
const javascript​Embedding = await embed('javascript')

cosineSimilarity(catEmbedding, felineEmbedding)     // → 0.91 (very similar)
cosineSimilarity(catEmbedding, javascriptEmbedding) // → 0.12 (unrelated)
```

| Model | Dimensions | Best For | Cost |
|---|---|---|---|
| **OpenAI `text-embedding-3-small`** | 1536 | General purpose, fast | $0.02/1M tokens |
| **OpenAI `text-embedding-3-large`** | 3072 | Highest quality | $0.13/1M tokens |

## 4. Generating Embeddings in Nuxt

```typescript
// server/utils/embeddings.ts
import { openai } from '@ai-sdk/openai'
import { embed, embedMany } from 'ai'

// Single text embedding
export const embedText = async (text: string): Promise<number[]> => {
  const { embedding } = await embed({
    model: openai.embedding('text-embedding-3-small'),
    value: text
  })
  return embedding
}

// Batch embedding (more efficient for multiple texts)
export const embedTexts = async (texts: string[]): Promise<number[][]> => {
  const { embeddings } = await embedMany({
    model: openai.embedding('text-embedding-3-small'),
    values: texts
  })
  return embeddings
}
```

## 5. Key Properties of Embeddings

### Semantic Clustering
Words and concepts group together in vector space:
- "king", "queen", "prince" → close to each other
- "Paris", "London", "Tokyo" → close to each other
- Famous analogy: `king - man + woman ≈ queen`

### Dimensionality Reduction
Embeddings compress complex meaning into hundreds of numbers. You can reduce dimensions further (at the cost of precision) using PCA or UMAP for visualization.

### Language Agnostic (Multilingual Models)
Multilingual embedding models create a shared semantic space across languages:
- "Hello" (English) ≈ "Hola" (Spanish) ≈ "Bonjour" (French)

## 6. What Embeddings Are NOT

- **Not keyword matching:** "cat" and "feline" are similar despite sharing no letters.
- **Not always right:** Embeddings can fail on domain-specific jargon not seen during training.
- **Not free:** Embedding 1M tokens costs ~$0.02 with OpenAI's small model.
- **Not updatable:** The embedding of a text doesn't change when new context is added — you must re-embed.
