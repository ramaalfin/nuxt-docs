# Vector Search Implementation

Vector search finds documents that are *semantically similar* to a query, even when they share no keywords. It is the core technology behind modern search engines, recommendation systems, and RAG pipelines. This chapter covers the full implementation from embedding storage to approximate nearest-neighbor (ANN) search.

## 1. How Vector Search Works

```
1. INDEXING (offline):
   Document text → Embedding model → Vector → Stored in Vector DB

2. SEARCHING (online):
   Query text → Embedding model → Query vector
   Query vector → ANN search against indexed vectors → Top-K results
   Results → Return original documents
```

## 3. Implementation with Supabase pgvector

pgvector is the most developer-friendly option for teams already using Supabase/PostgreSQL.

### Database Setup

```sql
-- Enable the pgvector extension
CREATE EXTENSION IF NOT EXISTS vector;

-- Create documents table with vector column
CREATE TABLE documents (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  content     TEXT NOT NULL,
  metadata    JSONB DEFAULT '{}',
  embedding   VECTOR(1536),     -- Match your embedding model's dimensions
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- Create an IVFFlat index for fast ANN search
-- lists = sqrt(number of rows) is a good starting point
CREATE INDEX ON documents USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);
```

### Indexing Documents

```typescript
// server/api/documents/index.post.ts
import { createClient } from '@supabase/supabase-js'
import { embedText } from '../utils/embeddings'

const supabase = createClient(process.env.SUPABASE_URL!, process.env.SUPABASE_SERVICE_KEY!)

export default defineEventHandler(async (event) => {
  const { content, metadata } = await readBody(event)

  // Generate embedding for the document
  const embedding = await embedText(content)

  // Store in Supabase with the vector
  const { data, error } = await supabase
    .from('documents')
    .insert({ content, metadata, embedding })
    .select('id')
    .single()

  if (error) throw createError({ statusCode: 500, message: error.message })
  return { id: data.id }
})
```

### Semantic Search Query

```typescript
// server/api/search.post.ts
export default defineEventHandler(async (event) => {
  const { query, topK = 5, threshold = 0.75 } = await readBody(event)

  // Embed the search query
  const queryEmbedding = await embedText(query)

  // Call Supabase RPC for vector similarity search
  const { data, error } = await supabase.rpc('match_documents', {
    query_embedding: queryEmbedding,
    match_threshold: threshold,  // Min similarity score (0-1)
    match_count: topK
  })

  if (error) throw createError({ statusCode: 500, message: error.message })
  return { results: data }
})
```

```sql
-- Supabase RPC function for vector search
CREATE OR REPLACE FUNCTION match_documents(
  query_embedding VECTOR(1536),
  match_threshold FLOAT,
  match_count INT
)
RETURNS TABLE (
  id UUID,
  content TEXT,
  metadata JSONB,
  similarity FLOAT
)
LANGUAGE SQL STABLE AS $$
  SELECT
    id,
    content,
    metadata,
    1 - (embedding <=> query_embedding) AS similarity
  FROM documents
  WHERE 1 - (embedding <=> query_embedding) > match_threshold
  ORDER BY embedding <=> query_embedding  -- <=> is cosine distance
  LIMIT match_count;
$$;
```

## 4. Implementation with Pinecone

```typescript
// server/utils/pinecone.ts
import { Pinecone } from '@pinecone-database/pinecone'

const pc = new Pinecone({ apiKey: process.env.PINECONE_API_KEY! })
const index = pc.index('my-app-docs')

export const upsertDocument = async (id: string, content: string, metadata: object) => {
  const embedding = await embedText(content)
  await index.upsert([{ id, values: embedding, metadata: { content, ...metadata } }])
}

export const queryDocuments = async (query: string, topK = 5) => {
  const queryEmbedding = await embedText(query)
  const results = await index.query({
    vector: queryEmbedding,
    topK,
    includeMetadata: true
  })
  return results.matches
}
```

## 5. Hybrid Search (Vector + Keyword)

Pure vector search misses exact keyword matches. Combine with BM25 full-text search for best results.

```typescript
const hybridSearch = async (query: string) => {
  // Run both searches in parallel
  const [vectorResults, keywordResults] = await Promise.all([
    vectorSearch(query),
    keywordSearch(query)  // Full-text search via Supabase/Postgres
  ])

  // Reciprocal Rank Fusion (RRF) — standard hybrid merging algorithm
  const k = 60
  const scores = new Map<string, number>()

  vectorResults.forEach((doc, rank) => {
    const score = scores.get(doc.id) ?? 0
    scores.set(doc.id, score + 1 / (k + rank + 1))
  })

  keywordResults.forEach((doc, rank) => {
    const score = scores.get(doc.id) ?? 0
    scores.set(doc.id, score + 1 / (k + rank + 1))
  })

  return [...scores.entries()]
    .sort(([, a], [, b]) => b - a)
    .map(([id]) => [...vectorResults, ...keywordResults].find(d => d.id === id)!)
}
```
