# Retrieval Augmented Generation (RAG)

RAG is the most important pattern for building production AI applications. It grounds LLM responses in your own data, solving the fundamental limitations of base LLMs: outdated knowledge, hallucination, and inability to access private information.

## 1. Why RAG

**Problem with base LLMs:**
- Knowledge cutoff date (can't answer "What's in our Q4 report?")
- No access to private data (docs, databases, emails)
- Hallucinate facts they don't know

**RAG Solution:** Before calling the LLM, retrieve relevant documents from your knowledge base and inject them into the context as ground truth.

```
User asks: "What is our refund policy for international orders?"
  ↓
RAG: Search knowledge base → Find relevant policy document
     "International orders may be returned within 30 days..."
  ↓
LLM: Answer based on retrieved document (not its training data)
```

## 2. The RAG Pipeline

```
INDEXING (one-time / on update):
Documents → Chunking → Embedding → Vector DB

RETRIEVAL + GENERATION (per query):
Query → Embed → Vector Search → Top-K Chunks → LLM Prompt → Answer
```

## 3. Full RAG Implementation in Nuxt

```typescript
// server/api/rag-chat.post.ts
import { streamText } from 'ai'
import { embedText } from '../utils/embeddings'
import { searchDocuments } from '../utils/vector-search'
import { getModel } from '../utils/llm'

export default defineEventHandler(async (event) => {
  const { question, conversationHistory = [] } = await readBody(event)

  // Step 1: Retrieve relevant documents
  const queryEmbedding = await embedText(question)
  const relevantDocs = await searchDocuments(queryEmbedding, {
    topK: 5,
    threshold: 0.72
  })

  if (relevantDocs.length === 0) {
    // Graceful fallback: No relevant context found
    return Response.json({
      answer: "I don't have information about that in my knowledge base. Please contact support.",
      sources: []
    })
  }

  // Step 2: Build context from retrieved documents
  const context = relevantDocs
    .map((doc, i) => `[Source ${i + 1}]: ${doc.content}`)
    .join('\n\n')

  // Step 3: Generate answer grounded in retrieved context
  const result = await streamText({
    model: getModel(),
    system: `You are a helpful assistant for our company.
    
Answer questions ONLY using the provided context below. 
If the context doesn't contain the answer, say "I don't have that information."
Do NOT make up information not in the context.
When citing information, reference the source number like [Source 1].

Context:
${context}`,
    messages: [
      ...conversationHistory,
      { role: 'user', content: question }
    ],
    maxTokens: 800
  })

  // Include sources in response headers for the client
  const headers = new Headers()
  headers.set('X-Sources', JSON.stringify(
    relevantDocs.map(d => ({ id: d.id, title: d.metadata?.title ?? 'Document' }))
  ))

  return result.toDataStreamResponse({ headers })
})
```

## 4. RAG Pipeline Component

```vue
<!-- components/RAGChat.vue -->
<script setup lang="ts">
import { useChat } from '@ai-sdk/vue'

const sources = ref<Array<{ id: string; title: string }>>([])

const { messages, input, handleSubmit, isLoading } = useChat({
  api: '/api/rag-chat',
  onResponse: (response) => {
    // Extract cited sources from response headers
    const sourcesHeader = response.headers.get('X-Sources')
    if (sourcesHeader) sources.value = JSON.parse(sourcesHeader)
  }
})
</script>

<template>
  <div class="rag-chat">
    <MessageList :messages="messages" />
    
    <!-- Show sources for the last response -->
    <div v-if="sources.length > 0" class="sources">
      <p class="sources-label">Sources:</p>
      <a v-for="source in sources" :key="source.id" :href="`/docs/${source.id}`">
        📄 {{ source.title }}
      </a>
    </div>

    <ChatInput v-model="input" :disabled="isLoading" @submit="handleSubmit" />
  </div>
</template>
```

## 5. RAG Quality Metrics

Monitor these to ensure your RAG pipeline is working well:

| Metric | Description | Good Threshold |
|---|---|---|
| **Retrieval Recall** | % of relevant docs actually retrieved | > 80% |
| **Answer Faithfulness** | Answer only uses retrieved context | > 95% |
| **Answer Relevance** | Answer actually addresses the question | > 85% |
| **Context Precision** | % of retrieved docs actually useful | > 70% |

Use **Ragas** (the RAG evaluation framework) to measure these automatically with LLM-as-judge.
