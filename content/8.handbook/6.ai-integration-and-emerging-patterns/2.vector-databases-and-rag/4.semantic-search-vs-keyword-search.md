# Semantic Search vs Keyword Search

Choosing between semantic (vector) search and keyword (lexical) search—or combining them—is one of the most impactful architectural decisions in a search implementation. Each has distinct strengths, and the best production systems use both.

## 1. Core Difference

| Aspect | Keyword Search (BM25/FTS) | Semantic Search (Vector) |
|---|---|---|
| **Matching** | Exact word overlap | Meaning and intent |
| **"cat" vs "feline"** | ❌ No match | ✅ Match (similar meaning) |
| **"Refund policy"** for "money back" | ❌ No match | ✅ Match |
| **Typo tolerance** | ⚠️ Fuzzy matching needed | ✅ Usually robust |
| **Exact term match** | ✅ Precise | ⚠️ May miss exact terms |
| **Speed** | ✅ Milliseconds | ⚠️ Requires ANN index |
| **Infrastructure** | Any database | Vector DB required |
| **Cost** | Free | Embedding API cost |

```sql
-- Supabase keyword search function using PostgreSQL FTS
CREATE OR REPLACE FUNCTION keyword_search(search_query TEXT, result_limit INT)
RETURNS TABLE (id UUID, content TEXT, rank FLOAT, title TEXT)
LANGUAGE SQL STABLE AS $$
  SELECT
    id,
    content,
    ts_rank(
      to_tsvector('english', content),
      plainto_tsquery('english', search_query)
    ) AS rank,
    metadata->>'title' AS title
  FROM documents
  WHERE to_tsvector('english', content) @@ plainto_tsquery('english', search_query)
  ORDER BY rank DESC
  LIMIT result_limit;
$$;

-- Index for fast FTS
CREATE INDEX ON documents USING GIN(to_tsvector('english', content));
```

## 3. When to Use Each

### Use Keyword Search When:
- Searching for **specific identifiers** (product codes, order numbers, error codes)
- **Legal/compliance** contexts where exact word match matters
- **Code search** (function names, variable names)
- **Simple lookup** queries where users know exact terms

```
"SKU-12345" → Keyword (exact match needed)
"Order #AB2024-001" → Keyword (exact match needed)
"NullPointerException" → Keyword (exact code term)
```

### Use Semantic Search When:
- **Natural language queries** ("how do I cancel my subscription?")
- **Concept-based search** (find documents about machine learning without using the exact phrase)
- **Multi-language** search (query in English, find Spanish documents with the same meaning)
- **Q&A over documents** (RAG pipelines)

```
"I want my money back" → Semantic (maps to "refund policy")
"how to delete account" → Semantic (maps to "account termination guide")
```

## 4. Hybrid Search (Best of Both)

Production systems combine both using **Reciprocal Rank Fusion (RRF)**:

```typescript
// server/api/search/hybrid.post.ts
export const hybridSearch = async (query: string, topK = 10) => {
  // Run both searches in parallel for minimum latency
  const [vectorResults, keywordResults] = await Promise.all([
    semanticSearch(query, { topK: topK * 2 }),
    keywordSearch(query, { limit: topK * 2 })
  ])

  // Reciprocal Rank Fusion (RRF)
  const k = 60 // RRF constant (standard value)
  const scores = new Map<string, { score: number; doc: any }>()

  const mergeResults = (results: any[], weight = 1) => {
    results.forEach((doc, rank) => {
      const existing = scores.get(doc.id) ?? { score: 0, doc }
      scores.set(doc.id, {
        score: existing.score + weight * (1 / (k + rank + 1)),
        doc: existing.doc ?? doc
      })
    })
  }

  // Weight semantic higher for natural language, keyword higher for exact terms
  mergeResults(vectorResults, 1.0)
  mergeResults(keywordResults, 0.8)

  return [...scores.values()]
    .sort((a, b) => b.score - a.score)
    .slice(0, topK)
    .map(({ doc }) => doc)
}
```

## 5. Search Mode Selection

Let the query type guide which search mode to use:

```typescript
const detectSearchMode = (query: string): 'keyword' | 'semantic' | 'hybrid' => {
  // Looks like an exact identifier (order number, SKU, error code)
  if (/^[A-Z0-9-]{6,}$/.test(query.trim())) return 'keyword'
  
  // Short query (1-2 words) — keyword likely more precise
  if (query.split(' ').length <= 2) return 'keyword'
  
  // Natural language question
  if (/^(how|what|why|when|where|which|who|can|is|are|do|does)\s/i.test(query)) return 'semantic'
  
  // Default: hybrid for best coverage
  return 'hybrid'
}
```
