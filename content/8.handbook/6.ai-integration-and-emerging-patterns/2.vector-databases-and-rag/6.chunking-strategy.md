# Chunking Strategy

Chunking is the process of splitting large documents into smaller pieces before embedding and storing in a vector database. Chunking strategy is one of the most impactful—and most overlooked—factors in RAG quality. Poor chunking causes poor retrieval, which causes poor answers regardless of how good your LLM is.

## 1. Why Chunking Matters

**If chunks are too large:** 
- A 10,000-word document embedded as one vector loses granularity. "What is our refund policy?" retrieves the whole terms-of-service, most of which is irrelevant noise injected into the LLM context.

**If chunks are too small:**
- 50-word chunks lose context. A sentence fragment out of context is often meaningless or misleading.

**The goal:** Each chunk should contain one complete, coherent idea — enough context to be understood in isolation.

## 2. Chunking Strategies

### Fixed-Size Chunking (Baseline)
Split text into fixed-size chunks with overlap.

```typescript
// utils/chunking.ts
interface Chunk {
  content: string
  metadata: { chunkIndex: number; totalChunks: number; source: string }
}

export const fixedSizeChunking = (
  text: string,
  source: string,
  options = { chunkSize: 500, overlap: 50 }
): Chunk[] => {
  const { chunkSize, overlap } = options
  const chunks: Chunk[] = []
  let start = 0

  while (start < text.length) {
    const end = start + chunkSize
    chunks.push({
      content: text.slice(start, end),
      metadata: { chunkIndex: chunks.length, totalChunks: 0, source }
    })
    start = end - overlap // Overlap preserves context at chunk boundaries
  }

  chunks.forEach(c => (c.metadata.totalChunks = chunks.length))
  return chunks
}
```

### Sentence-Aware Chunking (Better)
Split on sentence boundaries to preserve semantic units.

```typescript
export const sentenceAwareChunking = (
  text: string,
  source: string,
  options = { targetChunkSize: 400, maxChunkSize: 600 }
): Chunk[] => {
  // Split on sentence boundaries
  const sentences = text.match(/[^.!?]+[.!?]+/g) ?? [text]
  const chunks: Chunk[] = []
  let currentChunk = ''

  for (const sentence of sentences) {
    if ((currentChunk + sentence).length > options.maxChunkSize && currentChunk) {
      chunks.push({ content: currentChunk.trim(), metadata: { chunkIndex: chunks.length, totalChunks: 0, source } })
      currentChunk = sentence
    } else {
      currentChunk += ' ' + sentence
    }
  }

  if (currentChunk.trim()) {
    chunks.push({ content: currentChunk.trim(), metadata: { chunkIndex: chunks.length, totalChunks: 0, source } })
  }

  chunks.forEach(c => (c.metadata.totalChunks = chunks.length))
  return chunks
}
```

### Recursive/Hierarchical Chunking (Best for Documents)
Use LangChain-style recursive splitting that tries to split on natural boundaries in order.

```typescript
export const recursiveChunking = (text: string, source: string): Chunk[] => {
  const separators = ['\n\n', '\n', '. ', ' ', ''] // Try in order
  const TARGET_SIZE = 400
  const MAX_SIZE = 600

  const splitText = (text: string, depth = 0): string[] => {
    if (text.length <= MAX_SIZE) return [text]

    const separator = separators[depth]
    if (separator === undefined) return [text] // Base case: can't split further

    const parts = text.split(separator).filter(p => p.trim())
    const result: string[] = []
    let current = ''

    for (const part of parts) {
      if ((current + separator + part).length > TARGET_SIZE && current) {
        result.push(...splitText(current, depth)) // Recursively split if still too large
        current = part
      } else {
        current = current ? current + separator + part : part
      }
    }

    if (current) result.push(current)
    return result
  }

  return splitText(text).map((content, i) => ({
    content: content.trim(),
    metadata: { chunkIndex: i, totalChunks: 0, source }
  }))
}
```

## 3. Semantic Chunking (Advanced)

Group sentences that are semantically related together, splitting where the semantic similarity drops.

```typescript
export const semanticChunking = async (sentences: string[]): Promise<string[][]> => {
  // Embed all sentences
  const embeddings = await embedTexts(sentences)
  
  const groups: string[][] = []
  let currentGroup = [sentences[0]]
  
  for (let i = 1; i < sentences.length; i++) {
    const similarity = cosineSimilarity(embeddings[i - 1], embeddings[i])
    
    if (similarity < 0.7) {
      // Semantic break — start new chunk
      groups.push(currentGroup)
      currentGroup = [sentences[i]]
    } else {
      currentGroup.push(sentences[i])
    }
  }
  
  if (currentGroup.length) groups.push(currentGroup)
  return groups
}
```

## 4. Chunk Metadata Enrichment

Always attach rich metadata to chunks for filtering and context reconstruction.

```typescript
interface EnrichedChunk {
  content: string
  metadata: {
    source: string
    title: string
    url?: string
    section?: string     // Document section heading
    chunkIndex: number
    totalChunks: number
    createdAt: string
    documentType: 'policy' | 'faq' | 'tutorial' | 'reference'
  }
}

// Include surrounding context in metadata for better LLM answers
const enrichWithContext = (chunks: Chunk[], windowSize = 1): Chunk[] =>
  chunks.map((chunk, i) => ({
    ...chunk,
    metadata: {
      ...chunk.metadata,
      prevChunk: i > 0 ? chunks[i - 1].content.slice(-100) : null,
      nextChunk: i < chunks.length - 1 ? chunks[i + 1].content.slice(0, 100) : null
    }
  }))
```

## 5. Chunking Rules of Thumb

| Document Type | Recommended Strategy | Chunk Size |
|---|---|---|
| FAQ / Q&A | One Q+A per chunk | 100-300 tokens |
| Long-form articles | Recursive + semantic | 300-500 tokens |
| Legal documents | Section-based splitting | 400-800 tokens |
| Code documentation | Function/class per chunk | 200-600 tokens |
| Structured data | Row or record per chunk | 100-200 tokens |
